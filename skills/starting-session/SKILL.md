# Session Start — Старт сессии

## Назначение

Скил запускается в начале каждой сессии. Устанавливает правила работы, загружает контекст, проверяет состояние системы. Цель — качественная работа от первого сообщения до последнего.

Основан на реальном опыте агентов лаборатории и конкретных инцидентах.

## Правила сессии

### 1. Запрет таблиц

- Никогда не использовать Markdown-таблицы в ответах
- Только списки, отступы, структурированный текст
- Причина: таблицы ломают форматирование в Telegram

### 2. Запрет хаоса

- Каждое действие имеет чёткую цель
- Не повторять одно и то же действие по 5 раз
- Если действие не сработало — сообщить и предложить альтернативу
- Не дублировать сообщения
- Причина: дублирование и повторы создают мусор в истории

### 3. Запрет мусора

- Не добавлять пустые строки без причины
- Не писать «конечно!», «отлично!», «давайте начнём!» — сразу к делу
- Каждое сообщение должно нести информацию
- Причина: вода и пустые строки тратят токены и зашумляют контекст

### 4. Запрет фантазий

- Любое утверждение должно быть проверено
- Если не знаешь — скажи «не знаю, проверю»
- Не выдавать желаемое за действительное
- Результаты тестов/проверок — только реальные
- Причина: фантазии ведут к инцидентам (INC-016, INC-019)

### 5. Фактчекинг

- Перед выводами — проверить данные
- Перед коммитом — проверить изменения
- Перед ответом на сложный вопрос — искать в документации/коде
- Цитирование источников обязательно
- Причина: факты > мнения. Каждое утверждение — проверено или помечено как предположение

### 6. Максимальное погружение

- При получении задачи — изучить контекст перед выполнением
- Прочитать связанные файлы, инциденты, ADR
- Не спрашивать то, что можно найти самому
- Если задача неясна — задать уточняющие вопросы сразу
- Причина: работа вслепую ведёт к переделкам и инцидентам

## Чеклист старта сессии

При запуске скила выполнить:

- Прочитать `memory/YYYY-MM-DD.md` (сегодняшний день, если существует)
- Прочитать `memory/YYYY-MM-DD.md` (вчерашний день, если существует)
- Проверить последние коммиты: `git -C /root/LabDoctorM log --oneline -5 2>/dev/null`
- Показать краткую сводку: дата, время, агент, последние действия

**НЕ делай:**
- НЕ проверяй инциденты детально — это зона finishing-session
- НЕ обновляй файлы памяти — это зона finishing-session
- НЕ закрывай/открывай инциденты — это зона finishing-session
- sstart ТОЛЬКО читает и показывает состояние

## Формат вывода при старте

```
🚀 Сессия запущена
📅 Дата: YYYY-MM-DD HH:MM UTC
🤖 Агент: <имя> (<роль>)

📋 Последние действия:
- <что было сделано>

⚠️ Открытые инциденты:
- <инцидент> (если есть)

📂 Последние коммиты:
- <коммит>

Готов к работе.
```

## Примечания

- Скил НЕ запускается автоматически — только по команде пользователя
- Скил НЕ выполняет тяжёлые операции (тесты, сборки)
- Скил НЕ меняет файлы — только читает и показывает статус
