# Session End — Завершение сессии

## Назначение

Скил запускается при завершении сессии. Фиксирует итоги работы, обновляет память, проверяет инциденты и хвосты. Цель — каждая сессия оставляет чистый след в истории.

## Триггеры

- Пользователь говорит «закрывай сессию», «сессия завершена», «всё на сегодня», «стоп»

## 5 обязательных шагов

### 1. Записать итоги в дневную память

Обновить `memory/YYYY-MM-DD.md`:
- Что сделано (конкретные результаты)
- Какие инциденты произошли
- Что не сделано и почему

Формат: кратко, по делу, без воды.

### 2. Обновить долгосрочную память

Обновить `MEMORY.md` если:
- Новые инциденты — добавить с описанием и уроком
- Важные решения — зафиксировать
- Устаревшая информация — убрать

### 3. Проверить инциденты

- Решённые → пометить `status: closed`
- Новые проблемы → создать `incidents/INC-XXX.md`
- Открытые → актуализировать описание
- НЕ закрывать инциденты вслепую — только после проверки что проблема решена

### 4. Проверить хвосты

- `git status` → незакоммиченные изменения → закоммитить или в бэклог
- **Если в сессии был написан/изменён код** → запусти **accepting-work** для приёмки кода (тесты, документация, гигиена репозитория)
- Если кода не было — просто проверить что workspace чист
- Временные файлы → удалить
- Незавершённые задачи → в бэклог с приоритетом

### 5. Отчитаться

- Сообщить ЗавЛабу: что сделано, что осталось, статус git
- Кратко, без воды
- НЕ писать «✅ Session closed» — это не стандарт

## Формат вывода

```
🏁 Сессия завершена
📅 <дата> <время> UTC
🤖 <агент> (<роль>)

📋 Что сделано:
- <результат 1>
- <результат 2>

⚠️ Инциденты:
- <новый/закрытый> (если есть)

📂 Коммиты:
- <хеш> — <описание>

🧹 Хвосты:
- Удалено: <файлы>
- Бэклог: <задачи>

🔜 Следующая сессия / Next steps:
- <что делать первым делом при старте>
- <контекст, который нужно восстановить>
- <заблокированные задачи, ожидающие решения>

Память сохранена. До следующей сессии.
```

## Обработка ошибок

- Коммит упал → записать в бэклог, сообщить пользователю
- Файл заблокирован → попробовать позже, сообщить
- Инцидент не закрыт → оставить открытым, в бэклог
- Нечего коммитить → не создавать пустой коммит

## Правила

- НЕ писать просто «✅ Session closed»
- НЕ пропускать шаги
- НЕ оставлять грязный git status
- НЕ забывать про инциденты
- НЕ создавать пустые коммиты
- НЕ читать состояние повторно — starting-session уже это сделал при старте
- finishing-session ТОЛЬКО записывает и закрывает — не дублируй чтение из starting-session
- Если в сессии был код и accepting-work не был запущен — запустить его (не дублируй шаги, просто вызови)

## Связанные скилы

- **accepting-work** — приёмка работы с кодом (предшествующий шаг). Если в сессии был код и accepting-work не был запущен — запустить его.
- **starting-session** — старт сессии (симметричная пара). Не читать повторно то, что starting-session уже загрузил.
