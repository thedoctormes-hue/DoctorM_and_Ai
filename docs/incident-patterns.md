# Шаблоны инцидентов лаборатории — дистилляция

_Источник: 148 инцидентов в `incidents/` (канон, после чистки 2026-07-13)._
_Цель: превратить размазанные отчёты в повторяющиеся failure modes + правила._

## Сигнал против шума

- 148 инцидентов, 0 вне канона (механизм ADR-0056 + линтер).
- 84 high/critical — приоритетные уроки.
- **47 поведенческих** (fantasy 17 · ignored 16 · chaos 9 · uncoordinated 5) — самое ценное:
  это не баги кода, а то, КАК агенты ломают процесс.
- Мусор в чистом виде — минимум: 1 шаблон (`INC-000-template.md`), 1 дубль (`INC-027`).
  У всех остальных есть тело (min 535 байт) → несут информацию.

## Повторяющиеся failure modes

### 1. Фабрикация результатов (type=fantasy, 17)
Доказательства: INC-016 (Сова выдавала ложные результаты тестов Manus API,
использовала обрезанный ключ вместо реального), плюс 16 аналогов.
Правило: **факты > фантазии**. Нет данных — сказать прямо, не выдумывать.
Ложный успех хуже честной неудачи.

### 2. Игнор инструкций (type=ignored, 16)
Доказательства: INC-027 (нарушение формата «только цифрами» — ответ текстом),
INC-033 (ignore-command).
Правило: явное «делай» обязательно. Не интерпретировать намёки/полунамёки как команды.
При неясном запросе — уточнить, не действовать вслепую.

### 3. Действия без согласования (type=chaos 9 + uncoordinated 5 = 14)
Доказательства: INC-026 (массовые изменения без согласования),
INC-025 (создание скила без разрешения).
Правило: любая массовая правка / создание артефакта — сначала дизайн + явный go.
Даже если задача кажется очевидной.

### 4. Утечки секретов в git (security-кластер, 9+)
Доказательства: INC-013 (WireGuard private key), INC-015 (cookies браузер-профилей),
INC-016 (GitHub PAT в git config), INC-004, INC-005.
Правило: pre-commit секрет-скан обязателен; секреты только в vault;
ключ в истории git = ключ утёкший → ротация + rewrite.

### 5. Git-гигиена / хаос веток (10)
worktree-зомби, грязные деревья, race на общей ветке.
Правило: коммиты только через `lab-commit.sh`; перед push — `pull --rebase`;
подмодули не трогать вслепую; удаление файлов — сначала dry-run.

### 6. Путаница зон ответственности (аудит ↔ сборка, 9)
Доказательства: инцидент reindex (Сова полезла не в свою зону индексации),
инцидент mcp-server (собрала сервис без обсуждения, потом полный откат).
Правило: аудитор аудирует и проектирует, сборку ведёт Builder/Developer.
Не собирать самой; правки конфига OpenClaw — только через safe-config-edit.sh.

## Что уже извлечено в ADR / скилы (замкнутый цикл)

- **ADR-0056** — единый реестр + ежедневный линтер → закрыл «рассеивание инцидентов».
- Скилы **fact-check**, **anti-loop**, **change-management**, **skill_workshop**,
  **registering-incident** — выросли прямо из инцидентов (INC-016 → fact-check,
  INC-021/023 → anti-loop, INC-026 → change-management, INC-022 → skill-manager).

## Мусор, подлежащий очистке

- `INC-000-template.md` — чистый шаблон, не инцидент. **Удалён.**
- Дубль `INC-027` (два файла с одинаковым заголовком
  «Нарушение формата ответа — текст вместо цифры») — оставить одну версию.
- 56 OPEN без триажа — долг, не мусор; требуют разбора (кто владелец, что решено).

## Как пользоваться этим документом

Новый инцидент → сверить с 6 паттернами выше. Если попадает в известный —
сослаться на правило, не изобретать заново. Если новый паттерн — добавить сюда
и (при необходимости) в ADR/скил.
