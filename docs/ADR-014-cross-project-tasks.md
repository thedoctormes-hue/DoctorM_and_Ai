# ADR-014: Кросс-проектные задачи

## Статус

Active — 2026-06-24

## Контекст

Задачи, затрагивающие несколько проектов или лаборантов, не имели чёткого процесса. Каждый агент работает в своей зоне, но нет механизма координации на стыке зон.

## Решение

Определить чёткий маршрут для кросс-проектных задач и роль Штрейкбрехера как координатора.

## Определение

Задача считается **кросс-проектной**, если выполняется хотя бы одно условие:

- Затрагивает **более одного проекта** (например, API + фронтенд + миграция БД)
- Требует **компетенций более одного лаборанта** (например, бэкенд + инфраструктура)
- Влияет на **архитектуру** лаборатории в целом
- Имеет **dependencies** между задачами разных лаборантов

## Маршрут кросс-проектной задачи

```
Инициатор (любой лаборант / ЗавЛаб)
    │
    ▼
Штрейкбрехер (Technical Lead)
    │  ┌─ Определяет зоны ответственности
    │  ├─ Назначает ответственных по каждой зоне
    │  ├─ Согласовывает техническое решение
    │  └─ Контролирует зависимости
    │
    ▼
Исполнители (лаборанты по своим зонам)
    │
    ▼
Штрейкбрехер — приёмка и интеграция
```

## Правила

### Штрейкбрехер

1. **Не подменяет владельцев.** Координирует — не делает за других. Каждый лаборант работает в своей зоне.
2. **Согласовывает техническое решение.** Перед началом работы — единое видение архитектуры.
3. **Контролирует зависимости.** Если задача А блокирует задачу Б — отслеживает порядок.
4. **Эскалация на ЗавЛаба.** Если конфликт зон или приоритетов не удаётся разрешить — эскалация.

### Исполнители

5. **Сообщают о блокерах сразу.** Обнаружили зависимость от другой зоны → уведомляют Штрейкбрехера.
6. **Не меняют чужие зоны.** Даже если «быстро поправить» — уведомить координатора.
7. **Обновляют статус.** Переводят задачу в соответствующий статус на канбане.

### ЗавЛаб

8. **Стратегические решения.** Приоритеты между проектами, распределение ресурсов — за ЗавЛабом.

## Формат кросс-проектной задачи в myrmex.json

```json
{
  "id": "BL-XXX",
  "project_id": "proj-target",
  "title": "...",
  "assignee_id": "streikbrecher",
  "status": "backlog",
  "tags": ["cross-project"],
  "dependencies": ["BL-YYY", "BL-ZZZ"],
  "description": "## Зоны ответственности\n- Агент1: ...\n- Агент2: ...\n\n## Критерий готовности\n- ..."
}
```

Требования:
- **tag `cross-project`** — обязательный
- **dependencies** — все блокирующие задачи
- **В description** — раздел «Зоны ответственности» и «Критерий готовности»

## Метрики

- **Время координации:** от создания до назначения ответственных — не более 1 часа
- **Блокеры:** среднее время снятия блокера между зонами — не более 4 часов
- **Успех:** задача завершена без переделок между зонами

## Альтернативы (отклонены)

- **Каждый сам за себя** — приводит к конфликтам на стыках
- **ЗавЛаб координирует всё** — слишком дорого, ЗавЛаб должен думать стратегией
- **Без формального процесса** — уже пробовали, не работает
