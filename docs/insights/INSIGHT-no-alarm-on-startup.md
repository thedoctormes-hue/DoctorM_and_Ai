# 🚫 Не Кричать АЛЯРМ При Старте

## Проблема

При старте сессии Ворон (и другие агенты) читали state-файл lab-monitoring и выдавали список алертов по сервисам. 9 алертов по причине что лаборанты работают с проектами — это норма.

## Правила

- При старте сессии НЕ запускать чтение lab_monitoring_alerts.json
- Сервисы могут лежать до 24 часов — это норма при активной работе
- Ежедневный дайджест отправляется через systemd timer — этого достаточно
- Если ЗавЛаб явно просит — использовать `--critical` и фильтровать group != "services"

## Почему это важно

- ЗавЛаб: «Меня заебал спамить @labmonitoringrobot»
- Шум отвлекает от реальных задач
- Сервисные алерты (snablab, saas-api, prometheus, telegram-webhook и т.д.) = норма при активной работе лаборантов
