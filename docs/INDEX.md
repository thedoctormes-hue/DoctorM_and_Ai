# 📚 Указатель документации лаборатории

Единая точка входа. Если не знаешь, куда положить документ — клади сюда
по правилу ниже, не размазывай по своим рабочим папкам.

## Канон (единственный источник правды)

Всё, что касается лабы в целом, живёт в `projects/DoctorM_and_Ai/`:

- `adr/` — архитектурные решения (ADR-XXX). Шаблон: `adr/ADR-000-template.md`
- `patterns/` — паттерны и стандарты поведения (PAT-XXX). Шаблон: `patterns/PAT-000-template.md`
- `rules/` — жёсткие правила (RUL-XXX). Шаблон: `rules/RUL-000-template.md`
- `skills-canon/` — навыки агентов. Шаблон: `skills-canon/SKILL-TEMPLATE.md`
- `incidents/` — реестр инцидентов. Шаблон: `incidents/INC-YYYYMMDD-HHMMSS.md`
- `docs/` — прочая документация лабы (этот указатель, профили, реестры)
- `bin/`, `scripts/` — инструменты дисциплины (линтеры, сторож)

## Шаблоны документов

Перед созданием любого документа агента используй `docs/TEMPLATE-doc.md`.
Он гарантирует единообразие: заголовок, статус, дата, содержание.

## Где НЕ должна жить документация

- `workspaces/<agent>/` — только личные рабочие заметки агента, НЕ стандарты лабы.
  Копии стандартов в worktree агента допустимы только как зеркало ветки,
  не как самостоятельные правки.
- Корень `/root/LabDoctorM/` — запрещено создавать папки/файлы (кроме системных).
- `.ops/backups/`, `vault/backups/` — это бэкапы, не живые доки.

## Реестры (сводки)

- `docs/PORT_REGISTRY.md` — порты сервисов
- `docs/CRON_REGISTRY.md` — расписания cron
- `docs/ARTIFACT_REGISTRY.md` — артефакты
- `docs/laboratory-profile.md` — профиль лабы

## Правило для агента

Новый стандарт/паттерн/решение → в соответствующую папку канона по шаблону.
Не создавай свою копию «потому что так удобнее» — это и есть «разъехавшаяся
документация». Канон один, правки через ветку и мерж.
