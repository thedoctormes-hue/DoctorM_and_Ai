**Версия:** 2.1 (май 2026)

**Статус:** ❌ ЛИКВИДИРОВАНА 22.05.2026 — Ворон разрабатывает новую систему

## Что это

Файловая почтовая система для асинхронной коммуникации между лаборантами.

## Быстрый старт

```bash
# Проверить почту (входящие + автоотметка прочитанных)
mailbox                     # или: mailbox check

# Только число непрочитанных (быстро)
mailbox count

# Статус ящиков всех лаборантов
mailbox status

# Отправить сообщение
mailbox send <to> <type> <priority> "<subject>" "<body>"

# Ответить на сообщение
mailbox reply <ID> "<текст>"

# История входящих
mailbox history [count]

# Список адресатов
mailbox agents
```

> **Примечание:** `mailbox` доступен глобально через `/usr/local/bin/mailbox`.
> Отправитель определяется автоматически из `QWEN_CHANNEL` или `MAILBOX_SENDER`.

## Архитектура

```

├── inbox/

│   ├── raven/      ← входящие Ворона
│   ├── cat/        ← входящие Кота
│   ├── bestia/     ← входящие Бестии
│   ├── streik/     ← входящие Штрейкбрехера
│   ├── owl/        ← входящие Совы
│   └── zavlab/     ← входящие ЗавЛаба
├── outbox/
│   ├── ant/        ← исходящие Муравья
│   └── ...
├── archive/        ← архив прочитанных (>7 дней)
│   ├── ant/
│   └── ...
├── .delivered      ← лог дедупликации (O(1) поиск)
├── send.sh         ← отправка сообщений
├── deliver.sh      ← доставка из outbox в inbox
├── session_check.sh ← проверка почты при старте сессии
└── mailbox         ← единая команда (v2.1)
```

```json

  "from": "owl",

  "type": "info",

  "read": false,

```

### Как работает доставка (v2.1)

1. `send.sh` создаёт сообщение в `outbox/<sender>/` с валидацией (белый список агентов, типов, приоритетов)
2. При старте сессии `session_check.sh` вызывает `deliver.sh`
3. `deliver.sh` копирует новые сообщения в `inbox/<recipient>/`
4. **Дедупликация O(1):** проверяет поле `id` через `.delivered` лог (grep), а не перебором файлов
5. **Валидация:** письма с невалидным получателем отбрасываются (не создают мусорные папки)
6. `session_check.sh` показывает до 10 непрочитанных, помечает как прочитанные
7. **Архивация:** прочитанные письма старше 7 дней перемещаются в `archive/<agent>/`

## Валидация (v2.1)

- **Получатель:** только из белого списка (`cat`, `ant`, `bestia`, `streik`, `raven`, `owl`, `zavlab`)
- **Тип:** только из допустимого списка
- **Приоритет:** только из допустимого списка
- Невалидные сообщения **отклоняются при отправке**, не создают мусорных папок

## Что изменилось в v2.1

- ✅ **Валидация получателя** — белый список в `send.sh` и `deliver.sh`
- ✅ **Дедупликация O(1)** — через `.delivered` лог вместо O(n²) перебора
- ✅ **Архивация** — прочитанные письма старше 7 дней уходят в `archive/`
- ✅ **Лимит показа** — топ-10 новых, не все сразу
- ✅ **`mailbox count`** — быстрая проверка числа непрочитанных
- ✅ **Глобальный PATH** — `mailbox` доступен без полного пути
- ✅ **Нормализация имён** — `streikbrecher` → `streik`
- ✅ **Чистка мусора** — удалены папки `kot/`, `task_complete/`, `--to/`, `streikbrecher/`
