# 🚫 Протокол «Не Фантазировать»

## Принцип

При анализе любой проблемы — сначала grep/cat/ls, потом выводы. Никаких «очевидно» без данных.

## Правила

- Любой анализ начинать с `grep`/`cat`/`ls`
- Не использовать слова «очевидно», «ключевое решение» без фактической базы
- При расхождении данных с ожиданием — сначала прочитать ВСЕ поля записи
- Каждый симптом проверять отдельно (дифференциальная диагностика)
- Не выдвигать одну теорию и подтверждать её — проверять каждый случай

## Примеры нарушений

- Увидел 4 ошибки pytest → выдал «очевидные» фиксы → оказался неправ → 3 итерации переделки
- Увидел устаревший last_seen у 4 агентов → решил что все из-за сломанного API → не проверил source/status/dir → причины разные

## Почему это важно

- Фантазии вместо фактов = потеря времени + неверные решения
- ЗавЛаб: «сначала проверка, потом выводы. Если не знаешь — скажи.»
