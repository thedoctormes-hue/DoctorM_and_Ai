# 🏛 Консилиум экспертов v5

## Назначение
Коллективное решение сложных проблем через привлечение экспертов из штата лаборатории.

## ⚠️ ОБЯЗАТЕЛЬНО: ПЕРЕД РАБОТОЙ
1. Прочитай файл `~/.qwen/rules.md` и следуй этим правилам
2. Сформулируй проблему чётко перед сбором экспертов

## Императивные инструкции

### Шаг 0: Классификация
Определи режим:
- 🔴 **EMERGENCY** — сервис лежит, деградация, потеря данных → 3 эксперта, 5 мин
- 🟡 **INCIDENT** — инцидент severity >= P2 → полный состав, пост-мортем 24ч
- 🟢 **STANDARD** — архитектурный тупик, миграция, стратегия → полный цикл
- 🔵 **MIGRATION** — переезд сервера, смена стека → STANDARD + чеклист миграции

### Шаг 1: Преамбула (обязательна)
- **Контекст проекта:** что за проект, архитектура, стек
- **Симптом:** что именно не работает
- **Ограничения:** что нельзя ломать
- **Попытки решения:** что уже пробовали
- **Критерий успеха:** измеримое условие

### Шаг 2: Сбор участников
1. Сканируй `/root/.qwen/agents/` по релевантности теме
2. Минимум 2 реальных сотрудника
3. Веса: Primary (3x), Secondary (1x), Advisory (0.5x)
4. Обязательно включи диссентиента (атакует консенсус)

### Шаг 3: Позиции экспертов
Каждый эксперт выдаёт: **Тезис → Аргумент → Риск если игнорировать** (2-3 предложения)

### Шаг 4: Атака диссентиента
Обязан найти 2-3 слабых места в консенсусе. Замечания интегрируются в решение.

### Шаг 5: Синтез решения
Объедини позиции с учётом весов и замечаний диссентиента.

### Шаг 6: Верификация
- Совместимость со стеком
- Обратная совместимость
- Возможность отката
- Smoke-test / health-check

### Шаг 7: Отчёт
Выведи полный протокол консилиума.

## Чеклист качества
- [ ] Совместимость: не ломает >10% существующего кода
- [ ] Нет пивота: сохраняет стек и архитектуру
- [ ] Нет костылей: нет any, TODO-fix-later, хардкода
- [ ] Верифицируемо: есть реальный пример
- [ ] Минимальность: нет лишних зависимостей
- [ ] Диссентиент учтён


## 🔮 Маркировка инсайтов

При обнаружении инсайта в процессе работы, в конце вывода добавляй маркер:

```
[INSIGHT: <тип>] <краткое описание>
[layer: <rules|memory|skills|backlog|agents>]
[source: <откуда инсайт>]
```

## Границы
- Максимум 3 итерации → эскалация пользователю
- Не сошлись → разделить на подзадачи
- Принцип: консенсус > голосование
