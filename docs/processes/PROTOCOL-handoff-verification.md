# 🔍 Протокол Верификации HANDOFF

## Проблема

HANDOFF и CHECKPOINT содержат снимок состояния на момент закрытия сессии. Между сессиями данные меняются — HANDOFF может показывать устаревшую информацию.

## Правила

- При старте сессии: после чтения HANDOFF — ВСЕ пункты с утверждениями о состоянии верифицировать фактом
- Формирование списка задач — ТОЛЬКО после верификации
- Если HANDOFF содержит задачу — проверить актуальность перед включением в список
- Если HANDOFF содержит диагностику проблемы — проверить, решена ли она

## Примеры устаревших данных

- «4 лаборанта отсутствуют в agents» — к моменту старта уже синхронизировано
- Статус сервисов — мог измениться за время простоя
- Git состояние — другие агенты могли закоммитить

## Почему это важно

- Агенты тратят время на диагностику уже решённых проблем
- Систематический паттерн — доверие к HANDOFF без проверки
- ЗавЛаб требует: «сначала проверка, потом выводы»
