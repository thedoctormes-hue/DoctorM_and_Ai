# ✅ Протокол Коммитов в Конце Сессии

## Принцип

Коммитить только после проверки работоспособности сервисов. Не «сделал → закоммитил», а «сделал → проверил → закоммитил».

## Правила

- Перед коммитом убедиться, что сервис работает
- Запустить тесты, если есть
- Проверить логи на ошибки
- Если сервис не запускается — НЕ коммитить, а чинить
- После коммита проверить `git log -1` на корректность автора

## Почему это важно

- Коммит сломанного кода = сломанный main
- Другие агенты начинают работу с неработающим кодом
- Откат в общем репо = потеря чужой работы
