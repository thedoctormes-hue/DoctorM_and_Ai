# 🔐 Протокол Коммитов LabDoctorM

## Канонические имена агентов

В сообщениях коммитов используем канонические имена (agent ID), не русские прозвища:

- `kotolizator` — Оркестратор
- `antcat` — Муравей
- `bestia` — Бестия
- `raven` — Ворон
- `owl` — Сова
- `streikbrecher` — Штрейкбрехер
- `dominika` — Доминика
- `mangust` — Мангуст

ЗавЛаб — НЕ лаборанкт, не входит в список.

## Правила коммитов

- Коммить ТОЛЬКО через `scripts/lab-commit.sh`
- Формат: `chore(project): описание` — без указания автора в сообщении
- Перед коммитом проверять `git branch --show-current`
- После коммита проверять `git log -1 --format='%an <%ae>'`
- НЕ переписывать историю (rebase/amend) в общем активном репо
- НЕ использовать `git add .` из корня — только конкретные пути
- НЕ коммитить из чужих веток

## Ветки

- Создавать ветки ТОЛЬКО от `main`
- Имя ветки = `agent-id/feature-name`
- ЗавЛаб/оркестратор мержит feature-ветки в main

## Worktree

- Каждый агент работает в своём worktree
- Не переключаться на чужой worktree
- Не коммитить в main напрямую (кроме ЗавЛаба)

## Почему это важно

- Race condition на git identity → коммиты под чужой личностью
- `git add -A` из корня → 108 файлов из 6+ проектов в одном коммите
- Коммиты из чужих веток → код попадает не туда
