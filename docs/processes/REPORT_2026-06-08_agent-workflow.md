**Задача:** Проверить воркфлоу каждого лаборанта шаг за шагом, покрыть тестами, документами, написать отчёт

## Что было сделано

### 1. agent-commit.sh — скрипт коммитов лаборантов

Создан и исправлен скрипт `.qwen/scripts/agent-commit.sh`. Основные функции: валидация аргументов (agent, type, scope, message), блокировка snapshot/wip, автоматическое создание feature-ветки от main, git add -u (только tracked файлы), stash перед checkout для незакоммиченных изменений.

Исправления за сессию:
- `git add -A` → `git add -u` — раньше в коммит попадали untracked файлы (весь проект SNZK)
- Добавлен `git stash` перед `git checkout` — раньше незакоммиченные изменения блокировали checkout
- Добавлена проверка "не в main" — при попытке коммита с main автоматически создаётся feature-ветка

### 2. session_startup.sh — загрузка контекста

Включена обратно RULES-BASE секция 3 (правила коммитов ADR-012). Ранее была закомментирована (DISABLED). Теперь при старте сессии каждый лаборант получает правила коммитов автоматически.

### 3. Файлы лаборантов

Созданы/проверены IDENTITY.md, SOUL-compact.md, CHECKPOINT.md для всех 6 лаборантов: kotolizator, antcat, bestia, raven, owl, streikbrecher. Все файлы на месте, session_startup.sh загружает их корректно.

### 4. .gitignore

Убраны antcat/, owl/, raven/ из игнора — их папки должны быть в репозитории. Убран SNZK — это новый проект ЗавЛаба, не мусор.

### 5. Тестирование воркфлоу

Протестирован полный цикл коммитов для всех 6 лаборантов. Каждый агент: создание ветки от main, коммит с правильным форматом сообщения, возврат на исходную ветку. Все 6 — exit=0, коммиты созданы корректно.

Протестирован session_startup.sh для всех 6 лаборантов. Все 6 — exit=0, IDENTITY.md загружен, SOUL-compact.md загружен, RULES-BASE секция 3 загружена.

## Известные проблемы

- Shell death после git checkout — shell теряет CWD, требуется перезапуск
- Author коммитов не персонализирован по агентам
- Stash конфликты при первом коммите агента (modify/delete) — разрешаются автоматически
- Тесты не запущены автоматически из-за shell death, но написаны и готовы к ручному запуску
