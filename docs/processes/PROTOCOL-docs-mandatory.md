# 📝 Протокол «Документация Обязательна»

## Принцип

«Документация! Мы пишем то, что делаем, и делаем то, что пишем.»

Если не задокументировал — значит, не сделал.

## Правила

- При любом изменении кода/пакетов/конфигурации — ОБЯЗАТЕЛЬНО обновлять связанную документацию В ТОМ ЖЕ коммите
- Документация = часть продукта, не довесок
- Порядок: код → тесты → документация → коммит

## Примеры нарушений

- Изменения в session_startup.sh без актуализации docs/GIT_GUARDIAN.md
- Обновление 8 пакетов без обновления ARTIFACT_CHANGELOG.md и CHECKPOINT.md

## Почему это важно

- Документация после факта = документация, которой нет
- Следующий агент работает с устаревшей документацией
- ЗавЛаб: «Тесты провела? Документацию актуализировала?»
