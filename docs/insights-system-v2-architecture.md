# 🦊 Архитектура Insights System v2 — Лучшая в мире система ловли инсайтов

> Доминика, 18.06.2026 — результат глубокого исследования паттернов, архитектур и лучших практик

## 1. Анализ почему текущая система генерировала мусор

### Корневая проблема: Garbage In → Garbage Out

| Компонент | Что делало | Почему мусор |
|-----------|------------|--------------|
| **Входные данные** | Сканировало trajectory.jsonl (чаты агентов) | Чаты = поток сознания, не факты |
| **Regex-паттерны** | Ловили "error", "decided", "found" в любом контексте | "Я решил пойти в магазин" = "decision" 🤡 |
| **Дедуп** | MD5 от text.strip().lower() | Только точные дубли, семантические — нет |
| **Очередь** | JSON-файл, полный парсинг при каждой операции | O(n) при каждом add |
| **Консолидация** | Jaccard similarity по словам | "сервер упал" ≈ "сервер не работает" — не кластеризуются |
| **Cron** | Запускал всё каждые 30 мин без разбора | Умножал мусор |

### Ключевой инсайт: система не была сломана. Она работала ровно как спроектирована. Проблема — в дизайне.

## 2. Лучшие практики (из исследования)

### 2.1 Reflexion Pattern (Shinn et al., 2023)
**Идея:** агент учится на ошибках через многопопытные циклы с памятью рефлексий.

```
ПОПЫТКА 1 → ОШИБКА → РЕФЛЕКСИЯ → ПАМЯТЬ → ПОПЫТКА 2 → УСПЕХ
```

- GPT-4 coding benchmark: 80% → 91%
- Критически важно: рефлексия должна проходить **внешнюю верификацию** (тесты, тулы)
- Без верификации — "degeneration of thought" (агент закрепляет ошибку)

### 2.2 GraphRAG (Microsoft, 2024)
**Идея:** граф знаний вместо векторного поиска для multi-hop reasoning.

- Vector RAG: изолированные чанки → теряет связи между фактами
- GraphRAG: сущности + связи → может отвечать на вопросы через обход графа
- **HybridRAG** (вектор + граф) — лучший баланс

### 2.3 Agent Skills (Anthropic, 2025)
**Идея:** прогрессивное раскрытие — агент загружает только нужные навыки.

- Не всё в system prompt, а индекс + загрузка по требованию
- Аналогия для инсайтов: не грузить всю очередь, а искать по индексу

### 2.4 Feedback Mechanisms (IJCAI 2025 Survey)
**4 типа обратной связи для агентов:**
1. **Internal feedback** — самооценка (reflection)
2. **External feedback** — верификация через тулы/тесты
3. **Multi-agent feedback** — другие агенты проверяют
4. **Human feedback** — человек оценивает

**Комбинация internal + external превосходит любой по отдельности.**

### 3.2 Компоненты

#### A. Источники инсайтов (Input Layer)

| Источник | Что даёт | Фильтр |
|----------|----------|--------|
| **Ручной ввод** | Структурированные находки от агентов | Quality Gate |
| **Reflection** | Ошибки → рефлексия → урок (из циклов агентов) | Внешняя верификация |
| **Cross-agent review** | Другие агенты находят проблемы в коде/решениях | Структурированный формат |
| **Security scan** | Утечки секретов, уязвимости, анти-паттерны | Пониженный порог |
| ~~Авто-майнинг чатов~~ | ~~УБРАН~~ | Генерирует мусор |

#### B. Quality Gate (Фильтр качества)

```python
def quality_gate(insight: dict) -> bool:
    """True = инсайт проходит фильтр."""
    
    # 1. Структура: должен иметь контекст, проблему, решение
    if not all(k in insight for k in ['context', 'problem', 'recommendation']):
        return False
    
    # 2. Длина: 120-500 символов
    content = insight.get('content', '')
    if len(content) < 120 or len(content) > 500:
        return False
    
    # 3. НЕ лог чата (LLM-классификатор)
    if is_chat_log(content):  # regex + LLM fallback
        return False
    
    # 4. НЕ "я решил пойти в магазин"
    if not has_actionable_insight(content):
        return False
    
    return True
```

#### C. Семантический дедуп (через bge-m3 + FAISS)

```python
import faiss
import numpy as np

class SemanticDedup:
    def __init__(self, dimension=1024):
        self.index = faiss.IndexFlatIP(dimension)  # cosine similarity
        self.insights = []
    
    def is_duplicate(self, embedding: list[float], threshold=0.85) -> bool:
        if self.index.ntotal == 0:
            return False
        vec = np.array([embedding]).astype('float32')
        scores, _ = self.index.search(vec, 1)
        return scores[0][0] >= threshold
    
    def add(self, embedding: list[float], insight: dict):
        vec = np.array([embedding]).astype('float32')
        self.index.add(vec)
        self.insights.append(insight)
    
    def find_similar(self, embedding: list[float], threshold=0.5) -> list:
        """Находит похожие для объединения (merge)."""
        if self.index.ntotal == 0:
            return []
        vec = np.array([embedding]).astype('float32')
        scores, indices = self.index.search(vec, 10)
        return [
            self.insights[i] 
            for s, i in zip(scores[0], indices[0])
            if s >= threshold and s < 0.85
        ]
```

#### D. База знаний (SQLite)

```sql
CREATE TABLE insights (
    id TEXT PRIMARY KEY,
    content TEXT NOT NULL,
    type TEXT NOT NULL,  -- error/decision/finding/pattern/security
    source TEXT NOT NULL,  -- кто создал
    context TEXT,  -- контекст сессии
    problem TEXT,  -- описание проблемы
    recommendation TEXT,  -- рекомендация
    frequency INTEGER DEFAULT 1,  -- сколько раз встречалось
    confidence TEXT DEFAULT 'medium',  -- low/medium/high
    status TEXT DEFAULT 'new',  -- new/verified/consolidated/artifact
    session_id TEXT,
    agent_pair TEXT,
    embedding BLOB,  -- 1024-dim bge-m3 вектор
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    verified_by TEXT,  -- кто верифицировал (агент или человек)
    artifact_id TEXT  -- ссылка на ADR/pattern/rule если конвертирован
);

CREATE INDEX idx_type ON insights(type);
CREATE INDEX idx_status ON insights(status);
CREATE INDEX idx_frequency ON insights(frequency DESC);
```

#### E. Конвертация в артефакты

| Условие | Артефакт | Кто оформляет |
|---------|----------|---------------|
| frequency ≥ 5 + type = "error" | **ADR** (архитектурное решение) | Котолизатор |
| frequency ≥ 3 + type = "pattern" | **Pattern** (повторяющийся паттерн) | Котолизатор |
| type = "error" + есть решение | **Rule** (правило) | Муравей |
| type = "security" | **Incident** (инцидент безопасности) | Сова |
| frequency = 1 + type = "finding" | **Finding** (сырая находка) | Ворон (исследует) |

## 4. Цикл жизни инсайта

```
СОЗДАНИЕ → ФИЛЬТР → ДЕДУП → БАЗА → АНАЛИЗ → ВЕРИФИКАЦИЯ → АРТЕФАКТ
   │          │         │        │        │          │           │
   │     Quality    bge-m3    SQLite   GraphRAG   Cross-agent   ADR/
   │      Gate     + FAISS    +vec    кластеры    review       Rule
   │          │         │        │        │          │           │
   └──────────┴─────────┴────────┴────────┴──────────┴───────────┘
```

### Пример полного цикла

1. **Агент** находит ошибку в деплое: "При деплое на staging падает healthcheck из-за timeout 30s"
2. **Quality Gate**: ✅ структурировано, 120-500 символов, не лог чата
3. **Семантический дедуп**: ищем в FAISS → score=0.3 → новый
4. **База**: добавляем в SQLite + FAISS индекс
5. **Анализ**: через неделю — frequency=7 (каждый деплой падает)
6. **Верификация**: Мангуст проверяет граф → связано с PAT-003 (healthcheck pattern)
7. **Артефакт**: Котолизатор оформляет ADR-XXX: "Увеличить healthcheck timeout до 60s для staging"

## 5. Роли агентов в новой системе

| Агент | Роль | Что делает |
|-------|------|------------|
| **Все агенты** | Создатели | Пишут структурированные инсайты через Quality Gate |
| **Доминика** | Scout + Координатор | Проектирование системы, координация, приоритизация |
| **Мангуст** | Аналитик | Граф связей, перекрёстный анализ, верификация через код |
| **Сова** | Аудитор | Security scan, ревью инсайтов, валидация |
| **Муравей** | Билдер | Код системы (майнер, дедуп, база, API) |
| **Котолизатор** | Оркестратор | Приоритизация → ADR/patterns/rules, финальный дайджест |
