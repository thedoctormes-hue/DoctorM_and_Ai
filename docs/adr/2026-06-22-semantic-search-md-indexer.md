# ADR-0045: Semantic Search .md File Indexer

**Дата:** 2026-06-19
**Статус:** Accepted
**Автор:** Муравей (Antcat)
**Связанные коммиты:** `fe74d247`

## Контекст

Семантический поиск лаборатории (Context API, bge-m3) работал только с артефактами и памятью агентов. Документация (ADR, паттерны, правила, инциденты, спеки) не индексировалась, что делало поиск по «как у нас принято делать X» неполным.

## Решение

Создан **lab-index-md.py** — индексатор markdown-файлов:

- Источники: `adr/`, `patterns/`, `rules/`, `specs/`, `incidents/`, `docs/`, `projects/`, `workspaces/`
- **Resumable:** SHA-256 хэш для инкрементальной переиндексации
- **Chunking:** большие файлы разбиваются на чанки ~500 символов
- **Фильтрация:** пропуск `node_modules`, `.git`, `.pytest_cache`
- **Хранение:** SQLite `/root/.openclaw/memory/md-files.sqlite`

## Последствия

- Полное покрытие документации в семантическом поиске
- Инкрементальная переиндексация (только изменённые файлы)
- Зависимость от SQLite — требуется контроль размера БД

## Альтернативы

- Полная переиндексация при каждом запуске (отвергнуто: медленно на большом объёме)
- Векторное хранилище напрямую (отвергнуто: SQLite достаточен для текущего масштаба)
