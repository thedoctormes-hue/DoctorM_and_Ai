# ADR-032: openclaw.json как единственный источник реестра агентов

## Контекст
При вопросах о реестре агентов возникает путаница: кто-то смотрит в файловую систему (`workspaces/`), кто-то в `myrmex.json`, кто-то в IDENTITY.md. Это приводит к рассогласованию данных.

## Решение
Для вопросов о реестре агентов **единственный источник истины — `/root/.openclaw/openclaw.json`**, а не файловая система.

Файлы `IDENTITY.md`, `SOUL.md`, `AGENTS.md` в workspaces — это bootstrap-файлы для запуска сессий, а не источники правды о конфигурации.

## Следствия
- При добавлении/удалении агента меняем openclaw.json, а не файлы на диске
- myrmex.json синхронизируется с openclaw.json, а не наоборот
- Файловая система = bootstrap (запуск), openclaw.json = runtime (конфигурация)
