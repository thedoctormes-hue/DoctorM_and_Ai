# ADR 0042 — ONNX Embedder для семантического поиска

**Дата:** 2026-06-22
**Статус:** Принят
**Ответственный:** КотОлизатор

## Контекст

Семантический поиск по знаниям лаборатории требует:
- Низкой латентности (<500ms)
- Стабильной работы 24/7
- Независимости от внешних сервисов
- Оптимизации под наши ТТХ (4 CPU, 8GB RAM)

## Решение

Заменить Ollama + bge-m3 на ONNX Runtime + Qwen3-Embedding-0.6B.

### Почему ONNX, а не Ollama

| Критерий | Ollama + bge-m3 | ONNX + qwen3 |
|----------|-----------------|--------------|
| RAM | ~3GB (daemon + модель) | ~2.9GB (модель) |
| Зависимости | Ollama daemon | Только Python |
| OOM risk | Высокий | Низкий |
| Скорость | ~120ms (но зависал) | ~250ms (стабильно) |
| Отказоустойчивость | Нет | systemd + watchdog |

### Почему Qwen3-Embedding-0.6B

- MTEB multilingual: 64.33 (на 7.9% лучше bge-m3)
- Размер 0.6B — помещается в RAM
- Лучше работает с русским текстом
- Поддерживается через optimum + onnxruntime

## Архитектура

```
.md файлы → lab-index-md.py → SQLite → build-faiss.py → FAISS
                                                    ↓
lab-search-proxy (8081) → ONNX Embedder (8082) → Qwen3-0.6B ONNX
```

## Последствия

**Положительные:**
- Нет зависимости от Ollama
- Стабильная работа через systemd
- Автоматическая переиндексация через timer
- Лучшее качество на русском тексте

**Отрицательные:**
- Медленнее Ollama в 2 раза (250ms vs 120ms)
- Требует ручной пересборки FAISS после индексации
- Модель 2.3GB занимает значительную часть RAM

## Альтернативы

- **Chroma** — полноценная векторная БД, но требует больше RAM
- **sqlite-vec** — расширение SQLite, но менее зрелое
- **Ollama + bge-m3** — отклонена из-за нестабильности и OOM
