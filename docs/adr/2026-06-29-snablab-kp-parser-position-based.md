# ADR: Переход KP-парсера на позиционный парсинг

- **Дата:** 2026-06-29
- **Статус:** accepted
- **Автор:** Бестия
- **Проект:** snablab

## Контекст

KP-парсер (коммерческие предложения) содержал 845 строк кода с регулярными выражениями и эвристиками. Код был сложен в поддержке, содержал дублирование логики и хрупкие regex-паттерны.

## Решение

Рефакторинг на позиционный парсинг:
- извлечение данных по фиксированным позициям в строках вместо regex
- сокращение с 845 до 170 строк (−80%)
- сохранение публичного API: `parse_kp_file()` / `parse_kp_text()`
- все тесты зелёные после рефакторинга

## Последствия

- код стал читаемее и поддерживаемее
- позиционный подход менее гибкий при изменении формата КП
- если формат КП изменится — потребуется обновление констант позиций
- тесты покрывают основные сценарии
