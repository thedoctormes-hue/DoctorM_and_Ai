# ADR-0043: API Hub Search System — оркестратор с key rotation

**Дата:** 2026-06-21
**Статус:** Accepted
**Автор:** Бестия (Bestia)
**Связанные коммиты:** `74a44550`, `318f9ed3`

## Контекст

Лаборатории нужна единая система поиска через множество провайдеров (Tavily, Firecrawl, TinyFish, SearXNG и др.). Каждый провайдер имеет несколько API-ключей. Требуется маршрутизация запросов по типу задачи, параллельный опрос провайдеров и ротация ключей.

## Решение

Создан **api-hub** — система поиска с тремя компонентами:

- **search-orchestrator.sh** — маршрутизация по типу задачи (factual / content / dynamic / broad / deep_research)
- **search-parallel.sh** — параллельный поиск с дедупликацией URL
- **search-check-keys.sh** — валидация всех 15 API-ключей

**Архитектура ротации:** cyclic round-robin (каждый запрос использует следующий ключ, по кругу). При 429 — ключ пропускается, до 5 попыток (все ключи). Это маскирует мультиаккаунтный паттерн: провайдер видит запросы от разных ключей.

**Fallback:** локальный SearXNG при недоступности внешних провайдеров.

## Последствия

- Единая точка входа для всех поисковых запросов агентов
- Устойчивость к rate limit (5 ключей на провайдер)
- Параллельный опрос снижает latency
- Зависимость от gitignored `search-keys.yaml` — ключи не в git

## Альтернативы

- Error-based rotation (отвергнуто: медленнее, не маскирует паттерн)
- Один ключ на провайдер (отвергнуто: нет отказоустойчивости)
