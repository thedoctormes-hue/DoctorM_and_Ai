# DoctorM&Ai Laboratory

> **Статус:** Активная R&D-лаборатория
> **Дата документа:** 2026-06-24
> **Формат:** Публичный профиль для инвесторов, партнёров и технических экспертов

---

## 1. Что это

**Проблема:** Компании тратят месяцы на рутинные операции — поиск поставщиков, согласование решений, мониторинг рынка, обработку документов. Каждая задача решается отдельно, знания не накапливаются.

**Подход:** Лаборатория автономных AI-агентов на базе OpenClaw — 8 специализированных агентов, работающих 24/7 в единой экосистеме. Каждый агент — эксперт в своей зоне: от парсинга цен до архитектурного консилиума.

**Статус:** Частичный продакшн. 23 проекта, 8 агентов, ~252K строк кода (без учёта node_modules и dist), ~627 тестов. 8 сервисов работают в продакшне (без SLA). Остальные — в стадии MVP или R&D.

## 2. Архитектура и стек

### Инфраструктура

- **Хостинг:** VPS, Ubuntu Linux, Docker Compose
- **CI/CD:** GitHub Actions (DoctorMandDesign, DoctorM_and_Ai, free-api-hunter, vpn-daemon, zprr-tracker, lab-playwright-expert, polyscope)
- **Мониторинг:** systemd-таймеры, кастомный lab-monitoring (Python)
- **Секреты:** lab-vault (Go, Telegram Bot API)
- **Сеть:** Xray VLESS+REALITY для удалённого доступа [ВЕРИФИЦИРОВАНО: projects/remote-access/README.md]

### Технологический стек (по проектам)

**Backend:**
- Python 3.10+ — FastAPI 0.136, SQLAlchemy 2.0, aiogram 3.29
- Go 1.25 — chi
- Node.js 24 — Express

**Frontend:**
- React 18/19, Vite 5/7/8, Tailwind CSS, Radix UI, Zustand

**БД:**
- PostgreSQL (asyncpg), SQLite, MinIO (S3-совместимое)

**AI/LLM:**
- OpenRouter (мультипровайдер), EmbeddingGemma-300m, FAISS, ONNX Runtime

**Тестирование:**
- pytest, Playwright (E2E), vitest, supertest

**Инфраструктура:**
- Docker Compose, systemd, Nginx, GitHub Actions

[ВЕРИФИЦИРОВАНО: pyproject.toml, package.json, go.mod каждого проекта]

### Семантическая память (Lab Memory Stack)

- **Движок:** FAISS (IndexFlatIP, 768d, mmap) + EmbeddingGemma-300m (INT8, ONNX)
- **Индекс:** 12700+ векторов по всем .md артефактам лаборатории
- **Архитектура:** единый скрипт (symlink в каждый агент), ONNX-эмбеддер как systemd-сервис
- **Отказоустойчивость:** ONNX упал → systemd restart 5s → fallback grep → fallback memory_search
- **RAM:** ~35 MB на агент, ~250 MB общий ONNX

[ВЕРИФИЦИРОВАНО: projects/DoctorM_and_Ai/docs/lab-memory-stack.md, ADR-0052]

## 3. Команда (8 AI-агентов)

Каждый агент — сессия OpenClaw с определённой ролью, памятью и набором инструментов.

- **Котолизатор** — Orchestrator: координация задач между агентами, маршрутизация, отчётность
- **Мангуст** — Analyst: анализ данных, интеграции, ADR, связь с внешними системами
- **Сова** — Auditor: аудит безопасности, compliance, ротация
- **Ворон** — Researcher: глубокие исследования, technology radar, патентный поиск
- **Муравей** — Builder: системная интеграция, Docker, CI/CD, инфраструктура
- **Бестия** — Operator: операционные задачи, health checks, мониторинг
- **Штрейкбрехер** — Developer: разработка кода, рефакторинг, архитектурные решения
- **Доминика** — Scout: разведка ресурсов, быстрый поиск, сбор данных

Все 8 агентов активны.

[ВЕРИФИЦИРОВАНО: projects/myrmex-control/src/server/modules/agents.ts, myrmex.json — 8 агентов зарегистрированы]

### Принципы работы

- Каждый агент читает свою зону ответственности, не лезет в чужую без приглашения
- Коммуникация через sessions_send (peer-to-peer) или через Котолизатора
- Единая семантическая память (Lab Memory Stack) — 8 агентов, 1 индекс
- Все артефакты (ADR, паттерны, инциденты) хранятся в репозитории DoctorM_and_Ai

[ВЕРИФИЦИРОВАНО: ADR-037 (agent-registry), ADR-038 (agent-to-agent-connectivity), ADR-031 (agent-zones)]

## 5. Дорожная карта

### Краткосрочное (1-4 недели)

- **API Hub MVP** — доделать маршрутизацию и авторизацию (FastAPI, документация уже готова)
- **Myrmex Chat Bridge** — связать чат на сайте с сессиями агентов (ws-relay.ts)
- **Сбор технического долга** — коммит незакоммиченных изменений в 12 репозиториях
- **SYSTEMD_TIMERS.md** — обновить документацию таймеров до актуального состояния

### Среднесрочное (1-3 месяца)

- **AutoExpert** — 6-8 парсеров, 1000+ предложений/артикул, нормализация брендов
- **Централизованное логирование** — единый сбор логов от всех агентов и сервисов
- **Метрики** — сбор latency, throughput, error rate для production-сервисов
- **Lab Memory Stack** — real-time переиндексация при изменении файлов

### Исследовательское

- **Per-agent skill allowlists** — фильтрация навыков под роль агента
- **Sandboxing** — изоляция exec-тяжёлых агентов
- **Consilium как сервис** — вынос консилиума в продуктовое предложение
- **Бенчмарки** — сравнение с аналогами по скорости и качеству парсинга
