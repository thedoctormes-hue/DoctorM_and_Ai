# ADR-039: Heartbeat для всех агентов лаборатории

## Статус
Принято (2026-06-20)

## Контекст
До 20.06.2026 только 2 из 9 агентов (kotolizator, owl) имели heartbeat. Остальные 7 агентов были мертвы между сообщениями пользователя — не выполняли фоновые задачи, не мониторили инфраструктуру, не обслуживали память.

## Решение
Включить heartbeat для всех агентов с разными интервалами по приоритету:

| Агент | Интервал | Приоритет | Обоснование |
|-------|----------|-----------|-------------|
| bestia | 30 мин | Критический | Инфраструктура, серверы, бэкапы |
| kotolizator | 30 мин | Критический | Координация, аудит конфигов |
| owl | 30 мин | Критический | Аудит безопасности, качество |
| mangust | 60 мин | Высокий | Анализ документации, ADR |
| dominika | 60 мин | Высокий | Разведка ресурсов |
| raven | 60 мин | Высокий | Внешний мониторинг |
| antcat | 120 мин | Средний | Боты, мониторинг (экспериментальный) |
| streikbrecher | 120 мин | Средний | Разработка (экспериментальный) |
| main | — | — | Дефолтный, не нужен |

## Критерии частоты
- **30 мин** — критичные агенты (инфраструктура, координация, безопасность)
- **60 мин** — важные агенты (аналитика, разведка)
- **120 мин** — экспериментальные агенты (меньше шума)

## HEARTBEAT.md
Каждый агент получает персонализированный HEARTBEAT.md с задачами по его зоне ответственности. Формат:
- При каждом heartbeat — быстрые проверки
- Раз в 6 часов — глубокие проверки (с timestamp tracking)
- Раз в 12 часов — обслуживание

## Последствия
- Агенты работают проактивно, без команд пользователя
- Раннее обнаружение проблем (падения, диски, ошибки)
- Автоматическое обслуживание памяти
- Небольшой рост потребления токенов (heartbeat каждые 30-120 мин)

## Риски
- Ложные срабатывания → митигация: HEARTBEAT_OK если нечего делать
- Рост токенов → митигация: lightContext: true в defaults
- Шум в Telegram → митигация: только значимые результаты
