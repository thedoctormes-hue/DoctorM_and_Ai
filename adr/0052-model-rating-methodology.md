# ADR-0052: Стандарт составления рейтинга LLM-моделей

## Контекст

Free API Hunter отслеживает модели множества провайдеров. Для сравнительного анализа необходим единый, повторяемый метод оценки качества моделей, дающий числовой score. Рейтинг строится через `live API (GET /v1/models)` без использования веб-скрапинга прайсинговых страниц.

## Решение

### Данные для рейтинга

Источник: `GET /v1/models` (live API провайдера). Извлекаются:
- `name` / `id` — идентификатор
- `context_length` / `max_context_length` — размер контекстного окна
- `capabilities` — набор возможностей (vision, reasoning, function calling, и т.д.)
- `top_provider.max_completion_tokens` — максимальное число токенов в ответе
- Дополнительные поля специфичные для провайдера

### Формула скоринга

```
Score = Tier_score + Context_score + Capability_score
```

#### Tier_score (модельный ранг)

| Tier | Примеры | Баллы |
|------|---------|-------|
| Флагман (назв contains "large", "plus", "max") | command-a-plus, mistral-large, gpt-4o | +100 |
| Оптимальный (основная линейка средних/новых) | command-a, mistral-medium, mistral-small | +80 |
| Базовая/легковесная (tiny, 7b, 8b, mini, nano) | ministral-3b, mistral-tiny, llama-8b | +60 |
| Специализированная (embed, moderation, ocr, asr) | embed-v3, moderation, ocr, transcribe | +40 |

Примечание: точные названия tiers зависят от провайдера. Универсальный эвристический подход — по ключевым словам в названии модели и рейтингу контекста.

#### Context_score

| Контекст | Баллы |
|----------|-------|
| ≥ 400K | +50 |
| ≥ 200K | +40 |
| ≥ 100K | +30 |
| ≥ 32K | +20 |
| < 32K | +10 |

#### Capability_score

| Возможность | Баллы |
|-------------|-------|
| reasoning (рассуждения) | +10 |
| vision (изображения) | +10 |
| function_calling (tools) | +10 |
| audio (аудио понимание) | +10 |
| fine_tuning | +5 |
| fim (fill-in-the-middle) | +5 |

### Процесс составления рейтинга

1. **Получить** `GET /v1/models` через API-ключ
2. **Дедуплицировать** — убрать старые версии, оставить самую свежую (по контексту и дате версии)
3. **Классифицировать** по модальностям (chat, vision, reasoning, embed, asr, и т.д.) на основе capabilities и названия
4. **Применить скоринг** к каждой модели
5. **Отсортировать** по убыванию score
6. **Вывести топ-5** для каждой модальности

### Критерии включения в рейтинг

| Критерий | Включать | Исключать |
|----------|----------|-----------|
| Модель отвечает `200` на `GET /v1/models` | ✅ | ❌ |
| Ключ имеет доступ к chat completions | ✅ | ❌ |
| Провайдер заблокирован по TLS/сети | ❌ (фиксировать как недоступный) | — |
| Embedding/специализированные модели | ✅ (отдельная модальность) | — |

### Формат отчёта

Для каждого провайдера:
1. Сводка: общее число моделей, уникальных, по модальностям
2. Топ-5 для каждой модельности (в порядке убывания score)
3. Полный список уникальных моделей (отсортировано по score)
4. Ключевые выводы

## Пример: Mistral (2026-06-24)

- Всего: 75 → 36 уникальных
- Модальности: reasoning (5), vision (5), tools (5), chat (5), finetune (5), ocr (1), audio (2), tts (1), fim (1), moderate (1)
- Топ: magistral-medium-latest (score 240), mistral-medium-latest (230), mistral-small-latest (230)

## Пример: Cohere (2026-06-24)

- Всего: 20 уникальных
- Модальности: chat (9+), reasoning (1), vision (2), embed (8), asr (1), translate (1)
- Топ: command-a-plus-05-2026 (150), command-a-reasoning-08-2025 (145), command-a-03-2025 (120)

## Последствия

- Единый метод для всех провайдеров Free API Hunter
- Возможность сравнивать модели между провайдерами по score
- Воспроизводимость: любой агент может повторить рейтинг по этому стандарту
- Ограничение: score не отражает реальное качество (бенчмарки), только технические характеристики
