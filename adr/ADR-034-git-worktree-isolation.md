# ADR-034: Git worktree как стандарт изоляции рабочих зон агентов

## Контекст
Агенты работают в общих репозиториях. Одновременные изменения из разных агентов приводят к конфликтам, перезаписи и потере данных. Коммиты от имени неправильных агентов — частая проблема.

## Решение
Каждый агент получает **изолированный git worktree** через скрипт `create-worktree.sh`:
```bash
bash /root/LabDoctorM/shared/git-rules/create-worktree.sh <agent> projects/<project>
```

**Правила:**
- Запрещено использовать `git checkout` в общих каталогах
- Коммиты только через `lab-commit.sh <agent> -m "..."`
- Push только в свою ветку
- Merge в main — только через согласование с командой

## Следствия
- Полная изоляция рабочих зон агентов
- Исключение коммитов от чужого имени
- Чистая история git
- Требуется обучение новых агентов
