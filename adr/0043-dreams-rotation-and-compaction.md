# ADR-0043: DREAMS Rotation and Compaction

## Контекст

DREAMS — механизм OpenClaw для автоматической консолидации памяти агентов (dreaming cycle, запуск каждые 24ч в 03:00 UTC). Генерирует 3 типа файлов:

- `DREAMS.md` — нарративное саммари (поэтический формат, для human review)
- `memory/.dreams/events.jsonl` — структурированные события (machine-readable)
- `memory/.dreams/session-corpus/` — полные транскрипты сессий (сырой текст)

### Текущее состояние (22.06.2026)

- 8 агентов × ~350KB .dreams = **~4.6MB сырых данных**
- DREAMS.md: **903 строки** суммарно (8-16KB на агента)
- session-corpus: **~1.8MB** (полные транскрипты, дублируют /tmp/openclaw/*.log = 35MB)
- events.jsonl: **~2.8MB** (90-170 событий на агента, без лимита)
- **3 failed systemd-сервиса** (artifact-aging, artifact-audit, onnx-reindex) — ротация артефактов не работает

### Проблемы

1. **DREAMS.md — литературный журнал без редактора.** 70% содержимого — поэзия и метафоры, непригодные для извлечения технических инсайтов. Никто не читает.
2. **Нет разницы между «важно» и «не важно».** Архитектурные решения и «server hummed its midnight lullaby» хранятся с одинаковым приоритетом.
3. **Четверное дублирование данных.** Одни и те же данные лежат в: DREAMS.md, events.jsonl, session-corpus, /tmp/openclaw/*.log.
4. **Нет лимитов и ротации.** Файлы растут бесконечно. В отличие от MEMORY.md (bootstrap budget, сигнал переполнения), DREAMS.md не имеет механизма ограничения.
5. **Session-corpus хранит полные транскрипты** вместо саммари. У Доминики и Совы — по 316KB за 4 дня.
6. **artifact-aging не работает.** Таймер тикает, сервис мёртв. Артефакты не стареют, не архивируются.

### Исследование

Проведено глубокое исследование (4 поисковых провайдера, 15+ источников):

- **Microsoft Agent Framework:** compaction при 64% context window → memory sync, при 85% → compression
- **LangChain Deep Agents:** 4 стратегии — write, select, compress, isolate
- **Mem0 State of AI Agent Memory 2026:** «memory staleness» — одна из 6 открытых проблем production AI
- **Reddit r/AI_Agents:** TTL на память — стандартная практика для удаления stale data
- **Eric Broda (Medium):** «selective promotion, not blanket storage»
- **JetBrains Research:** эффективный context management — core research problem, а не engineering detail

## Решение

### 1. DREAMS.md — удалить

**Обоснование:** файл никто не читает, технические инсайты тонут в поэзии, потребителя нет.

**Действие:** удалить DREAMS.md у всех 8 агентов. Инсайты и решения записывать напрямую в MEMORY.md и memory/YYYY-MM-DD.md — туда, где они реально используются.

### 2. Session-corpus — удалить

**Обоснование:** полные транскрипты уже хранятся в /tmp/openclaw/*.log (35MB). Дублирование бессмысленно.

**Действие:** удалить session-corpus/ у всех агентов. Если нужна история сессий — читать из /tmp/openclaw/*.log.

### 3. Events.jsonl — оставить с лимитом

**Обоснование:** единственный полезный формат — machine-readable, компактный, пригоден для автоматической обработки.

**Действие:** установить лимит **500 последних событий** на файл. При превышении — удалять старые.

### 4. Настроить ротацию /tmp/openclaw/*.log

**Обоснование:** 35MB логов без ротации — потенциальная проблема при 77% заполнении диска.

**Действие:** настроить logrotate: хранить 7 дней, сжимать, удалять старше 30 дней.

### 5. Починить artifact-aging.service

**Обоснование:** скрипт artifact_aging.py написан и работает, но сервис не запускается. Артефакты (ADR, PAT, incidents) не стареют, не архивируются.

**Действие:** диагностировать и починить сервис, проверить таймер.

## Ожидаемый результат

- **Экономия места:** ~4.6MB → ~0.5MB (только events.jsonl с лимитом). Минус ~80% мусора.
- **Ускорение:** меньше файлов = быстрее bootstrap агентов
- **Чистота:** технические инсайты в MEMORY.md, а не в поэзии
- **Работающая ротация:** artifact-aging запускается, артефакты стареют

## Риски

- **Потеря «поэтического» контекста.** Если ЗавЛаб читает DREAMS.md для понимания «настроения» агентов — потеряем это. Митигейция: оставить последнюю запись перед удалением.
- **Зависимость от OpenClaw dreaming config.** Нужно проверить, можно ли отключить генерацию DREAMS.md через конфиг, или удаление файла вызовет ошибку.
