# ADR-028: openclaw.json как единственный источник истины для реестра агентов

**Статус:** proposed
**Дата:** 2026-06-16
**Автор:** owl (Сова)
**Связанный инцидент:** INC-018

## Контекст

Инцидент INC-018: агент owl трижды выдал неверное количество агентов (26 → 24 → 8), каждый раз представляя число как факт. Корень — агент использовал файловую систему (find, визуальный подсчёт) вместо чтения openclaw.json.

Проблема системная: любой агент при вопросе «сколько агентов?» пойдёт искать файлы на диске, считать директории, читать IDENTITY.md — и каждый получит свой «правильный» ответ.

## Решение

**openclaw.json → agents.list — единственный авторитетный источник для реестра агентов.**

Любой вопрос о количестве, именах, статусах, каналах агентов — оттуда.

## Правило для всех агентов

При вопросах вида:
- «Сколько агентов?»
- «Какие агенты существуют?»
- «Кто является агентом?»
- «Какой агент за что отвечает?»

**Первичный источник:** `openclaw.json → agents.list` (для реестра) + `bindings` (для маршрутизации).

**Не использовать:** подсчёт файлов, директорий, IDENTITY.md — это вторичные источники и могут расходиться с реальным конфигом.

## Обоснование

- openclaw.json — конфигурация системы, она определяет что запущено
- IDENTITY.md — декларация агента, она может быть создана но не подключена
- Файловая система — отражение проектов, не все проекты = агенты
