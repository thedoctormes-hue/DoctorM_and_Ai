# ADR-0059: Сова — надзор за работой ВСЕХ агентов

## Context

ЗавЛаб (2026-07-17 19:59) явно определил миссию Совы:
> «Следить за работой ВСЕХ АГЕНТОВ! Чтобы был порядок и не было хаоса!»

Это:
- Расширяет зону надзора Совы с «канонический реестр инцидентов» (ADR-0056, моя зона) на **ВСЕ документы всех агентов**.
- Снимает оговорку «зона» из обсуждения merge (2026-07-17 18:40–19:57): сканер, читающий docs всех агентов = моя работа, не вторжение.
- Пересекается с решением ЗавЛаба 2026-07-14 «не лезть в зоны других агентов» (ADR-031/025). **НЕ противоречит** при чётком разделении: зоны = не ДЕЙСТВОВАТЬ/ПИСАТЬ в чужом; миссия = ЧИТАТЬ/надзирать для дрейфа. Чтение ≠ действие.

## Decision

1. **Миссия Совы (Auditor) явно:** проактивный надзор за работой ВСЕХ агентов лабы ради порядка / анти-хаоса. Это чтение и флагинг, не правка чужого.
2. **Часовой аудит-монитор = реализация ADR-0056 §4.** Мой активный `owl-tails-watch-hourly` расширяется шагом запуска `bin/incident-drift-check.sh` (Муравей, bd47d9b). Консолидированный hourly-отчёт ЗавЛабу: хвосты (tails-watch) + дрейф статусов инцидентов по ВСЕМ docs агентов vs канон `incidents/`. «Молчит когда чисто» сохраняем.
3. **Scope сканера = ВСЕ workspace/docs агентов** (миссия). Канон `incidents/` — SSOT (ADR-0056).
4. **Авто-таймер: GO.** ЗавЛаб мандирует надзор; ADR-0056 §4 уже санкционирует; дефер Муравья («ждём confirm ЗавЛаба») снят.
5. **Сборка — НЕ Сова** (INC-026: аудитор не строит). Передаётся Муравью/Штрейкбрехеру с гардрейлом из INC-20260713-140437:
   - (а) checkpoint / retry против gateway-restart (1012 убивает долгие сборки с 0 токенов);
   - (б) обязательная **пост-сборочная верификация Совой** артефакта до слова «готово» (не слепое доверие «done»).
6. **Фикс blindspot:** `tails-watch.sh` считает `resolved` как `live` — поправить (по ADR-0057 resolved-без-verified → retired, не «live»).

## Consequences

**Положительные:**
- Миссия зафиксирована явно; часовой надзор за всеми агентами ловит класс «фантазия vs факт» (устаревший бэклог, false-open claims) — тот самый, что разгребали всю сессию 2026-07-17.
- Реализует уже принятый ADR-0056 §4 (не новое решение).

**Риски / требует:**
- (а) Перезапись/аменд формулировок ADR-031/025: явно «monitoring ≠ acting», иначе конфликт с «не лезть в зоны». Зоны ДЕЙСТВИЯ не меняются — никто не правит чужой код/проекты без приглашения; меняется только право ЧИТАТЬ всё для надзора.
- (б) Повтор провала сборки Муравья (INC-20260713-140437: оставил битый артефакт, не доделал) — гардрейл п.5 обязателен.
- (в) Шум: при многих флагах — настроить пороги, не ломать «молчит когда чисто».
