# ADR-002: writeState() должен мержить, а не перезаписывать

## Статус
proposed

## Контект
`writeState()` в `myrmex.js` полностью перезаписывает `myrmex.json` из памяти процесса. Поля, которые не загружены в память (например `users`, `refresh_tokens`), теряются при записи. Это привело к инциденту INC-20260512111530 — потеря всех пользователей и открытый setup на 3 минуты.

## Решение
Изменить `writeState()` на мерж с существующим файлом:
1. Прочитать текущий `myrmex.json` с диска
2. Прочитать состояние из памяти
3. Смержить: поля из памяти перезаписывают поля файла, но поля файла которых нет в памяти — сохраняются
4. Записать результат

## Последствия
- ✅ Потеря данных при рестарте сервиса невозможна
- ✅ Прямое редактирование файла безопасно
- ⚠️ Нужно аккуратно мержить массивы (tasks, agents) — не дублировать элементы
- 📋 Реализация: BL-012

## Альтернативы

- **Полная перезапись (текущий):** Просто, Потеря данных, Неприемлемо
- **Мерж:** Безопасно, Сложнее реализовать, ✅ Выбран
- **SQLite вместо JSON:** Атомарность, Миграция, сложность, На будущее
