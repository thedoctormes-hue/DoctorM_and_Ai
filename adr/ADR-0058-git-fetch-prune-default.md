# ADR-0058: Обязательный `fetch.prune=true` в git-конфиге лаборатории

## Контекст

Аудит git-гигиены 2026-07-16 (по сигналу ЗавЛаба «ужас сколько висячих хвостов»)
выявил ~50 веток `owl/*` + ~13 локальных `backup/owl-*`, выглядевших как незакрытые
хвосты в `git branch -r`.

Проверка через `git ls-remote` показала: **ни одной из этих `owl/*` веток на GitHub
уже не существует**. Все были удалены с remote давно (после мержа июньской чистки
git-hygiene 2026-06-29). Локально висели **фантомные remote-tracking рефы**
(`refs/remotes/origin/owl/*`), потому что git по умолчанию НЕ удаляет устаревшие
remote-tracking ветки при обычном `fetch`/`pull`.

### Корневые причины
1. **Главный:** отсутствие авто-prune. `git fetch`/`pull` без `--prune` накапливает
   мёртвые remote-tracking рефы неограниченно. За месяцы это дало иллюзию полусотни
   «хвостов».
2. **Вторичный:** patch-id аудит (`git cherry`) сравнивал против устаревшего локального
   `origin/main`, из-за чего древние ветки казались «живыми», хотя их security-ценность
   давно переприменена в main другими коммитами.
3. **Организационный:** мерж в main у Совы идёт через Кота/ЗавЛаба; после мержа ветки
   удаляли на remote, но локальный кэш клонов никто не прунил.

**Реальной невлитой работы — ноль.** Проблема была чисто в устаревшем локальном кэше git.

## Решение

1. Установить глобально: `git config --global fetch.prune true`
   (`fetch.pruneTags false` — теги не трогаем автоматически).
   → При каждом `fetch`/`pull` мёртвые remote-tracking рефы удаляются автоматически.
2. Разовая зачистка накопленного выполнена: `git remote prune origin` по всем 24 проектам
   (`owl/*` фантомы → 0) + удаление 13 локальных `backup/owl-*` (содержимое уже в main).

## Последствия

- **Плюс:** `git branch -r` впредь отражает реальное состояние remote; ложные «хвосты»
  не копятся; аудиты гигиены не тратят время на фантомы.
- **Нейтрально:** prune удаляет ТОЛЬКО remote-tracking рефы удалённых веток; локальные
  ветки и незапушенная работа не затрагиваются.
- **Осторожно:** `fetch.pruneTags` намеренно `false` — авто-удаление тегов опасно.

## Проверка (verified)

- `git config --global --get fetch.prune` → `true` (2026-07-16).
- После prune: `owl/* + backup/owl` рефов во всех projects/* → **0**.
- `git ls-remote --heads origin 'refs/heads/owl/*'` во всех репо → пусто (подтверждает,
  что фантомы, а не реальные ветки).
