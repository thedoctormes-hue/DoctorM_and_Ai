# ADR-0056 (DRAFT): Единый реестр инцидентов и дисциплина хранения

## Контекст

Аудит 2026-07-13 выявил: инциденты лаборатории размазаны по **12 директориям** — корневой
`/root/LabDoctorM/incidents/` (6), `projects/DoctorM_and_Ai/incidents/` (89), и per-agent
`workspaces/<agent>/incidents/` (48, у 8 агентов). Всего **146 живых файлов**, из них
**53 (36%) без поля `status`**, множество дублей и устаревших меток `open`.

Корень рассинхрона (fact-check подтверждён):

1. **Скил `registering-incident` сам противоречив.** В разделе «Хранилище» он декларирует
   абсолютный канонический путь `/root/LabDoctorM/projects/DoctorM_and_Ai/incidents/...`
   (строка 94), НО в коде генерации имени пишет **относительно**:
   `FILENAME="incidents/$(date ...)-${HASH}.md"` (строка 105). Относительный путь резолвится
   в CWD агента → файл попадает в `workspaces/<agent>/incidents/`. Даже compliant-агент рассеивает.
2. **Нет принуждения к использованию скила.** 53/146 файлов без `status` → созданы вручную,
   мимо скила (который требует frontmatter id/timestamp/category/type/severity/status/agent/title).
3. **Нет автоматической проверки.** QUALITY_STANDARDS §3 требует frontmatter для `.md`, PAT-005
   запрещает засорение корня — но нет ни правила «единый реестр инцидентов», ни сканера,
   ловящего нарушения. Рассинхрон остаётся незамеченным (см. INC-20260713-071955 — doc-drift).

## Решение

### 1. Единый канонический реестр (обязательно)
- Единственное место хранения инцидентов: **`/root/LabDoctorM/projects/DoctorM_and_Ai/incidents/`**.
- `workspaces/<agent>/incidents/` и `/root/LabDoctorM/incidents/` **объявляются недопустимыми**
  для хранения инцидентов (допускаются только как временные рабочие заметки, не реестр).
- Имя файла: `INC-YYYYMMDD-HHMMSS-<hash>.md` (hash обязателен, из скила).

### 2. Обязательный frontmatter (строгий)
Каждый инцидент ДОЛЖЕН содержать: `id, timestamp, category, type, severity, status, agent, title`.
Отсутствие `status` = инцидент невалиден (ловится линтером).

### 3. Фикс скила `registering-incident` (enforcement point #1)
Строка 105: `FILENAME="incidents/..."` → `FILENAME="/root/LabDoctorM/projects/DoctorM_and_Ai/incidents/..."`.
Один абсолютный путь устраняет рассеивание для ВСЕХ compliant-агентов. Дополнительно: запретить
создание инцидентов вне канонической директории (guard в скиле).
→ *Задача Builder/ЗавЛаб (skill-improvement). Не моя зона сборки.*

### 4. Автоматический линтер / doc-drift checker (enforcement point #2)
Cron (расширяет требование INC-20260713-071955) периодически сканирует ВСЕ `incidents/` в лабе и:
- флагает файлы вне канонической директории → предлагает перенос;
- валидирует frontmatter (наличие `status`, допустимые значения);
- детектит дубликаты (по id/заголовку/содержимому);
- при расхождении поднимает инцидент или шлёт отчёт ЗавЛабу.
→ *Задача Builder/ЗавЛаб (build). Не моя зона сборки.*

### 5. Распространение на документацию (цель «в идеале всю документацию»)
QUALITY_STANDARDS уже требует frontmatter для `.md` и хранение проекта в `projects/<project>/`
(PAT-005). Добавить: документация проекта живёт в `projects/<project>/docs/` (канонически),
тот же линтер проверяет наличие/корректность frontmatter у всех `*.md` лабы.

### 6. Мандат на использование скила
Правило: инцидент создаётся ТОЛЬКО через `registering-incident` (после фикса п.3), не руками.
Закрепить в QUALITY_STANDARDS.

## Последствия

- Плюсы: один источник правды по инцидентам; машиночитаемый статус; исчезает «невидимость»
  открытых инцидентов (старт-скил больше не sampl'ит через `tail`).
- Минусы/риски: нужна миграция существующих 146 файлов (скрипт консолидации уже создал
  `projects/DoctorM_and_Ai/incidents/_consolidated/` с `PROVENANCE.json` — основа для merge);
  линтер — это new-сервис (нужен go ЗавЛаба на сборку).
- Не закрывает поведенческий корень (агенты всё ещё могут творить руками) — п.4+п.6 это
  компенсируют технически, но дисциплина «спроси прежде чем действуй/утверждай» остаётся
  культурной (см. INC-026, INC-016, INC-20260712, INC-20260713-002844).

## Решения ЗавЛаба (2026-07-13, все утверждены)
1. **Канон утверждён:** единый реестр — `projects/DoctorM_and_Ai/incidents/`. `workspaces/*/incidents/` и `/root/LabDoctorM/incidents/` объявлены вне закона.
2. **Миграция выполнена:** `_consolidated/` слит в канон (147 файлов), размазанные копии удалены (бэкап: `.ops/incident-scatter-backup-20260713.tar.gz`).
3. **Go на сборку:** фикс скила `registering-incident` (enforcement #1) + авто-линтер (enforcement #2) — задача передана Builder (Муравей).
4. **Пуш Доминики `c54d4dd` легализован** (INC-2 закрыт как accepted).
