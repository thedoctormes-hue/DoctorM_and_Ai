# Стек LLM-моделей лаборатории на OpenRouter

## Статус
Proposed

## Дата
2026-05-22

## Автор
Сова (по запросу ЗавЛаба)

## Контекст

Лаборатория использует OpenRouter как единый API-шлюз к LLM-моделям. На момент аудита (22.05.2026) OpenRouter предлагает 23 собственные модели, из которых лаборатория использует минимальное количество. Нет документированного стандарта — какой моделью пользоваться для каких задач.

Текущее состояние:
- Ворон работает на `openrouter/owl-alpha` (бесплатная, 1M контекст)
- Hype Pilot использует `google/gemini-2.5-flash` + `llama-3.3-70b` через OpenRouter
- Остальные лаборанты не имеют явных назначений моделей
- Многие модели OpenRouter бесплатны в тестовом режиме, но логируют промпты

## Решение

### Назначение моделей по ролям и задачам

**Агентские задачи (лаборанты-агенты):**
- `openrouter/owl-alpha` — основная модель для агентов. Бесплатная, 1.05M контекст, нативная поддержка инструментов, сильная в коде. Подходит для Ворона, Совы, Кота.
- `openrouter/aurora-alpha` — для быстрых агентских задач в ботах (Муравей). 128K контекст, оптимизирована для скорости. Medium reasoning по умолчанию, high — для агентского кода.

**Длинный контекст:**
- `openrouter/sonoma-dusk-alpha` — 2M токенов, бесплатная, поддерживает изображения и параллельные вызовы инструментов. Для анализа больших кодовых баз и длинных документов.

**Сложное рассуждение и планирование:**
- `openrouter/hunter-alpha` — 1T параметров, 1.05M контекст. Для долгосрочного планирования, архитектурных решений, сложного рассуждения. Штрейкбрехер — как основная.

**Мультимодальные задачи:**
- `openrouter/healer-alpha` — зрение + слух + рассуждение + действия, 262K контекст. Для задач со скриншотами, аудио, визуальным анализом.

**Автоматическая маршрутизация:**
- `openrouter/auto` — мета-модель, выбирает оптимальную из 38 кандидатов. Для задач, где не важна конкретная модель, а важно качество результата.

### Модели, которые НЕ используем

- `openrouter/horizon-alpha` — deprecated, заменена на Horizon Beta
- `openrouter/cinematika-7b` — 8K контекст, 2023 год, устарела
- Любые cloaked-модели для чувствительных данных (все логируют промпты)

### Правило конфиденциальности

Все модели OpenRouter с пометкой «cloaked» логируют промпты и ответы для улучшения моделей. НЕ отправлять в них:
- API-ключи, токены, пароли
- Персональные данные клиентов
- Внутреннюю переписку лаборатории
- Данные VPN-клиентов

## Альтернативы

**Использовать только одну модель (Owl Alpha) для всего:**
- Плюс: простота, предсказуемость, бесплатно
- Минус: не оптимально для задач с длинным контекстом (Sonoma лучше), мультимодальности (Healer лучше), скорости (Aurora лучше)

**Переключиться на платные модели (GPT-5.1, Claude 4):**
- Плюс: проверенное качество, нет логирования промптов
- Минус: стоимость, нет преимущества перед бесплатными OR-моделями для наших задач

**Использовать Auto Router для всего:**
- Плюс: автоматическая оптимизация
- Минус: непредсказуемость, невозможно контролировать стоимость, нельзя использовать для чувствительных данных

## Последствия

**Плюсы:**
- Каждая задача получает оптимальную модель
- Большинство моделей бесплатны
- Документированный стандарт = меньше хаоса
- Мониторинг Вороном = всегда актуальная информация

**Минусы:**
- Нужно следить за изменениями в линейке OpenRouter (модели могут стать платными или исчезнуть)
- Cloaked-модели логируют данные — риск утечки при неосторожном использовании
- Больше моделей = больше сложности в настройках

**Риски:**
- OpenRouter может изменить ценообразование → митигация: Ворон мониторит, мы держим fallback-варианты
- Модель может быть deprecated → митигация: в ADR указаны альтернативы
- Логирование промптов → митигация: правило конфиденциальности выше
