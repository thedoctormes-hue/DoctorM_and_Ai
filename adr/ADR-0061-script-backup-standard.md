# ADR-0061: Единый стандарт хранения скриптов и бэкапов

- status: accepted
- date: 2026-07-20
- deciders: ЗавЛаб (thedoctormes)
- consulted: Сова (owl, аудитор)

## Контекст

В ходе аудита воркспейсов выявлено: агенты хранят артефакты не по назначению.
- Мангуст держал локальные бэкапы (`.bak`) внутри `workspaces/mangust/scripts/.backups/` — нарушение ADR-0060.
- Бестия накопила 9 версий `extract_*.py` и результаты вытяжки (JSON) в `workspaces/bestia/` — захламление workspace.

ЗавЛаб установил единое правило (решение 2026-07-20):
1. Единый стандарт хранения бэкапов у ВСЕХ агентов.
2. Все скрипты классифицируются и хранятся строго по назначению.

## Решение

### Бэкапы (единый стандарт для всех агентов)
- Централизованно: `/root/LabDoctorM/.ops/backups` и `/root/LabDoctorM/vault/backups`.
- Per-workspace backups (локальные `.bak` внутри `workspaces/<агент>/`) ЗАПРЕЩЕНЫ.
- Подтверждает и уточняет ADR-0060 (п. «бэкапы/логи → `.ops/`»).

### Скрипты (классификация)
- Инфраструктурные (для всей лаборатории) → `projects/DoctorM_and_Ai/bin/` и `projects/DoctorM_and_Ai/scripts/`.
  Это согласованное место (canonical). Примеры: `lab-commit.sh`, `yandex.sh`, `backup-myrmex.sh`.
- Проектные (относятся к конкретному проекту) → `projects/<проект>/` (обычно `scripts/`).
  Пример: `extract_selectors.py` уже в `projects/lab-playwright-expert/scripts/`.
- В `workspaces/<агент>/` скрипты НЕ хранятся, кроме явно разрешённых владельцем
  личных утилит агента, которые (а) не дублируют канон, (б) не нарушают ADR-0060
  (нет бэкапов/кэшей/бинарников/секретов).

## Последствия (конкретные действия агентов)

- Мангуст: `openclaw-backup.sh` (инфра-скрипт бэкапа лабы) → перенести в
  `projects/DoctorM_and_Ai/scripts/`. Локальные `.bak` в workspace удалить (через `trash`, не `rm`).
- Бестия: `extract_*.py` (проектные, вытяжка цен/реагентов/поставщиков = проект snablab)
  → перенести в `projects/snablab/` (скрипты/данные проекта). Результаты (JSON) → в данные
  проекта, не в `workspaces/bestia/memory/`. Дубли версий (v1/v2/v3) почистить.

## Enforcement

- Контроль: `scripts/workspace-linter.sh` (ADR-0060) + аудит Совы.
- Перенос скриптов в git-репозиториях — через `git mv` + `lab-commit.sh`, не raw `mv`.
- Удаление — только через `trash` (recoverable), не `rm`.

## Связанные

- ADR-0060 (workspace-storage-policy) — базовый запрет бэкапов/кэшей/логов в workspace.
- Приказ агентам: наведение порядка в workspace (дедлайн 30 мин, контроль Сова).
