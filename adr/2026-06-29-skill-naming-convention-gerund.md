# ADR: Стандарт именования скилов — gerund form

- **Дата:** 2026-06-29
- **Статус:** accepted
- **Автор:** Сова
- **Проект:** DoctorM_and_Ai (skills)

## Контекст

Скилы имели имена в imperative form: `accept`, `finish`, `incident`, `sstart`. Это:
- не соответствовало конвенции naming (глагол vs существительное/процесс)
- создавало путаницу с командами (accept — команда или скил?)
- не отражало суть скила как процесса

## Решение

Переименование в gerund form (отглагольное существительное):
- `accept` → `accepting-work`
- `finish` → `finishing-session`
- `incident` → `registering-incident`
- `sstart` → `starting-session`

Дополнительно:
- единый frontmatter для всех скилов (version, author, last_reviewed, status)
- разделение: SKILL.md (кратко, ~80 строк) + REFERENCE.md (детали)
- чёткое разделение: sstart = только чтение, finish = только запись
- 115 тестов валидации (frontmatter, ссылки, конфиг, размеры)

## Последствия

- имена самодокументируются (accepting-work = процесс принятия работы)
- требуется обновление всех перекрёстных ссылок
- обратная совместимость через конфиг OpenClaw (skills.entries)
