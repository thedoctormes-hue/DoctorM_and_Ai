# ADR-001: Context API — HTTP-сервис загрузки контекста

## Контекст

Лаборанты (raven, owl, bestia, ant, kot, streik) при каждом старте сессии загружают свой контекст из файлов на диске. Ранее это делалось через `agent_context.sh` — отдельный bash-скрипт, который:
- Увеличивал количество перемычек в системе
- Дублировал логику маппинг алиасов → директории
- Был дополнительной точкой отказа

## Решение

Создать единый HTTP-сервис (Context API), который:
1. Отдаёт контекст лаборантов по HTTP
2. Кеширует файлы в памяти (TTL 120 сек)
3. Имеет rate limiting (120 req/min на клиент)
4. Предоставляет метрики использования

`session_startup.sh` v7.0 встроил логику загрузки контекста напрямую, убрав зависимость от `agent_context.sh`.

## Альтернативы


- ****Context API (выбрано)**:** Кеширование, метрики, единая точка входа, Дополнительный сервис
- **Только локальные файлы:** Простота, Нет кеширования, дублирование логики
- **agent_context.sh:** Простота, Дополнительная перемычка, нет кеширования
- **Встроить в Myrmex:** Единый стек, Привязка к Node.js, сложность


## Последствия

**Положительные:**
- Единая точка загрузки контекста для всех агентов
- Кеширование снижает нагрузку на диск
- Метрики позволяют отслеживать использование
- Убран `agent_context.sh` — на одну перемычку меньше

**Отрицательные:**
- Дополнительный сервис нужно мониторить
- При падении API лаборанты теряют кеширование (но fallback работает)
