# ADR-033: Два файла myrmex.json — корневой и серверный

## Контект
В лаборатории существуют **два файла myrmex.json**:
1. **Корневой** (`/root/LabDoctorM/projects/myrmex-control/myrmex.json`) — источник правды, редактируется через Myrmex Control (порт 3000)
2. **Серверный** (`/root/LabDoctorM/myrmex.json`) — копия, используемая скриптами и агентами

## Решение
- Корневой myrmex.json — **единственный источник правды**
- Серверный myrmex.json — **автоматическая копия**, синхронизируется через post-commit hook
- Редактирование только через Myrmex Control API или через утверждённые скрипты (`lab-commit.sh`)
- Прямое редактирование серверного файла запрещено (перезапишется при следующей синхронизации)

## Следствия
- Исключаются гонки при одновременном редактировании
- История изменений хранится в changelog корневого файла
- Серверный файл можно смело читать из скриптов — он консистентен
