# INC-009: Хаос в Git — нарушение всех правил ветвления

## Описание
Лаборанты работают напрямую в main (не в worktrees), ветки агентов отстают от main, незакоммиченные изменения хаотично разбросаны по всей рабочей директории. Git Guardian (ADR-012) и стандарты (QUALITY_STANDARDS §12) нарушены.

## Severity
🟠 HIGH — система работает, но история засорена, конфликты неизбежны, откат отдельных изменений невозможен.

## Обнаружено
- **Дата:** 2026-06-11T07:15:00+00:00
- **Источник:** КотОлизатОр, аудит по запросу ЗавЛаба
- **Инструмент:** git status, git diff, git branch, git worktree list

## Что произошло

### 1. Лаборанты работают в main напрямую
Согласно ADR-012 и QUALITY_STANDARDS §12.1, каждый лаборант обязан работать в своём worktree. Фактически:
- `worktrees/` — пуст (только main)
- Бестия, Сова, Штрейкбрехер вносили изменения напрямую в рабочую директорию main
- Незакоммиченные изменения от разных агентов перемешаны в одном working tree

### 2. Ветки агентов отстают от main
- `ant/myrmex-control-artifacts` — отстаёт на 4 коммита, откатывает artifact-pulse, ADR, patterns
- `owl/lab-vault-project-scoping` — отстаёт на 4 коммита, откатывает artifact-pulse
- `streikbrecher/lab-vault-treatment` — отстаёт на 4 коммита, откатывает artifact-pulse
- `antcat/disable-agents-page`, `bestia/snablab-bot-go`, `kotolizator/git-basics` — пустые

### 3. Незакоммиченные изменения хаотичны
22 изменённых файла + 13 untracked от разных агентов в одной рабочей директории:
- **artifact-pulse** (Бестия/Сова): artifact_health.py, artifact_constants.py, artifact_dirs.yaml, README.md
- **streikbrecher** (Штрейкбрехер): CHECKPOINT.md, SESSION_HANDOFF.md, ADR-024
- **snablab** (Бестия): tech_cards/ модуль (12 новых файлов), api/v1/__init__.py
- **ADR** (Муравей/общее): ADR-012-git-guardian.md, ADR-012-myrmex-control-dual-auth.md (удалён), ADR-023 (новый), ADR-013
- **specs** (различные): BL-001, BL-002, BL-008, BL-019, BL-026, BL-041, BL-043
- **hype-pilot** (cron): 4 поста переодобрены

### 4. 4 коммита main не запушены
Муравей сделал 4 коммита (SESSION_HANDOFF, Git Flow docs), которые не в origin/main.

## Корневые причины

1. **Worktree-ориентированный workflow не внедрён.** ADR-012 описывает систему, но лаборанты не используют worktrees. Нет автоматического создания worktree при старте сессии.
2. **Pre-commit hook не блокирует коммиты в main для лаборантов.** ЗавЛаб обходит хуки через `--no-verify`, лаборанты коммитят напрямую.
3. **Нет контроля со стороны Муравья.** Тимлид не отслеживает, кто и где коммитит.
4. **Нет автоматического ребейза веток.** Ветки не синхронизируются с main.
