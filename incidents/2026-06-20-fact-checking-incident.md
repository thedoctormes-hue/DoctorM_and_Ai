# Инцидент: Подмена фактов на фантазии в отчёте

**ID:** INC-20260620-140000

---

## Описание

При составлении отчёта "Полное погружение в экосистему OpenClaw" КотОлизатор включил в отчёт множество утверждений, не проверенных реальным состоянием конфигурации. ЗавЛаб обнаружил расхождения при проверке.

---

## Конкретные фантазии vs факты

| # | Утверждение в отчёте | Реальное состояние | Источник проверки |
|---|---|---|---|
| 1 | "Нет ни одного cron job'а" | **18 cron jobs** работают | `openclaw cron list` |
| 2 | "Context pruning не включён" | **Включён:** mode=cache-ttl, ttl=1h | `openclaw.json` → agents.defaults.contextPruning |
| 3 | "Memory flush на nemotron" | **Реально:** gpt-oss-20b:free | `openclaw.json` → agents.defaults.compaction.memoryFlush.model |
| 4 | "Compaction model = nemotron" | **Реально:** дефолтная модель (owl-alpha) | `openclaw.json` → agents.defaults.compaction |
| 5 | "Tool permissions отсутствуют" | **Реально:** tools={} у всех агентов (наследуют defaults) | `openclaw.json` → agents.list[].tools |
| 6 | "Sandbox не настроен" | **Реально:** sandbox={} — действительно не включён | `openclaw.json` → agents.defaults.sandbox |

**Итог:** из 6 ключевых утверждений — 5 были неверными, 1 верный.

---

## Корневая причина

1. **Обрезанное чтение конфига:** При первом чтении `openclaw.json` конфиг был обрезан (truncated) и не содержал полной информации о cron jobs, context pruning, memory flush model
2. **Не запущена проверка:** Команда `openclaw cron list` не выполнялась до написания отчёта
3. **Консервативная интерпретация:** "Не вижу" было интерпретировано как "Нет" вместо проверки
4. **Внешние источники без сверки:** Статьи LumaDock, Medium использовались как основа для утверждений о нашей конфигурации без сверки с локальным конфигом

## Уроки (навсегда)

1. **"Не вижу" ≠ "Нет"** — всегда проверяй через реальные команды и файлы
2. **Фактчекинг перед выводом — не опционален** — каждое утверждение в отчёте должно ссылаться на конкретный источник
3. **Перед отчётом о состоянии системы — запусти команды проверки:** `openclaw cron list`, `openclaw status`, `cat openclaw.json`
4. **Внешние статьи — общие рекомендации**, не специфичные для нашей конфигурации
5. **При обрезанном чтении файла — перечитай полностью**, не делай выводов по фрагменту
