# ClawHub-скилл notebooklm-skill в зоне Мангуста (ClawHavoc)

## Описание
В рабочей зоне Мангуста (Analyst) обнаружен скилл `notebooklm-skill` с
происхождением из ClawHub: `.clawhub/origin.json` содержит
`registry: https://clawhub.ai`, `slug: notebooklm-skill`, основан на
GitHub-репозитории PleasePrompto/notebooklm-skill. ClawHub-скиллы запрещены
политикой лабы (кампания ClawHavoc, см. навык `research`). Скилл заточен под
Claude Code (browser-automation к NotebookLM), НЕ интегрирован в OpenClaw
(отсутствует в openclaw.json allowBundled / agents.defaults.skills).

## Обнаружение
В ходе `/research` по связке Gemini Pro + NotebookLM — перекрёстная сверка с
семантической памятью лабы (lab_search) нашла упоминания NotebookLM-скилла;
локализован в workspaces/mangust/skills/notebooklm-skill/. ЗавЛаб подтвердил:
"Скил там не законный по факту".

## Решение
Удалён из live-зоны по явному приказу ЗавЛаба ("Удаляй", 2026-07-15 08:35):
`rm -rf workspaces/mangust/skills/notebooklm-skill`.
- В OpenClaw не был подгружен → удаление не повлияло на gateway.
- Копия сохранена в ночном снапшоте .ops/backups/full/snapshots/20260715_020359/... (recoverable).
- В openclaw.json не прописан; риск supply-chain устранён с диска.

## Уроки
- ClawHub-происхождение (.clawhub/origin.json) в скиллах агентов = автоматический
  бан по политике ClawHavoc; требует карантина/удаления.
- Нужен гард (расширить incident-linter или добавить скан workspaces/*/skills)
  на маркер .clawhub/origin.json — ловить до попадания в зону агента.
- При аудите чужих зон действовать по явному "го" ЗавЛаба (урок INC-026).
