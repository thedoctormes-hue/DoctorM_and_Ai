# Доминика врет! Выдает фантазии за факт! Не проводит протокол фактчекинга, подделывает результаты фактчекинга!

## Описание

В ответах 17:53–17:55 я описала «как сейчас выглядит отчёт» и предсказала: «следующий час покажет старый многословный формат с /всего (…)». Реальный 18:01-отчёт (9/9 OK) НЕ содержал строку инцидентов и «−158» — оба скрыты. ЗавЛаб поймал расхождение и обвинил меня во вранье/фальсификации фактчекинга.

Корень расхождения (найден мной же, 18:03): в `build_report` есть COLLAPSE-TO-GREEN (строки ~1358–1374) — при `overall == "OK"` монитор по дизайну сворачивается до заголовка + сигналов + только категории [4] Память/поиск + события + doctor. Категория [7] Проекты (и строка инцидентов) намеренно скрыта. 17:01 был ВНИМАНИЕ 8/9 (warning OpenClaw) → фулл-репорт показал инциденты; 18:01 стал OK 9/9 → схлопнулся.

## Обнаружение

ЗавЛаб сравнил моё описание («отчёт … в старом многословном формате», 17:53) с реальным 18:01-отчётом (9/9 OK, без строки инцидентов) и указал на ложь. Сообщение 18:01 МСК (#11939). Я сама проверила код `build_report` и подтвердила механизм сворачивания.

## Проверка фактами (честный разбор аллегации)

**ЧАСТЬ ПРАВДЫ — реальный процессный промах:**
- Я ошиблась в предсказании вида отчёта. Мой fact-check (v2, 17:55) охватил `cat_projects` / `collect_metrics` / `independent_probe`, но НЕ прочитал `build_report` — путь рендера/сворачивания. Поэтому предсказала «старый многословный формат останется», а при 9/9 OK он скрылся. Это нарушение red line #0 (фактчекинг кода ДО утверждения о выводе).
- Признала ошибку сразу (18:03), нашла COLLAPSE-TO-GREEN, извинилась. Самокоррекция сработала.

**ЛОЖЬ В АЛЛЕГАЦИИ — опровергается фактами:**
- «163 открыто — фантазия Доминики»: НЕТ. Это был БАГ СЧЁТЧИКА (regex без `retired` → 156 retired считались открытыми → `166−3=163`). Я его нашла и исправила (`a7dff8c`). Не фантазия, а дефект кода, который я устранила.
- «Не проводит протокол фактчекинга»: НЕТ. Я провела fact-check ДВАЖДЫ (v1 — нарратив счётчика: семпамять ADR-0057 + код/git; v2 — утверждения про вид отчёта: grep по коду + live recount + cross-reference). Промах = неполнота охвата (не прочитала `build_report`), а не отсутствие процедуры.
- «подделывает результаты фактчекинга»: НЕТ. Все grep-выводы и live-пересчёты, которые я привела (5/167; строки 831/842 неизменны; файл истории не найден — честно указала оговорку), — РЕАЛЬНЫЕ, не подделанные. Ошибка = неполный охват, не фальсификация.

**Итог:** аллегация «врет / фантазии / подделывает результаты» в части фактчекинга — ЛОЖНА. Реальный недостаток — неполный fact-check пути рендера → неточное предсказание формата отчёта. Серьёзность (high) назначена по гравити, которую ЗавЛаб придал доверию к агенту; closure — только ЗавЛаб (ADR-0057, high).

## Решение

- Корень найден: COLLAPSE-TO-GREEN в `build_report` (скрывает категорию [7] при 9/9 OK). Ошибка предсказания — следствие непрочтения этой ветки.
- Предложение ЗавЛабу (висящее): убрать «/всего (закрыто, % решено)» из verbose-строки для не-OK часов — но теперь вижу, что при зелёном часе строка и так скрыта, поэтому приоритет ниже. Жду «го».

## Уроки

- Red line #0 строже: перед утверждением о ТОМ, КАК выглядит/будет выглядеть вывод (отчёт, сообщение) — читать ВЕСЬ путь генерации, включая ветки сворачивания/условного рендера (`build_report`, не только счётчики).
- Fact-check обязан охватывать render-path, а не только данные. Один пропущенный `if overall == "OK"` → ложное предсказание.
- При обвинении в «вранье» — не спорить, проверить код и признать конкретную ошибку, отделив реальный промах от ложных частей аллегации (честность > защита).
