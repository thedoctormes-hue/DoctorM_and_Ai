# Удаление nvidia/ сломало onnx-embedder

## Описание

При очистке мёртвого груза удалён каталог `/usr/local/lib/python3.10/dist-packages/nvidia/` (2.7 GB). Это привело к падению `onnx-embedder.service` с ошибкой:

```
ValueError: libcublasLt.so.*[0-9] not found in the system path
```

Цепочка зависимостей: `transformers` → `torch` → `nvidia CUDA libraries`. Модуль `torch` при импорте пытается загрузить CUDA shared libraries через `_load_global_deps()`. Без nvidia/ библиотек загрузка падает.

onnx-embedder генерирует эмбеддинги для семантического поиска (lab-search-proxy:8081 → onnx-embedder:8082). Его падение означало невозможность генерации новых эмбеддингов (FTS-поиск при этом продолжал работать).

## Обнаружение

- Время: 2026-06-23 ~04:21 UTC
- Метод: после очистки при запуске `/accepting-work` проверка `/health` вернула FAIL
- Подтверждение: `systemctl status onnx-embedder` показал `exit-code`, `journalctl` — `ValueError: libcublasLt.so not found`

## Решение

Замена `torch` (полная версия с CUDA, 1.2 GB) на `torch-cpu` (версия без CUDA, 192 MB):

```bash
pip3 uninstall -y torch
pip3 install torch --index-url https://download.pytorch.org/whl/cpu
```

Результат: torch 2.12.1+cpu, CUDA: False, onnx-embedder запустился успешно.

Дополнительный эффект: экономия ~1 GB (torch-cpu 192 MB vs torch+CUDA 1.2 GB).

## Уроки

1. **НЕ удалять Python-пакеты только по отсутствию прямых импортов в коде.** Проверять полную цепочку зависимостей: `pip show <pkg> --requires` → рекурсивно.
2. **CUDA-библиотеки в nvidia/ — зависимость torch**, даже если сервер не имеет GPU. torch импортирует их при инициализации.
3. **После удаления Python-пакетов — перезапускать и проверять ВСЕ зависимые сервисы**, а не только те, что явно импортируют удалённый пакет.
4. **torch-cpu — правильный выбор для CPU-only серверов.** Экономия ~1 GB, нет зависимости от CUDA.
