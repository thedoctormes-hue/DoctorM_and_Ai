# INC-20260708-065200: api-hub оркестратор поиска лежит (crash loop)

> Обнаружено в ходе плановой диагностики лаборатории (ЕБШ, тип audit) по роли Operator.

---

## Описание
Сервис `apihub.service` (API Hub — унифицированный шлюз/web-search оркестратор, основной инструмент поиска Бестии по MEMORY) находится в бесконечном цикле перезапуска (RestartSec=5, status=200/CHDIR).

Proximate cause (проверено live):
- `WorkingDirectory=/root/LabDoctorM/projects/api-hub` — **отсутствует** (CHDIR failed: No such file or directory).
- `/root/LabDoctorM/projects/api-hub` нет в filesystem (подтверждено `ls projects/` и `find`).
- `python3` на месте (`/usr/bin/python3 -> python3.10`), не причина.
- Docker-контейнер `api-hub-db-1` (postgres) при этом **Up 9 days** — БД жива, только код приложения исчез.

## RCA (уточнение 2026-07-08 06:58, ЕБШ + fact-check)
**Настоящий корень проблемы: осиротевший systemd-юнит после незавершённой миграции/удаления проекта api-hub.**

> Уточнение 2026-07-08 07:11 (ЗавЛаб): слияние api-hub + free-api-hunter — **осознанное решение**, не случайное удаление. Дефект НЕ в самом слиянии, а в **незавершённом cutover**: при консолидации не выключили/не перенастроили `apihub.service` и не обновили ссылки воркфлоу (MEMORY/HEARTBEAT), которые гонят поиск через api-hub. Механическая причина та же — orphaned unit, — но первопричина: намеренная миграция без плана переключения запущенного сервиса и зависимых процессов.

1. История: api-hub и free-api-hunter планировалось слить в один проект (план Штрейкбрехера `api-hub-migration-plan.md`; коммит `1fd46de` 2026-06-27 «merge free-api-hunter into api-hub»; ветка `migration/free-api-hunter` на origin).
2. Локальная директория `/root/LabDoctorM/projects/api-hub` была удалена (crash loop стартует 2026-07-08 01:45 UTC — значит удаление ~в это время), НО systemd-юнит `apihub.service` не был выключен командой `systemctl disable --now` перед удалением.
3. Результат — классический «сиротский unit»: юнит ссылается на несуществующий WorkingDirectory → Restart=always гонит бесконечный CHDIR-loop (restart counter 118579+).
4. Это задокументированный анти-паттерн (см. MEMORY: «Удаление systemd unit — ВСЕГДА disable --now перед rm. иначе WANT-symlink остаётся → restart storm»). Консолидация в free-api-hunter НЕ завершена: free-api-hunter — отдельный Go-сервис, api-hub-код в него не слит (grep по projects/free-api-hunter пуст).

**Важно (fact-check самокоррекция):** заявление из первичной диагностики «/root/LabDoctorM/backups/ отсутствует → gap бэкапов» — ЛОЖНОЕ. Реальные бэкапы живут в `/root/LabDoctorM/.ops/backups/` (incremental + full snapshots). Бэкап api-hub от 2026-06-28 присутствует и содержит валидный `.git` с remote.

**Recovery path существует (код НЕ потерян):**
- GitHub: `https://github.com/thedoctormes-hue/api-hub.git` (ветка `migration/free-api-hunter`)
- Локальный бэкап: `/root/LabDoctorM/.ops/backups/incremental/project-backups/20260628_020257/api-hub` (полный git-репозиторий)
