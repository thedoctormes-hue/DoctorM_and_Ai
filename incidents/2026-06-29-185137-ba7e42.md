# Таблицы в Telegram — нарушение PAT-006

## Описание

Subagent `memory-spike` при завершении работы сгенерировал markdown-таблицу с результатами проверок (столбцы: #, Проверка, Статус). Telegram Bot API не поддерживает markdown-таблицы — они ломаются и пользователь видит нечитаемый мусор.

Корень: нарушение правила PAT-006 (без таблиц в Telegram) на уровне subagent. Я, как оркестратор, не проверил формат вывода перед передачей пользователю.

## Обнаружение

Зав_LAB в 15:48: «Я просил объяснить понятно, ты объяснил очень непонятно!»

Проверка: от subagent пришёл текст с таблицей:
```
| # | Проверка | Статус |
| --- | ... | ... |
```
Telegram не отрендерил — пользователь увидел сырой markdown.

## Решение

1. Пересуммаризировать отчёт текстом (выполнено в 15:51)
2. Добавить в промпт subagent-ов явное правило: «НЕ использовать таблицы в ответах — только списки и форматирование»
3. Проверить, не было ли аналогичных нарушений от других subagent-ов в этой сессии (не обнаружено)

## Уроки

1. Subagent-ы нужно инструктировать про формат ответа, если результат пойдёт напрямую пользователю
2. PAT-006 применяется ко всем агентам и субагентам без исключений
3. После спавна subagent — проверять формат вывода перед передачей дальше
