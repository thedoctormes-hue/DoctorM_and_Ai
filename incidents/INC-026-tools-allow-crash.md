# INC-026: Поломка gateway через tools.allow в agents.list

## Описание

Агент Муравей (или другой агент с доступом к конфигу) добавил секцию `tools.allow: ["message"]` в каждого агента в `agents.list` файла `~/.openclaw/openclaw.json`. Это было сделано для исправления предупреждения доктора об отсутствии инструмента `message`.

## Severity

� CRITICAL — gateway упал при старте, все 8 агентов недоступны

## Причина

В версии OpenClaw 2026.6.9 структура `tools.allow` не поддерживается внутри каждого агента в `agents.list`. Это либо глобальный параметр, либо делается через `tools.profile`. Когда gateway пытается загрузить конфиг с такой структурой — падает при старте.

## Почему doctor не помог

1. Doctor проверяет синтаксис и логическую валидность конфига (наличие обязательных полей, правильные типы)
2. Doctor **не проверяет**, поддерживает ли конкретная версия OpenClaw ту или иную структуру
3. Валидация прошла (Config valid), doctor сказал «ошибок нет» — но gateway всё равно упал

## Уроки (для всех агентов)

1. **НЕ** добавлять `"tools": { "allow": [...] }` в конфиг агентов
2. Инструмент `message` встроен системой автоматически — не требует явного добавления
3. Doctor проверяет синтаксис, но не проверяет совместимость с версией
4. **Перед изменением конфига:** бэкап → валидация → doctor → мониторинг 24ч
5. Если doctor говорит «OK» — это не значит что запустится
