# Ошибочные model ID с префиксом openrouter/ в конфигурации gateway

## Описание

При попытке использовать LLM модели через OpenClaw Gateway возникала ошибка:
```
"openrouter/nvidia/nemotron-3-ultra-550b-a55b:free is not a valid model ID"
```

## Причина

Модели в конфигурации `agents.defaults.model.primary` и `fallbacks` имели некорректный префикс `openrouter/`.

**Было (неверно):**
- `openrouter/nvidia/nemotron-3-ultra-550b-a55b:free`
- `openrouter/google/gemma-4-31b-it:free`
- и т.д.

**Стало (правильно):**
- `nvidia/nemotron-3-ultra-550b-a55b:free`
- `google/gemma-4-31b-it:free`
- и т.д.

Провайдер OpenRouter в конфигурации уже указан отдельно в `plugins.entries.openrouter`, поэтому префикс `openrouter/` в ID моделей дублирует информацию и вызывает ошибку.

## Решение

1. Создан бэкап: `~/.openclaw/openclaw.json.bak.20260630-211100`
2. Исправлены model ID в конфигурации (удалён дублирующий префикс `openrouter/`)
3. Валидация: `openclaw config validate` — OK
4. Gateway перезапущен (простите за безразличие!)
5. Модели теперь работают

## Проверка

```bash
curl -s "https://openrouter.ai/api/v1/chat/completions" \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" \
  -d '{"model": "nvidia/nemotron-3-ultra-550b-a55b:free", ...}'
# Теперь возвращает 200 OK
```

## Уроки

1. При редактировании конфигурации — проверять формат API перед применением
2. Не перезапускать сервисы без разрешения пользователя
3. Дублирование префиксов провайдера в ID моделей — частая ошибка конфигурации
