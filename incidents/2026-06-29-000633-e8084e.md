# autoexpert билд падает из-за глобального npm omit=dev

## Описание

`npm install` и `npm ci` в `/root/LabDoctorM/projects/autoexpert/front_preview/app/` устанавливали только 110 из 347 пакетов. `vite`, `@vitejs/plugin-react`, `vitest` и другие devDependencies были проигнорированы. Билд падал с `UNRESOLVED_IMPORT` / `Cannot resolve entry module index.html`.

## Обнаружение

При выполнении задачи taste-skill интеграции (интеграция EmptyState в autoexpert frontend). После `npm install` директория `node_modules/vite/` была пустая, хотя npm сообщал "added 180 packages".

## Корневая причина

Глобальный npm config (`~/.npmrc`) содержал `omit=["dev"]`. Это исключало ВСЕ devDependencies из установки — не только в autoexpert, но и в других проектах лаборатории.

## Решение

Муравей обнаружил проблему, создал `.npmrc` в директории проекта:
```
include=dev
```
Это переопределяет глобальный `omit=dev` для конкретного проекта.

После `npm install` установилось 347 пакетов, билд прошёл успешно.

Коммиты:
- `e4945f5` — fix: add .npmrc to override global omit=dev
- `46e2a14` — fix: regenerate package-lock.json with devDependencies included

## Уроки

1. **Проверять npm config при проблемах с установкой** — `npm config list` покажет omit/include настройки
2. **`.npmrc` в проекте** — если глобальный конфиг мешает, локальный `.npmrc` с `include=dev` решает проблему
3. **Не верить "up to date"** — npm может говорить что всё ок, но физически не ставить пакеты
4. **Красная линия:** при проблемах с npm install — сначала `npm config list`, потом локальный `.npmrc`
