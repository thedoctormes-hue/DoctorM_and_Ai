INC-040: Остановить → Исправить → Перезапустить
Корень: fast-reindex шёл 80+ минут с MarkItDown (который мы решили вынести из reindex.py). Половина работы впустую.
Решение: немедленная остановка, вырезание MarkItDown, перезапуск только по MD за 10 мин.
Правило: если задача делает работу, которая будет отменена — останавливай сразу.

INC-041: Монолит — зло
Корень: reindex.py = scan + MarkItDown convert + chunk + embed + FAISS. Падение одной = падение всех.
Решение: разделение на 2 независимых процесса.
Правило: если процесс делает >2 разных вещей — это монолит, пора разделять.

INC-042: Проверять пути перезапуском long-running процесса
Корень: fast-reindex работал с путём который я не проверил, ушёл в мусорные директории.
Решение: перед запуском долгого процесса — `ls -la <path>` + проверить что путь ведёт туда, куда должен.
Правило: отчёт = верифицированный факт. Проверь путь — потом запускай.

INC-043: Два независимых процесса > один с "умным scheduler"
Корень: попытка втянуть converter.py в reindex через '.converted/' директорию — всё ещё монолит.
Решение: converter.py и reindex.py полностью независимы, общаются только через файловую систему.
Правило: два процесса = два скрипта, два systemd unit, два лога. Нет "общей оркестрации".

INC-044: Manual > Timer для непредсказуемых данных
Корень: 8 агентов создают/изменяют файлы непредсказуемо. Таймер всегда в хвосте, тратит ресурсы.
Решение: converter работает только по явному запросу с указанием папки.
Правило: для систем с непредсказуемым объёмом изменений — ручной запрос лучше периодического таймера.

INC-045: При удалении функции — grep все ресы
Корень: после вырезания MarkItDown остался `is_converted` в if-блоке, NameError при рантайме.
Решение: `grep -n "convert\|CONVERT\|MarkItDown" reindex.py` → найти и почистить все упоминания.
Правило: после удаления кода — grep по всему файлу на все связанные имена.

INC-046: Документ системных таймеров обязателен
Корень: не было единого списка всех системных таймеров. При удалении/перезапуске — непонятно что включено, что отключено.
Решение: вести документ со списком всех таймеров.
Правило: каждый systemd timer должен быть задокументирован — unit, интервал, что делает, когда последний раз менялся.
