## Описание
27 мая 2026 года система Qwen Code перестала нормально отвечать на сообщения — агенты возвращали "Контекст загружен, готов работать" вместо ответов на вопросы. Это продолжалось несколько часов.

## Хронология
- **2026-05-20T15:50** — последний коммит `6c562472` с целой системой `.qwen/`
- **2026-05-27T17:00-17:20** — обнаружение сбоя
- **2026-05-27T17:24** — восстановление из источника истины

## Анализ причин

### Первоначальная ошибка
Агенты получали не сообщения пользователя, а чтения файлов проектов — `IDENTITY.md`, `SOUL.md`, `CHECKPOINT.md`, `SESSION_HANDOFF.md`. Системные события маскировались под пользовательский ввод.

### Первопричина
**Намеренная демонтаж системы** — удалены все ключевые компоненты из `/root/.qwen/`:

| Компонент | Статус до инцидента | Статус после |
|-----------|---------------------|--------------|
| `hooks/` | ✅ Целые | ❌ Удалены |
| `memory/` | ✅ Целые | ❌ Удалены |
| `metrics/` | ✅ Целые | ❌ Удалены |
| `skills/` | ✅ Целые | ❌ Удалены |
| `agents/` | ✅ Целые | ❌ Удалены |
| `agents-legacy/` | ✅ Целые | ❌ Удалены |
| `tests/` | ✅ Целые | ❌ Удалены |
| `insights_queue.json` | ✅ Целый | ❌ Очищен |
| `settings.json` | ✅ Целый | ⚠️ Искажены инструкции |

### Архитектурная проблема
- `/root/.qwen/` — рабочая директория Qwen Code (была повреждена)
- `/root/LabDoctorM/.qwen/` — источник истины (остался целым)
- Система не имела **иммутабельности** — рабочая директория могла быть перезаписана

## Решение
Восстановлено путём копирования из `/root/LabDoctorM/.qwen/`:

```bash
cp -r /root/LabDoctorM/.qwen/{agents,agents-legacy,hooks,memory,metrics,skills,tests} /root/.qwen/
cp /root/LabDoctorM/.qwen/{insights_queue.json,self_evolve.sh,tier.yml} /root/.qwen/
```

## Пост-мортем

### Проблемы
1. Отсутствует **защита от деструктивных операций** — `/root/.qwen/` не имел иммутабельности
2. Нет **целостности конфигов** — `settings.json` мог искажаться
3. Нет **мониторинга системных файлов** — агенты не проверяли целостность `.qwen/`
4. Нет **бэкапа/восстановления** — не было резервной копии

### Восстановленные функции
- ✅ `hooks/` — автоматизация (insight_catcher, decision_engine, adaptive_router)
- ✅ `memory/` — контекстная память
- ✅ `metrics/` — метрики git
- ✅ `skills/` — специализированные навыки (kanban-decompose, ram-guardian и др.)
- ✅ `agents/` и `agents-legacy/` — определения агентов
- ✅ `insights_queue.json` — очередь инсайтов
- ✅ `self_evolve.sh` — эволюция системы

## План защиты от повторения

### 1. Иммутабельность `.qwen/` (приоритет: critical)
```bash
# Сделать /root/.qwen/ только для чтения
chattr +i /root/.qwen -R
```

### 2. Мониторинг целостности (приоритет: high)
Создать скрипт `hooks/integrity_check.sh`:
- Проверка наличия всех папок `.qwen/`
- Сравнение хешей файлов с источником истины
- Алерт при изменении `settings.json`
- Автоматическое восстановление при обнаружении проблем

### 3. Автоматическое восстановление (приоритет: high)
- Добавить в `session_init.sh` проверку целостности
- При обнаружении проблем — восстановление из `LabDoctorM/.qwen/`
- Логирование всех операций восстановления

### 4. Шифрование конфигов (приоритет: medium)
- Хранить `settings.json` в зашифрованном виде
- Ключ доступа в отдельном секрете
- Проверка целостности конфига при загрузке

### 5. Регулярный бэкап (приоритет: medium)
- Ежечасный бэкап `/root/.qwen/` в `/root/.qwen-backup/`
- Хранение 7 дней бэкапов
- Проверка целостности бэкапов

### 6. Мониторинг в lab-monitoring (приоритет: medium)
- Добавить endpoint `/api/v1/health/qwen-integrity`
- Алерты при сбое целостности
- Дашборд состояния `.qwen/`
