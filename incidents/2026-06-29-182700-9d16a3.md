# Старт сессии не по протоколу starting-session

## Описание

Два раза подряд — в сессии #1 (15:24) и в текущей сессии (18:22) — стартаа сессия не по протоколу «starting-session». ЗавЛаб явно просил «по протоколу», но скилл не был применён. Результат: частичная загрузка контекста, отсутствие сводки состояния, некорректный запрос задачи.

## Обнаружение

ЗавЛаб дважды поймал:
1. **15:27 UTC** — «Старт сессии не по протоколу! Оформляй!»
2. **18:24 MSK** — аналогичная ситуация в новой сессии

Скилл starting-session читает SKILL.md, выполняет параллельные проверки (инциденты, диск, сервисы, Docker, ONNX, anti-loop), выводит форматированный статус. Без него — пустой «привет» без данных.

## Корневая причина

- Не проверяю триггеры скиллов при старте сессии
- «Привет» вместо протокола — это игнорирование явного запроса пользователя

## Решение

- Зарегистрирован инцидент (данный файл)
- В MEMORY.md: обновить красную линию — **при старте сессии ВСЕГДА применять starting-session, если ЗавЛаб просит «по протоколу»**

## Уроки

- Явное требование «по протоколу» — это не рекомендация, это директива.
- Два провала подряд по одному месту = systemic issue, не случайность.
- Красная линия: **никогда не стартовать сессию без starting-session, если есть намёк на протокол.**
