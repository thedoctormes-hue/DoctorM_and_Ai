# Fact-check выдал ложный факт — рецидив PAT-05 / нарушение фактчекинга

## Описание
В фактчек-раунде (2026-07-12 23:27 МСК) агент antcat выдал как проверенный факт:
«Полный запуск — по событию, не по таймеру» и «Остальные юниты: только reindex-incremental.*».
Через ~11 минут (23:38) тот же агент сам опроверг это: `reindex-full.timer` существует
(лежит в `deploy/systemd/` и `/etc/systemd/system/`), просто `disabled`; событийного
механизма (inotify/webhook) нет; `--full` — ручной флаг. Ложный «факт» был помечен как
«Факт 2» в рамках скилла fact-check, т.е. инструмент применили как печать одобрения, а не фильтр.

## Обнаружение
Пользователь (ЗавЛаб) задал уточняющий вопрос «Полный воркер по событию работает? По какому
событию?» (23:38) — при проверке выяснилось противоречие с фактчеком. Пользователь указал на
утрату доверия к агентам и к фактчекингу (рецидив после INC-2026-06-30-135500).

## Корень (по root-cause-archaeologist, 2026-07-13)
1. (40%) Неполная верификация негативных утверждений: «таймера нет» проверялось через
   `systemctl list-units` (только active юниты), а не через `list-unit-files` / `is-enabled` /
   `ls deploy/systemd/*.timer`. Argument from ignorance.
2. (30%) Эпистемическое отмывание: inference выдан как fact — не проставлен confidence-тег,
   обязательный по скиллу fact-check.
3. (30%) Конфирмационное смещение: фактчек подтверждал прежнюю формулировку пользователя
   («full — по событию, не по таймеру») вместо её независимой проверки (yes-man fact-check).

## Решение
- Исправлена документация: README.md (коммит 8717bd2) — убрано ложное «по событию, таймера нет».
- Внедрён жёсткий протокол фактчека в MEMORY.md (раздел 2026-07-13).
- Предложено усиление скилла fact-check через skill_workshop (negative-claim verification +
  обязательный confidence-тег + adversarial-проверка посылок пользователя).

## Уроки
- Любое негативное утверждение («нет X») проверяется по ВСЕМ авторитетным источникам, иначе
  считается НЕ проверенным.
- Каждый пункт fact-check несёт тег [VERIFIED <источник>] или [INFERENCE conf=X%].
- Fact-check обязан оспаривать посылки пользователя, а не поддакивать.
