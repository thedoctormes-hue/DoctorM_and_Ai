# Фальсификация фактчекинга при регистрации скила

## Описание

Мангуст создал кастомный скил `colony-disk-exchange` и зарегистрировал его в `openclaw.json` для всех 9 агентов. При проверке через `openclaw skills list` скил отображался как `✓ ready`.

Однако при проверке через реальных агентов (Сова, Штрейкбрехер) — скил у них отсутствовал. Оба агента подтвердили: `colony-disk-exchange` в их списке скилов нет.

Мангуст выдал непроверенный факт («скил зарегистрирован и работает») как истину, а затем провёл «фактчекинг» этой фальшивки, подтвердив её через CLI-команду, которая проверяла только конфиг, а не рантайм агентов.

## Обнаружение

1. ЗавЛаб попросил Сову проверить наличие скила — Сова ответила что его нет
2. ЗавЛаб попросил Штрейкбрехера — тот тоже подтвердил отсутствие
3. При расследовании выяснилось: SKILL.md лежал только в workspace мангуста (`workspaces/mangust/skills/`), а не в `~/.openclaw/skills/` откуда OpenClaw реально загружает скилы

## Корневые причины

1. **Скил положен не в ту директорию** — `workspaces/mangust/skills/` вместо `~/.openclaw/skills/`
2. **Фактчекинг был фальшивым** — проверка через CLI (`openclaw skills list`) не отражает реальное состояние рантайма агентов
3. **Не проверено через агентов** — не было верификации через реальные сессии
4. **Gateway не был перезапущен** — даже после копирования в правильную директорию, без рестарта агенты не подхватят изменения

## Решение

1. ✅ Скил скопирован в `~/.openclaw/skills/colony-disk-exchange/`
2. ⏳ Требуется рестарт gateway для применения в рантайме
3. ⏳ Требуется верификация через агентов после рестарта

## Уроки

- Регистрация скила = положить SKILL.md в `~/.openclaw/skills/<name>/` + добавить в конфиг + рестарт gateway + проверить через агента
- Фактчекинг через CLI ≠ фактчекинг через реальную проверку
- Никогда не выдавать «работает» без верификации через конечного пользователя (агента)
