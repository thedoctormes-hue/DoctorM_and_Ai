# Сова (аудитор) оказалась на чужой ветке Кота без команды

## Описание
В ходе починки сканера tails-watch.sh выяснилось, что рабочая директория проекта DoctorM_and_Ai неоднократно физически находилась на ветке `kotolizator/skill-link-lint-skip-codeblocks` (ветка Кота). Переключение выполняли субагенты Совы при выполнении задач по очистке/миграции. Фактических изменений в ветку Кота Сова не вносила — коммиты туда делал сам КотОлизатор (его ветка, легитимно; author=КотОлизатор). Но сам факт нахождения HEAD на чужой ветке = zone-trespass (риск случайного коммита в чужое), особенно недопустимый для контролёра порядка.

## Обнаружение
ЗавЛаб увидел в логах сессии упоминание «я сейчас физически нахожусь на ветке Кота» и потребовал зарегистрировать инцидент. Проверка reflog подтвердила переключения owl/incident-canon-migrate → kotolizator/skill-link-lint-skip-codeblocks (минимум дважды) и обратно. Коммит f614b13 в той ветке сделал КотОлизатор, не Сова.

## Паттерн (инстинкт)
Рецидив PAT-019 I-04 (Zone-trespass в семпамяти / несанкц. мутация общего ресурса). ID дописан в PAT-019 I-04.
- **Root:** субагент Совы выполнил git checkout/switch на чужую ветку и не вернул HEAD в исходное состояние.
- **Signal:** текущая ветка проекта != ожидаемая рабочая ветка агента; упоминание «нахожусь на ветке X» где X — чужой префикс.
- **Превенция:** субагентам запретить менять HEAD без явной нужды + обязанность вернуться; либо работать через `git push origin <local>:<remote>` без переключения HEAD; старт/финал сессии валидирует текущую ветку.
