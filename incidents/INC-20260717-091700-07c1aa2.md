# INC-20260717-091700: Удаление myrmex.json через git rm сломало backup-myrmex и вызвало тихую деградацию myrmex-control

- **Дата:** 2026-07-17 09:17 МСК
- **Инициатор:** Муравей (commit 00d1eb7), go ЗавЛаба (merge 7c38f2f «go ЗавЛаба 2026-07-16»)
- **Действие:** `git rm myrmex.json` (18 696 deletions) в коммите 00d1eb7 «chore: ignore auto-generated myrmex.json» (23:14 МСК 16 Jul)
- **Причина:** auto-generated файл помечен ignore при git-гигиене, но `git rm` удалил его с диска (а не `--cached`), хотя `.env` (MYRMEX_FILE) и сервис/бэкап от него зависят
- **Симптомы:**
  - `backup-myrmex.service` — FAILED каждый час (не находит файл)
  - `myrmex-control.service` — active, но `myrmex.ts:71` fallback `createDefaultState()` → тихо отдаёт пустое состояние с 23:14 16 Jul (реальные projects/agents/tasks не загружаются)
- **Root cause (верифицировано Вороном, live probes):** не миграция на БД, а `git rm` при git-гигиене
- **Lessons:** `git rm` vs `git rm --cached` при untrack auto-generated файлов; алерт на FAILED systemd-сервисы (тихая деградация недопустима)

## Решение
- **myrmex.json восстановлен** из `/var/backups/myrmex/auto-2026-07-16T20-00-00.json` (531 380 B) → файл на диске (09:35 МСК 17 Jul, 531380 B). `backup-myrmex.service` работает: status=0/SUCCESS, бэкап создан 10:00 (`/var/backups/myrmex/auto-2026-07-17T07-00-00.json`).
- **myrmex-control.service собран и перезапущен:** отсутствовал `dist/server/index.js` (сервис падал в crash-loop, NRestarts=516). Выполнен `npm run build:server` + `npm run build:client` (10:46 МСК), `systemctl restart myrmex-control.service` → `active (running)`, `NRestarts=0`. Тихая деградация (пустой fallback) устранена.
- **Root cause подтверждён:** коммит 00d1eb7 (`git rm myrmex.json`) удалил файл с диска, сломав зависимые сервисы. Зафиксировано как паттерн I-15 (git rm auto-gen ломает сервисы) и I-16 (запуск без build → crash-loop) в `patterns/PAT-019-lab-instincts-library.md` (commit `aba58d5`).
- **verified_by:** thedoctormes (ЗавЛаб), не author (antcat) → closure integrity по ADR-0057 соблюдена.
