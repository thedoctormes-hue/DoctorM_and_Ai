# Incident: OOM / Swap Thrashing — ONNX Embedder Wedged, Reindex Blocked

**Date**: 2026-07-09 09:35 UTC (12:35 MSK)
**Severity**: CRITICAL
**Reporter**: streikbrecher (heartbeat)

## Summary
Host experienced OOM at 09:02 UTC. Swap filled to 80% (3.3/4.1 GB). ONNX embedder (`onnx-embedder.service`) is perpetually swapped out and enters `D` (uninterruptible sleep) on every page fault when trying to serve requests. Reindex process (PID 316503) is blocked waiting for embeddings.

## Timeline
- **09:02 UTC**: Kernel OOM ("Out of memory and no killable processes..."). systemd-journald SIGTERM'd.

- **09:22 UTC**: `reindex-incremental.timer` fired (last successful incremental).
- **09:29 UTC**: `reindex-full.timer` started with `Persistent=true` → catch-up run → triggered `reindex-full.service` → conflicted with incremental timer (Conflicts=) → stopped incremental timer.
- **09:30 UTC**: Heartbeat detected issues. Actions taken:
  - Killed orphan embedder (SIGKILL).
  - Re-armed `reindex-full.timer` (active, next 04:34 tomorrow).
  - Re-armed `reindex-incremental.timer` (active, next 09:52).
  - Attempted `onnx-embedder.service` restart → new instance (PID 321911) loaded model but immediately swap-thrashed → D state.
  - Sent SIGKILL to 321911 → auto-restarted → PID 323303 → again D state with 2.5 GB swapped.
- **09:37 UTC**: `reindex-incremental.service` fired but exited: "Reindex already running (PID 316503). Exiting."

## Root Cause
System-wide memory pressure (OOM at 09:02) pushed embedder and other processes to swap. Once swapped, the embedder's 2.5+ GB stays in swap until accessed. Every HTTP request triggers massive swap-in I/O (local disk) → process enters `D` state for seconds/minutes → client times out → embedder never recovers.

## Remediation Required (Host Level)
1. **Identify and stop memory hog** — top swap users:
   - PID 321911/323303 (embedder): 2.5 GB swap
   - PID 134377 (MainThread): 302 MB swap, 919 MB RSS
   - PID 314069 (python3): 7.5 MB swap, 399 MB RSS
2. **Free swap pressure** — kill non-critical swappy processes, or add swap, or reboot.
3. **Protect embedder** — `MemoryMax=700M` in unit is ignored for swap (only limits RSS). Consider `MemorySwapMax=0` or cgroup v2 memory.swap.max=0.
4. **Prevent OOM recurrence** — audit what caused 09:02 spike (likely full reindex + concurrent workloads).
