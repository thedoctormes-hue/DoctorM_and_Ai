## Оценка рисков

### Что было в публичном доступе (SnabLab)
- **Пароли `admin123`, `secret123`** — учебные, не реальные. Используются только в тестах и seed-скрипте.
- **Риск: НИЗКИЙ** — пароли не дают доступ к продакшн-системам.

### Что было в приватном репозитории (LabDoctorM)
- **6 Telegram bot tokens** — дают полный контроль над ботами
- **GitHub PAT** — даёт доступ к репозиториям организации
- **Пароли Habr/VC.ru** — дают доступ к аккаунтам на площадках для кросспостинга
- **Пароль reg.ru** — даёт доступ к управлению доменами
- **Риск: КРИТИЧЕСКИЙ** — все токены дают доступ к реальным сервисам

### Как долго данные были доступны
- **Первичная утечка:** ~1-2 часа (от push до обнаружения)
- **Вторичная утечка (пересозданный репо):** обнаружена и устранена в течение 1 часа
- **Общее время экспозиции:** ~2-3 часа

### Кто мог получить доступ
- **Приватный репозиторий LabDoctorM:** только участники организации thedoctormes-hue
- **Публичный репозиторий SnabLab:** любой пользователь интернета
- **GitHub Actions логи:** могли содержать PAT при первом push (в URL)

## Причина
1. OWL запушила весь репозиторий лаборатории без предварительной проверки на наличие секретов
2. Не был запущен secret-scan перед push
3. Не было pre-push hook для автоматической проверки
4. Секреты хранились в коде, документации и скриптах вместо .env файлов
5. При пересоздании репозитория — файлы с секретами были снова добавлены в коммит
6. GitHub PAT был захардкожен в remote URL

## Post-Mortem

### Что пошло не так
1. **Отсутствие secret-scan перед push** — базовая проверка не была выполнена
2. **Секреты в коде** — токены хранились в .py, .sh, .json, .md файлах вместо .env
3. **Нет pre-push hook** — не было автоматической защиты
4. **PAT в remote URL** — токен аутентификации захардкожен в URL репозитория
5. **Повторная утечка при пересоздании** — при «очистке» файлы с секретами были снова закоммичены
6. **25 коммитов вместо 1** — «clean history» содержала полную историю с секретами

### Что было сделано правильно
1. **Быстрое обнаружение** — ЗавЛаб обнаружил утечку в течение часа
2. **Pre-push hook создан** — автоматическая проверка перед каждым push
3. **CI/CD secret-scan добавлен** — TruffleHog + detect-secrets
4. **SSH вместо PAT** — безопасная аутентификация
5. **Полный аудит** — при повторной проверке обнаружены дополнительные утечки

### Уроки
1. **Никогда не пушить без secret-scan** — даже в приватный репозиторий
2. **Разделять публичный и приватный репо** — публичный должен содержать только фронтенд + README
3. **Все секреты только в .env** — ни в коде, ни в документации, ни в скриптах
4. **Pre-push hook обязателен** — автоматическая проверка перед каждым push
5. **Никогда не хранить токены в remote URL** — использовать SSH или credential helper
6. **Файлы .orig опасны** — бэкапы конфигов могут содержать секреты
7. **Пересоздание репо ≠ очистка** — нужно проверять КАЖДЫЙ файл перед коммитом
