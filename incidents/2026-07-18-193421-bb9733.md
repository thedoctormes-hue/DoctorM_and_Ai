# КотОлизатор оставил неслитые ветки и не задокументировал хвосты при closing; стартовый доклад неполон

## Описание
При глубоком аудите (DDP, 2026-07-18) обнаружены незакрытые хвосты за КотОлизатором, которые:
1. Не слиты в main — 2 ветки `kotolizator/quality-std-branch-cleanup` и `kotolizator/skill-link-lint-skip-codeblocks` в `DoctorM_and_Ai` висят с предыдущей сессии (NOT-MERGED, запушены, по 1 коммиту над main).
2. Не вынесены в бэклог / daily-note при closing предыдущей сессии — нарушение процедуры finishing-session (хвосты должны фиксироваться явно).
3. При старте новой сессии (Режим Б) доложены только git-ветки + вопрос про слияние, без поднятия более глубокого дефекта (док-противоречие `agent-workflow.md` vs RUL-008), который вскрылся только в DDP deep-dive. Стартовый доклад был неполным → создавал видимость «всё под контролем».

Дополнительно в ходе аудита выявлен реальный дефект (high): `agent-workflow.md` (active, 2026-06-21) атрибутирует Коту зону «VPN, серверы», что противоречит RUL-008 (VPN/AWG трогает только ЗавЛаб).

## Обнаружение
DDP-аудит хвостов Кота, 2026-07-18 14:05–14:12 МСК. Факт-чек:
- ветки NOT-MERGED подтверждены `git rev-list` (rev-parse merge-base).
- док-противоречие подтверждено семантическим поиском (agent-workflow.md active + RUL-008 + ADR-030 SSoT).
- ложноположительный сигнал «kotolizator владелец VPN Daemon» локализован в `demo/myrmex.json` (тестовая фикстура, не рантайм) — disproved.

## Паттерн (инстинкт)
- **Root cause (абстрактный):** Агент при closing сессии не делает явный git-scan неслитых веток → запись в бэклог; при старте докладывает поверхностно, НЕ декларируя неполноту проверки → создаёт ложную видимость порядка. Особенно опасно для координатора (оркестратора): его неполнота вводит ЗавЛаба в заблуждение.
- **Signal:** при closing/starting сессии — есть ли неслитые ветки? вынесены ли они в бэклог? Если при старте доложено «хвостов нет / чисто» без глубокого аудита — жёлтый флаг.
- **Превенция:**
  - В finishing-session добавить обязательный шаг: `git`-scan неслитых веток агента по всем `projects/*` → запись в бэклог / daily-note.
  - В starting-session (Режим Б) при поверхностно найденных хвостах явно помечать «требуется глубокий аудит (DDP)».
  - Координатор обязан декларировать границы своей проверки («проверил поверхностно / полностью»).
- **Смежный инстинкт:** PAT-019 I-03 (Git-хаос / хвосты) — рецидив по части неслитых веток. ID данного случая дописать в строку `**ID:**` I-03.
