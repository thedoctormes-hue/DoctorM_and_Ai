# Security-патруль перенёс workspaces/*/secrets → vault, сломав хардкоды путей → семпамять легла ~2ч (lab-wide)

## Описание
Около 21:01 МСК security-патруль (корректно по политике безопасности) перенёс каталоги секретов агентов `workspaces/*/secrets/` в единый сейф `/root/LabDoctorM/vault/`. Агенты (включая меня) хранили там токены ALM, `.env`-файлы и **хардкодили абсолютные пути** к `workspaces/<agent>/secrets/...` в инфраструктурных скриптах:
- `.ops/shared/anythingllm-sync/sync.py` → `TOKEN_FILE` / `MAP_FILE` / `STATE_FILE`
- `projects/mcp-tools/memory-gateway/memory_gateway/config.py` → `TOKEN_FILE` / `MAP_FILE`
- `.ops/shared/anythingllm-sync/start-anythingllm.sh` → `--env-file workspaces/.../secrets/anythingllm.env`

После переноса все эти пути стали битыми → векторный слой семантической памяти (`memory-gateway` → ALM `:3002`) стал недоступен для **ВСЕХ 8 агентов**. Семантический поиск лёг lab-wide на ~2 часа.

## Обнаружение
- MCP vector-search возвращал `degraded` / пусто для всех агентов с ~21:01.
- Корень найден при диагностике: `memory-gateway__gateway_health` → `vector_layer.reachable=false`, причина «token file not found: /root/LabDoctorM/workspaces/streikbrecher/secrets/anythingllm_token.txt».
- Проверено: `/root/LabDoctorM/vault/anythingllm_token.txt` существует и валиден (на тот момент `vectorCount≈8509`). То есть токен жив, сломан только хардкод пути.

## Решение
- Все три хардкода перенаправлены на `/root/LabDoctorM/vault/`:
  - `sync.py`: `TOKEN_FILE`/`MAP_FILE` → `vault/`, `STATE_FILE` → каталог скрипта (операционные данные).
  - `config.py`: `TOKEN_FILE`/`MAP_FILE` → `vault/`.
  - `start-anythingllm.sh`: `--env-file` → `vault/anythingllm.env`.
- Перед каждой правкой снят бэкап (`*.bak.pre-vaultfix-*`).
- Перезапущен `memory-gateway.service` → векторный слой восстановлен lab-wide ~23:14.
- Подтверждено: `memory-gateway__gateway_health` → `vector_search.functional=true`, `degraded=false`.
- Отдельный инцидент `INC-20260719-233741-975de5` (ALM не пережил ребут) — независимый root cause, закрыт отдельно.

## Уроки
1. **Секреты НЕ хранятся в workspaces.** Workspaces — рабочая зона агента (код, память, навыки), НЕ сейф. Секреты только в `/root/LabDoctorM/vault/`.
2. **Никаких хардкодов абсолютных путей к секретам** в инфра-скриптах. Читать из единого источника (`vault/`) через переменные окружения / централизованный конфиг.
3. Любая миграция общего ресурса (перенос secrets, смена расположения) = **cutover**: найти ВСЕ зависимые хардкоды ДО переноса, обновить, проверить. Иначе молча ложится зависимый слой (RUL-009/010 соблюдён — сам ALM не трогал, только пути).
4. Семантическая память — общая инфра. Один битый путь к токену убил поиск у всех 8 агентов. Нужен post-migration health-check.
