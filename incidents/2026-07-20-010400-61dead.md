# Коллизия номеров ADR — ADR-0059 (надзор Совы) vs ADR-0059 (branch-ownership guard)

## Описание
В DDP-аудите чекпоинта обнаружено: утверждение «main защищён ADR-0059» ошибочно по атрибуции.
Файл `adr/ADR-0059-owl-all-agents-oversight.md` описывает надзор Совы за всеми агентами и НЕ содержит
правила защиты ветки `main`/merge-gating. При этом живой branch-ownership guard при блокировке моего
пуша выдал: `BLOCKED by branch-ownership guard (ADR-0059 / PAT-019 I-03)`, а `ADR-046` в разделе
«Связанные» цитирует: «ADR-0059: защита ветки master (branch-ownership guard)».

Итог: номер **0059 используется для ДВУХ разных ADR**:
- ADR-0059 = надзор Совы (заголовок файла `ADR-0059-owl-all-agents-oversight.md`);
- ADR-0059 = branch-ownership guard (код гейта + ссылка в ADR-046).

Корень: номера ADR назначаются без централизованной проверки уникальности. `ADR-INDEX.md`
(предполагаемый SSOT-индекс) сам содержит системные коллизии: `ADR-0056` встречается 3× (gatekeeper-ssot,
skill-creation-standard, «Единый реестр инцидентов»), `ADR-001` — 2× (custom-skills-restructure, Myrmex Control).
При этом `0059` в `ADR-INDEX.md` ВООБЩЕ отсутствует, хотя его claimят два документа — значит индекс не является
надёжным SSOT.

Последствие: неоднозначность ссылок и ложная атрибуция ответственности. Мой DDP-отчёт изначально (неверно)
приписал защиту main ADR-0059 про надзор Совы именно из-за этой коллизии. Любая ссылка «ADR-0059» неоднозначна.

## Паттерн (инстинкт)
- **Root cause (абстрактный):** номера документов/ADR назначаются без принудительной проверки уникальности
  при создании → два документа делят один номер → неоднозначные ссылки и ложная атрибуция ответственности/блейма.
- **Signal:** ссылка «ADR-NNN» резолвится в >1 документ; код/гейт цитирует номер ADR, заголовок которого
  не совпадает с цитируемым назначением; `ADR-INDEX.md` показывает дубли номеров; номер заявлен документом,
  но отсутствует в индексе.
- **Превенция:** принудительный lint уникальности номеров ADR через `ADR-INDEX.md` на этапе создания
  (отклонять дубли NNN); канонический SSOT = `ADR-INDEX.md` (довести до полноты); при цитировании ADR
  ссылаться на slug (имя файла), а не только на номер.
- **Рецидив существующего инстинкта в PAT-019:** НЕТ — `I-19` (doc-drift) покрывает рассинхрон статуса
  инцидента между прозой и реестром, иной root cause. Это новый класс (нумерная коллизия ADR).
  После закрытия — завести новый инстинкт `I-NN` в PAT-019 (автор библиотеки — Сова; предложить ей).
