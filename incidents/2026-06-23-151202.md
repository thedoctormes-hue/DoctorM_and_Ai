# Агенты без message tool — delivery может не работать

## Описание

`openclaw doctor` показал warnings для ВСЕХ 8 агентов:
- antcat, bestia, dominika, kotolizator, mangust, owl, raven, streikbrecher

Все они маршрутизируются через Telegram, но не имеют инструмента "message" в tool allowlist.
Это означает, что channel actions (sendAttachment, upload-file, thread-reply, reply) могут не работать.

**Влияние:** cron-задачи с delivery на accountId="dominika", "mangust" и другие могут корректно
работать только при наличии message tool у агента, указанного в delivery.accountId.

## Обнаружено

23.06.2026 при запуске `openclaw doctor`.

## Решение

1. Добавить "message" в tools.allow агентов, ИЛИ
2. Использовать accountId агента, у которого есть message tool (kotolizator)

**Примечание:** Мы уже исправили mangust-adr-scan delivery на accountId="kotolizator". Нужно проверить и остальные.

## Уроки

При создании cron с delivery на конкретный accountId — убедиться что у этого агента есть message tool.
