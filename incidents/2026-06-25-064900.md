# Завышенное число скилов в ответе без проверки

## Описание
ЗавЛаб попросил «Отвечай только цифрами» и задал два вопроса:
1. Сколько скилов в `~/.openclaw/skills/`
2. Сколько из них зарегистрировано у агента

Ответ: «1. 34, 2. 34».

**Проблема:** Число 34 — это количество скилов из системного промпта `<available_skills>`, который включает подмножество всех скилов системы (built-in + workspace + plugin-skills), **а не** количество скилов в конкретной папке `~/.openclaw/skills/`.

Реальное число в `~/.openclaw/skills/` — **14** (проверено через `ls -1d`).

Агент выдал число из одного контекста (available_skills) как ответ на конкретный вопрос о другой папке (`~/.openclaw/skills/`), без верификации.

## Обнаружение
ЗавЛаб потребовал фактчекинга. Агент перепроверил через `ls -1d` и обнаружил расхождение:
- `~/.openclaw/skills/` → 14 (не 34)
- `~/.nvm/.../openclaw/skills/` (built-in) → 57
- `~/.openclaw/plugin-skills/` → 4
- `~/LabDoctorM/workspaces/*/skills/` → 9
- **ВСЕГО на диске: 84**

Число 34 из `<available_skills>` — это **подмножество** из разных источников, а не только `~/.openclaw/skills/`.

## Решение
1. Инцидент создан
2. Урок записан в MEMORY.md
3. Фактчекинг проведён публично

## Уроки (навсегда)
1. **Конкретный вопрос — конкретный ответ с указанием источника.**
2. **available_skills ≠ конкретная папка на диске.** Это разные вещи.
3. **НЕ выдавай число из одного контекста как факт о другом объекте** без верификации.
4. **При ответе на вопрос о конкретном пути/папке — проверяй через `ls`, не через память или системный промпт.**
5. **«Отвечай только цифрами» не освобождает от фактчекинга** — цифра должна быть точной.
