# Нарушение изоляции worktree — анткат и сова работали в общем checkout, коллизия коммитов (рецидив I-03)

## Описание
В лаборатории правило изоляции (PAT-019 I-03) требует работать в изолированном git-worktree, а не в общем checkout. Фактически агенты пишут напрямую в ОДИН общий checkout `/root/LabDoctorM/projects/DoctorM_and_Ai` (git worktree list показывает ровно один worktree). 2026-07-18 в этом общем checkout анткат и сова работали параллельно: анткат правил `skills-canon/accepting-work/SKILL.md`, сова гнала миграцию реестра инцидентов (ветка `owl/incident-canon-migrate`).

Из-за отсутствия изоляции мой `git commit` (без флага -a, но с застейженным добром совы в общем индексе) схавал ЕЁ незакомиченные ренеймы инцидентов + мой файл и записал это на ЕЁ ветку под моим именем. Произошло дважды (коммиты 75bf720, 078d4f2 на ветке совы). Данные совы не потеряны — анткат откатил свои коммиты с её ветки, вытащил свой файл патчем на main (7b75497), её ренеймы/WIP зафиксировал под ЕЁ именем (bfe3513, запушено). Дерево везде чистое, конфликтов нет.

## Обнаружение
Анткат применил собственный же паттерн verification-loop: перед утверждением о статусе сверил git-факты (git status / git log / git diff), а не память. Обнаружил, что его коммит ушёл не на main, а на ветку совы, и схавал её staged-правки.

## Паттерн (инстинкт)
Рецидив PAT-019 **I-03** (Git-хаос / worktree/branch contamination, прямой commit в общий checkout).
- **Root cause (абстрактный):** агент работает напрямую в общем checkout вместо изолированного worktree; нет механического guard, запрещающего коммит/правку в общий checkout.
- **Signal:** `git worktree list` показывает один worktree, а в нём параллельно пишут несколько агентов; коммит одного агента забирает staged-правки другого.
- **Превенция:** перед правкой в DoctorM_and_Ai — `git worktree list`; если checkout общий и в нём кто-то ещё — создать свой worktree (`git worktree add -b <agent>/<task> <path> main`) и работать там. ID этого случая дописать в PAT-019 I-03 (зона Совы).

## Resolution (draft — подготовлено для закрытия ЗавЛабом, ADR-0057)
- **Корень:** PAT-019 I-03 не был принудительным — нет guard против коммита в общий checkout `/root/LabDoctorM/projects/DoctorM_and_Ai`.
- **Фикс (внедрён):** `bin/check-worktree-isolation.sh` блокирует коммит в главный checkout; вызывается из `git-hooks/pre-commit` и `bin/lab-commit.sh`. Bypass для инфры/ЗавЛаба: `LAB_ALLOW_MAIN_COMMIT=1`. Слито в main (75be3cc). Протестировано: главный checkout -> BLOCKED (exit 1), изолированный worktree -> pass.
- **PAT-019 I-03 обновлён** (ab2b638): добавлен ID `INC-20260718-101833-c4f1d4` и конкретная ссылка на guard.
- **Результат:** коллизия antcat<->owl (съеденный WIP) теперь невозможна механически.

## Lessons (draft)
- Изоляция worktree должна быть enforce (guard), а не только прописана в PAT.
- При внедрении guard сразу дать агентам worktree, иначе парализует их коммиты (Сова сама создала `DoctorM_and_Ai-wt-owl`).
- Инцидент-ветку мержить в main сразу, иначе инцидент «не в каноне».
