# INC-025: Darkbloom/Harmony multiple tool_calls при failover и compaction

## Ошибка
```
Upstream error from Darkbloom: inference failed after 64 attempt(s):
assistant message contains multiple tool_calls;
Harmony supports one tool call per assistant message
```

## Корневая причина
1. `openai/gpt-oss-20b:free` на OpenRouter маршрутизируется на провайдера **Darkbloom**
2. Darkbloom использует **Harmony response format** (gpt-oss модели обучены на нём)
3. Harmony требует **1 tool call per assistant message**
4. OpenClaw отправляет 2+ tool_calls в одном assistant message (стандарт OpenAI API)
5. Darkbloom отвергает запрос с ошибкой 502 → 64 retry → compaction timeout → killed session

## Масштаб
- **29 ошибок** по всем агентам (множественные tool_calls + Harmony)
- antcat (4 сессии), dominika (2), main (2) — Darkbloom ошибки
- raven, owl, streikbrecher, kotolizator, bestia, mangust — compaction timeout и killed sessions (вторичный эффект)
- gpt-oss-20b использовался в 3 местах: fallbacks, compaction.memoryFlush, intention-hint

## Решение
- Замена `openai/gpt-oss-20b:free` → `openai/gpt-oss-120b:free` во всех 3 местах
- gpt-oss-120b с reasoning лучше справляется с Harmony format
- Коммит: `87d4251`, ветка `fix/darkbloom-harmony-compaction-2026-06-21`

## Остаточные проблемы
- gpt-oss-120b тоже таймаутится при compaction (2 случая) — требует доп. исследования
- SessionWriteLockTimeout для raven (8 раз)

## Урок
- Harmony response format имеет ограничение 1 tool call per message
- Free-модели на OpenRouter маршрутизируются на разных провайдеров (Darkbloom, Arcee и т.д.)
- При выборе fallback-модели нужно учитывать не только цену, но и формат ответа

## Затронутые агенты
antcat, dominika, main (прямые ошибки); raven, owl, streikbrecher, kotolizator, bestia, mangust (косвенные — timeout/kill)
