# INC-20260717-0917: Удаление myrmex.json через git rm сломало backup-myrmex и вызвало тихую деградацию myrmex-control

- **Дата:** 2026-07-17 09:17 МСК
- **Инициатор:** Муравей (commit 00d1eb7), go ЗавЛаба (merge 7c38f2f «go ЗавЛаба 2026-07-16»)
- **Действие:** `git rm myrmex.json` (18 696 deletions) в коммите 00d1eb7 «chore: ignore auto-generated myrmex.json» (23:14 МСК 16 Jul)
- **Причина:** auto-generated файл помечен ignore при git-гигиене, но `git rm` удалил его с диска (а не `--cached`), хотя `.env` (MYRMEX_FILE) и сервис/бэкап от него зависят
- **Симптомы:**
  - `backup-myrmex.service` — FAILED каждый час (не находит файл)
  - `myrmex-control.service` — active, но `myrmex.ts:71` fallback `createDefaultState()` → тихо отдаёт пустое состояние с 23:14 16 Jul (реальные projects/agents/tasks не загружаются)
- **Root cause (верифицировано Вороном, live probes):** не миграция на БД, а `git rm` при git-гигиене
- **Решение (рекомендация археолога, delegated to antcat):** восстановить из `/var/backups/myrmex/auto-2026-07-16T20-00-00.json` (531 380 B, 23:00 МСК 16 Jul); `git rm --cached` + `.gitignore`; перезапуск сервиса; фикс backup; алерт на FAILED
- **Статус:** делегировано анткату (spawn subagent agentId=antcat, taskName=backup_myrmex_restore)
- **Lessons:** `git rm` vs `git rm --cached` при untrack auto-generated файлов; алерт на FAILED systemd-сервисы (тихая деградация недопустима)
