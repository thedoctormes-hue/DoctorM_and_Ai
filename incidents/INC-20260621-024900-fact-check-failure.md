# Инцидент: Фактчекинг рекомендаций по плагинам OpenClaw

**Дата:** 2026-06-21 02:49 UTC
**Агент:** Доминика (Scout)
**Тип:** Фантазии вместо фактов — не проверила рекомендации на актуальность
**Серьёзность:** Medium
**Статус:** Закрыт

## Что произошло

При составлении экспертного отчёта по плагинам OpenClaw выдала рекомендации по активации плагинов **без проверки их актуальности на живой системе**.

## Конкретные ошибки

### 1. Рекомендация: ollama — "локальные LLM для приватных задач"

**Факты:**
- Ollama установлена, но есть только embedding-модели (bge-m3), нет LLM-моделей
- Нет GPU, 8 GB RAM — LLM на CPU не запустится или будет работать на пределе
- Для 7B+ модели нужно ~4-5 GB RAM минимум

**Вывод:** Рекомендация неактуальна. Активация ollama без скачивания LLM-модели бессмысленна, а скачивание модели на 8 GB RAM без GPU — проблематично.

### 2. Рекомендация: web-readability — "извлечение контента из HTML"

**Факты:**
- Плагин существует, но disabled
- У нас уже есть tavily (работает) и firecrawl (работает)
- web-readability дублирует функциональность существующих плагинов

**Вывод:** Рекомендация неактуальна. Реальной потребности нет — функциональность закрыта существующими плагинами.

### 3. Рекомендация: llm-task и file-transfer

**Факты:**
- Плагины существуют и disabled
- Не проверено, есть ли реальные задачи лаборатории, которые они закроют

**Вывод:** Рекомендация без подтверждения потребности — потенциально бесполезная.

## Корневая причина

**Не провела фактчекинг рекомендаций перед публикацией отчёта.**

Отчёт содержал раздел "Топ плагинов для активации в лаборатории", который был основан на:
- Общих предположениях о полезности плагинов
- Документации OpenClaw (что плагин делает)
- Без проверки: работает ли, нужен ли, дублирует ли существующий функционал

## Что должно было быть сделано

Перед рекомендацией каждого плагина:
1. Проверить наличие зависимостей (Docker, GPU, RAM, внешние сервисы)
2. Проверить, не дублирует ли функциональность существующих плагинов
3. Проверить наличие реальных задач лаборатории, которые плагин закроет
4. Если проверка невозможна — явно указать "требует дополнительной проверки"

## Исправление

- Отчёт обновлён до v2 с фактчекингом всех утверждений
- Рекомендации по ollama и web-readability признаны неактуальными
- Добавлен раздел "Что было исправлено" для прозрачности

## Уроки

1. **Рекомендации = фактчекинг.** Нельзя рекомендовать плагин без проверки его работоспособности и актуальности на живой системе.
2. **Дублирование.** Перед рекомендацией нового плагина проверять, не закрывает ли его функциональность существующий.
3. **Ресурсы.** Проверять наличие ресурсов (RAM, GPU, внешние сервисы) перед рекомендацией.
4. **Потребность.** Рекомендовать только при наличии подтверждённой потребности, а не "на всякий случай".
