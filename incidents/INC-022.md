# INC-022: Сова отключила deep-research-lab у всех агентов при очистке дубликатов

## Хронология

1. 22:02 UTC — Сова проводит исследование кастомных скилов
2. 22:07 UTC — формируется отчёт с рекомендацией удалить дубликат deep-research-lab из raven workspace
3. 22:22 UTC — ЗавЛаб согласовывает удаление (пункт 4)
4. 22:23 UTC — Сова удаляет `workspaces/raven/skills/deep-research-lab/` и коммитит
5. 22:38 UTC — ЗавЛаб спрашивает почему deep-research-lab не зарегистрирован
6. 22:40 UTC — ЗавЛаб требует немедленную проверку
7. 22:49 UTC — Сова подтверждает: скил не был в allowlist, удаление workspace-копии было ошибкой

## Корневая причина

Сова при очистке дубликатов скилов из workspaces **не проверяла наличие каждого скила в allowlist агентов**. Действовала по принципу "дубликат = удалить", не убедившись что canonical-версия зарегистрирована в конфигурации.

## Что сломалось

- deep-research-lab существовал на диске (`~/.openclaw/skills/deep-research-lab/`) но **не был зарегистрирован** в `agents.defaults.skills` и ни в одном explicit списке агентов
- после удаления workspace-копии скил остался только как файл на диске, невидимый для всех 9 агентов
- скил был фактически отключён (не подтягивался в system prompt)

## Что исправлено

- deep-research-lab добавлен в `agents.defaults.skills` и explicit списки всех 9 агентов
- конфигурация обновлена через gateway runtime

## Урок

**Перед удалением любого workspace-скила — ВСЕГДА проверять:**
1. Есть ли скил в `agents.defaults.skills`?
2. Есть ли скил в explicit списке агентов (`agents.list[].skills`)?
3. Если нет — сначала добавить в allowlist, потом удалять дубликат
