# INC-030: Потеря FAISS-индекса при reindex --full

## Что произошло
Запустила `reindex.py --full` — он стёр все 14307 чанков из SQLite (`DELETE FROM chunks`) и начал грузить с нуля. Процесс завис (OOM), и labsearch остался на 7% мощности (~1029 векторов из бэкапа).

## Корень проблемы
- `--full` делает `DELETE FROM chunks` без бэкапа
- Нет механизма отката если процесс упал посреди переиндексации
- FAISS-индекс не пересобирается автоматически — остаётся устаревшим

## Влияние
- labsearch на ~7% мощности ~8 часов (с 14:00 до 22:00 UTC)
- Все агенты колонии не могли нормально искать по памяти

## Решение
1. Восстановила FAISS из .bak файла
2. Переключилась на инкрементальный reindex (не стирает старые данные)
3. Пересобрала FAISS вручную через build_faiss_and_meta()

## Урок
- **НЕ запускать --full без подготовки** — сначала сделать бэкап FAISS
- **Сначала инкрементальный** — он безопасен
- **Думать о влиянии на продакшен ДО действия**

## Красная линия #12
НЕ запускать reindex.py --full без подготовки — стирает все чанки, labsearch падает. Сначала инкрементальный.
