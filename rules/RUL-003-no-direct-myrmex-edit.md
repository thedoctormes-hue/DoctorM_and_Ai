---# RUL-003: Запрет прямого редактирования myrmex.json

## Категория
api

## Описание
Файл `myrmex.json` — это БД сервиса myrmex-control. Прямое редактирование файла приводит к перезаписи через `writeState()` с потерей данных (users, refresh_tokens и др. поля, которые не загружены в память процесса).

## Обязательно
- [ ] Все изменения в данные myrmex — только через API сервиса
- [ ] При необходимости прямого редактирования — остановить сервис, отредактировать, запустить
- [ ] Перед прямым редактированием — создать резервную копию
- [ ] `writeState()` должен мержить с файлом, а не перезаписывать (BL-012)

## Исключения
Только с остановленным сервисом и резервной копией.

## Наказание за нарушение
**incident** — потеря пользователей, открытый setup, полная недоступность сервиса.

## Связанные артефакты
- INC-20260512111530 — Потеря users в myrmex.json
- BL-012 — Исправление writeState()

## Примечание
Инцидент 2026-05-12: прямое редактирование myrmex.json → writeState() перезаписал файл → потеря users и refresh_tokens → форма создания администратора открыта для всего интернета на ~3 минуты.
