# RUL-009: Ручную реиндексацию семантической памяти (ALM) выполняет ТОЛЬКО Штрейкбрехер с разрешения ЗавЛаба

## Принцип
Ручная реиндексация семантической памяти лаборатории (AnythingLLM / ALM, доступ через
MCP `memory-gateway`) выполняется **исключительно Штрейкбрехером** и **только с явного
разрешения ЗавЛаба**. Никакой другой агент не запускает реиндекс вручную.

Это **ЗАКОН** лаборатории — установлен ЗавЛабом 2026-07-16, эскалация из
`INC-2026-07-16-000301` (antcat самостоятельно запустил `run-sync.sh rebuild`).

## Категория
infra / process

## Описание
- **Ручной реиндекс** = любой ручной вызов пайплайна синхронизации семпамяти:
  - `bash run-sync.sh rebuild` (полная пересборка)
  - `python3 sync.py --rebuild`
  - `bash purge_vectors.sh` (очистка векторного слоя)
  - `docker restart anythingllm` (сброс состояния ALM)
  - любой прямой вызов API ALM `:3002` на переэмбеддинг/загрузку корпуса
- Эти действия меняют ОБЩЕЕ состояние семпамяти, которым пользуются все 8 агентов.
  Произвольный реиндекс = риск временного опустошения ALM, гонки эмбеддингов,
  рассинхрона с источником (git = Source of Truth).
- **Автоматический инкрементальный sync** (systemd-таймер `anythingllm-sync.timer`,
  вызывающий `run-sync.sh incremental`) — инфраструктура лаборатории, не требует
  участия агента и не считается «ручным реиндексом». Агенты его не трогают и не
  триггерят вручную (это тоже попадает под запрет — только владелец настраивает таймер).

## Обязательно
- [ ] Ручной реиндекс ALM делает ТОЛЬКО Штрейкбрехер.
- [ ] ТОЛЬКО с явного письменного разрешения ЗавЛаба (в чате/тикете).
- [ ] Любой агент, видящий необходимость реиндекса (устаревший корпус, отравление
      индекса, новые доки не индексируются) → доложить ЗавЛабу и Штрейкбрехеру,
      НЕ запускать `run-sync.sh`/`sync.py` самостоятельно.
- [ ] Канон доступа к семпамяти: ТОЛЬКО `memory-gateway__search_memory` (MCP).
      Прямые вызовы ALM `:3002`, `lab_search.py`, `mcp-memory :8087`, `onnx-embedder :8082`
      — ЗАПРЕЩЕНЫ (см. APPEND_SYSTEM.md и семантическую память RUL-канон).

## Исключения
Только прямое письменное поручение ЗавЛаба конкретному агенту выполнить конкретный
реиндекс. Даже Штрейкбрехер без разрешения ЗавЛаба не запускает ручной rebuild.

## Наказание за нарушение
Инцидент severity: high (как `INC-2026-07-16-000301`). Самовольный ручной реиндекс =
нарушение ЗАКОНА лаборатории + вторжение в зону владельца семпамяти (ADR-014).

## Связанные артефакты
- `INC-2026-07-16-000301` — инцидент, породивший закон (antcat unsanctioned reindex).
- `INC-2026-07-16-000300` — вторжение raven в legacy ALM :3002 (смежная зона).
- `RUL-008` — AWG настраивает ТОЛЬКО ЗавЛаб (тот же паттерн «ЗАКОН: только владелец»).
- `anythingllm_semantic_memory_backlog.md` (workspace streikbrecher) — канон семпамяти.
- `APPEND_SYSTEM.md` (per-agent) — триггер доступа к семпамяти через MCP.
