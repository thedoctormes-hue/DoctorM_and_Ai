# RUL-012: VPN-инфраструктура (AWG + X-ray + Mtproto) — ТОЛЬКО ЗавЛаб

## Принцип
Конфигурация и порты VPN-инфраструктуры сервера изменяются **исключительно ЗавЛабом**:
- **AmneziaWG (AWG):** контейнер `amnezia-awg2`, порт 36713 (базовый закон — RUL-008).
- **X-ray:** vless-reality (`127.0.0.1:443`) + vless-xhttp (`127.0.0.1:10443`); systemd unit `xray-poland.service`; конфиг `/usr/local/etc/xray/config.json`.
- **Mtproto:** MTProto-прокси (порт по конфигу ЗавЛаба).

Никакой агент не трогает порты, конфиги, контейнеры и процессы этих сервисов без прямого письменного поручения ЗавЛаба. Это **ЗАКОН** лаборатории.

## Категория
infra / security

## Описание
- Агенты не редактируют конфиги, не меняют порты, не перезапускают сервисы (systemd unit / docker container), не правят nginx-stream маршрутизацию, затрагивающую VPN-порты.
- Вмешательство агента (даже с «благими» намерениями — «починить VPN») ломает клиенты ЗавЛаба и отрезает связь.
- Исключение: прямое письменное поручение ЗавЛаба конкретному агенту выполнить конкретное действие.

## Обязательно
- [ ] Порты/конфиги AWG, X-ray, Mtproto меняет ТОЛЬКО ЗавЛаб.
- [ ] Агенты не правят конфиги, не перезапускают сервисы, не трогают занятые порты (36713, 443, 10443, порт Mtproto).
- [ ] Любой запрос на изменение VPN → направлять ЗавЛабу, не делать автономно.
- [ ] Если агент видит проблему в VPN — доложить ЗавЛабу, НЕ чинить самому.
- [ ] Сервисы держать managed (systemd `xray-poland.service` / docker `amnezia-awg2`), не nohup-процессами от сессии агента.

## Исключения
Только прямое письменное поручение ЗавЛаба конкретному агенту выполнить конкретное действие.

## Наказание за нарушение
Инцидент severity: high / critical. Самовольное изменение VPN-инфраструктуры = нарушение ЗАКОНА лаборатории.
