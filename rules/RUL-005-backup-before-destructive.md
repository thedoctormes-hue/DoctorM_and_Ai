---# RUL-005: Резервная копия перед деструктивными операциями

## Категория
deploy

## Описание
Перед любой деструктивной операцией (удаление кода, замена файлов, рефакторинг) необходимо создать резервную копию. Без резервной копии — нет отката.

## Обязательно
- [ ] Перед удалением кода — создать backup файла
- [ ] Перед заменой рабочего кода — создать backup
- [ ] Перед масштабным рефакторингом — создать commit с пометкой `pre-refactor`
- [ ] Резервные копии хранятся минимум 7 дней

## Исключения
Нет.

## Наказание за нарушение
**warn** — при первой попытке, **error** — при повторном.

## Связанные артефакты

- ADR-012 — Dual Auth: деплой перезаписывает server-dist, backup обязателен
- ADR-015 — LLM Stack: изменение конфигурации моделей требует backup
- INC-20260508 — Некорректное удаление функционала
- INC-20260509 — Уничтожение dashboard проекта

## Примечание
Два инцидента подряд (2026-05-08 и 2026-05-09) связаны с отсутствием резервных копий. Потеря: неделя разработки.
