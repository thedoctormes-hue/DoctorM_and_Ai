---# RUL-001: Запрет токенов и секретов в git

## Категория
security

## Описание
Токены, API-ключи, пароли и другие секреты НЕ ДОЛЖНЫ попадать в git history. Даже в legacy-проектах. Даже в документации. Даже в комментариях.

## Обязательно
- [ ] Все секреты хранятся только в `.env` файлах
- [ ] `.env` добавлен в `.gitignore`
- [ ] Перед коммитом проверяется отсутствие токенов в изменяемых файлах
- [ ] При обнаружении токена в history — немедленная ротация + BFG cleanup
- [ ] **Pre-push hook установлен** (`.githooks/pre-push`) — блокирует push при наличии секретов
- [ ] **Перед push нового проекта — полный secret-scan всей директории**
- [ ] **Публичный репозиторий — только фронтенд + README, без бэкенда и кредов**
- [ ] **Никогда не хранить токены в remote URL** — использовать SSH или credential helper
- [ ] **Файлы .orig, .bak, .backup — в .gitignore** (бэкапы конфигов содержат секреты)
- [ ] **При пересоздании репо — проверять КАЖДЫЙ файл перед коммитом**

## Исключения
Нет. Никаких исключений для секретов в git.

## Наказание за нарушение
**incident** — автоматическое создание инцидента безопасности, ротация всех утёкших токенов.

## Примечание
Инцидент 2026-05-06: токены Telegram ботов и OpenRouter API были зафиксированы в git history.

**Инцидент 2026-05-17 (INC-004):** OWL запушила весь репозиторий лаборатории без secret-scan. В репо попали:
- 4 Telegram bot tokens
- GitHub PAT
- Пароли от Habr/VC.ru
- Файл platform_credentials.md с логинами/паролями

**Урок:** Даже приватный репозиторий требует полной проверки перед push. Публичный репозиторий должен содержать ТОЛЬКО фронтенд + README.

**Инцидент 2026-05-17 (INC-004, повторная утечка):** При пересоздании репозитория «чистая история» содержала 25 коммитов с секретами. Файлы .orig не были в .gitignore. GitHub PAT был в remote URL. Урок: пересоздание репо не заменяет secret-scan.
