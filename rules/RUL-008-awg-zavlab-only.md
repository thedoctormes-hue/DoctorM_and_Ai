# RUL-008: AmneziaWG (AWG) на сервере настраивает ТОЛЬКО ЗавЛаб

## Принцип
Конфигурация AmneziaWG 2.0 на VPN-сервере (`amnezia-awg2`, порт 36713) изменяется
**исключительно ЗавЛабом**. Никакой агент не трогает AWG, его контейнер и порты,
которые AWG занимает. Это **ЗАКОН** лаборатории — установлен ЗавЛабом 2026-07-14,
эскалация из `INC-20260624-145900`.

## Категория
infra / security

## Описание
- AWG server-side параметры (Jc, Jmin, Jmax, S1-S4, H1-H4, I1-I5) — **server-only**.
  Клиент AmneziaVPN получает их по management-протоколу, а не из `.conf`-файла.
- Вмешательство агента (даже с «благими» намерениями) ломает VPN-клиенты ЗавЛаба
  и отрезает связь.
- Агенты не редактируют AWG-конфиги, не перезапускают контейнер `amnezia-awg2`,
  не трогают занятый порт 36713 и сопутствующие AWG-порты.

## Обязательно
- [ ] AWG server config меняет ТОЛЬКО ЗавЛаб.
- [ ] Агенты не правят AWG-конфиги, не перезапускают `amnezia-awg2`, не трогают порт 36713 (и другие порты AWG).
- [ ] Любой запрос на изменение AWG → направлять ЗавЛабу, не делать автономно.
- [ ] Если агент видит проблему в AWG — доложить ЗавЛабу, НЕ чинить самому.

## Исключения
Нет. Единственное исключение — прямое письменное поручение ЗавЛаба конкретному
агенту выполнить конкретное действие.

## Наказание за нарушение
Инцидент severity: critical (как `INC-20260624-145900`). Самовольное изменение AWG =
нарушение ЗАКОНА лаборатории.

## Связанные артефакты
- `INC-20260624-145900` — инцидент, породивший закон.
- `SOUL.md` / `AGENTS.md` (kotolizator) — red line «не трогать AWG».
- `PORT_REGISTRY.md` — порт 36713 закреплён за AWG (не трогать).
