**От:** ЗавЛаб + Ворон

## Задача

Посчитай реальную экономию токенов при загрузке контекста через Context API vs локальные файлы.

## Что нужно сделать

### Шаг 1: Текущее потребление (локальные файлы)

Запусти новую сессию и посчитай сколько токенов/строк ты читаешь при старте. Вот что ты обычно грузишь:

1. `QWEN.md` — основной контекст лаборатории
2. `QWEN-core.md` — ядро системы
3. `QWEN-rules.md` — правила
4. `QWEN-staff.md` — лаборанты
5. `QWEN-projects.md` — проекты
6. `IDENTITY.md` — своя идентичность
7. `SOUL.md` — своя душа
8. `CHECKPOINT.md` — чекпоинт
9. Файлы памяти из `/root/.qwen/projects/-root-LabDoctorM/memory/`
10. `SESSION_HANDOFF.md` если есть

Для каждого файла:
- Путь к файлу
- Количество строк
- Примерное количество токенов (строки × 4 — грубая оценка, или используй `wc -l` и `wc -c`)

### Шаг 2: Потребление через API

Теперь посчитай сколько токенов ты потратишь если будешь грузить через Context API:

1. `QWEN-core.md` — читаешь локально (там описана схема работы с API)
2. Дальше вместо файлов — API запросы:
   - `GET /api/v1/context/staff` — вместо QWEN-staff.md
   - `GET /api/v1/context/projects` — вместо QWEN-projects.md
   - `GET /api/v1/context/core` — вместо QWEN.md (частично)
   - `GET /api/v1/memory/<topic>` — вместо файлов памяти
   - `GET /api/v1/project/<name>` — только нужный проект

Для каждого API запроса:
- Какой эндпоинт
- Размер ответа (строки/токены)
- Что из этого реально нужно Бестии

### Шаг 3: Сравнение

- Итого токенов сейчас (локальные файлы)
- Итого токенов через API
- Разница в процентах

### Шаг 4: Критический вопрос — теряешь ли ты что-то важное?

Вот что нужно проверить:

1. **Полнота данных** — всё ли из локальных файлов есть в API? Что отсутствует?
2. **Твои специфичные нужды** — Бестия работает с инфраструктурой, серверами, VPN. Есть ли в API данные о:
   - Серверах и их статусах
   - VPN конфигурациях
   - Зонах ответственности
   - Критических сервисах
3. **Контекст принятия решений** — ADR, инсайты, логи решений. Доступны ли они через API?
4. **Свежесть данных** — API отдаёт данные с диска в реальном времени, или есть кеширование? Если файл изменился — API вернёт актуальное?
