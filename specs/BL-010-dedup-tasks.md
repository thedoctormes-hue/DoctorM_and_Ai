# BL-008: Реализовать дедупликацию задач в бэклоге

## Контекст
При эволюции лаборатории в specs/ могут появляться дублирующие задачи. Нет механизма обнаружения и устранения дублей — агенты делают одну и ту же работу дважды.

## Цель
Реализовать скрипт дедупликации specs/ на основе семантического сравнения заголовков и summary.

## Зачем
Исключить двойную работу и поддерживать бэклог в чистоте.

## Проект/контекст
муравейник/бэклог

## Что сделать
- [ ] Создать dedup_specs.sh — сравнение по ключевым словам + Levenshtein distance
- [ ] Реализовать порог схожести 0.8 — выше считается дублем
- [ ] При обнаружении дубля — добавлять тег duplicate + ссылку на оригинал
- [ ] Интегрировать в pre-commit hook
- [ ] Добавить отчёт дедупликации в session_startup.sh

## Критерии готовности
- [ ] dedup_specs.sh находит дубли с порогом 0.8
- [ ] Дубли помечаются тегом duplicate
- [ ] Pre-commit hook предупреждает при добавлении дубля
- [ ] Отчёт показывает найденные дубли при старте сессии

## Зависимости
- нет
