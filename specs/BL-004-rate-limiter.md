# BL-003: Добавить rate limiter для API вызовов

## Контекст
Агенты могут делать множество API-вызовов за сессию. Без rate limiter возможны блокировки со сторонних сервисов и бессмысленная трата квот.

## Цель
Реализовать rate limiter с очередью запросов и exponential backoff.

## Зачем
Защитить API ключи от блокировки и равномерно распределить нагрузку.

## Проект/контекст
муравейник/инфраструктура

## Что сделать
- [ ] Создать rate_limiter.sh с очередью и приоритетами
- [ ] Реализовать sliding window counter (60 запросов/мин)
- [ ] Добавить exponential backoff при 429 ответах
- [ ] Интегрировать в agent_tool_call wrapper
- [ ] Добавить алертинг при достижении лимита

## Критерии готовности
- [ ] Запросы не превышают 60/мин на сервис
- [ ] При 429 — backoff с джиттером
- [ ] Приоритетные запросы проходят первыми
- [ ] Алерт при > 80% квоты
