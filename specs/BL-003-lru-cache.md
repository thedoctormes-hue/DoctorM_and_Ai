# BL-002: Реализовать LRU cache для частых запросов

## Контекст
Частые запросы к одному и тому же ресурсу (например, projects.json, myrmex.json) каждый раз читают файл с диска. При росте числа агентов это создаёт нагрузку.

## Цель
Добавить LRU cache с TTL для часто читаемых файлов.

## Зачем
Снизить I/O нагрузку и ускорить ответы агентов на 40-60%.

## Проект/контекст
муравейник/инфраструктура

## Что сделать
- [ ] Реализовать cache.sh с функциями get/set/ttl/invalidate
- [ ] Добавить LRU eviction при превышении лимита записей (100)
- [ ] Интегрировать в session_startup.sh — кэшировать projects.json
- [ ] Добавить метрики cache_hits / cache_misses
- [ ] Протестировать при параллельных запросах

## Критерии готовности
- [ ] Повторные чтения не идут на диск
- [ ] TTL корректно инвалидирует старые записи
- [ ] LRU выталкивает при > 100 записях
- [ ] Метрики доступны через cache_stats
