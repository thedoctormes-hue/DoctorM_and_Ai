# BL-001: Реализовать confidence фильтр для инсайтов

## Контекст
Инсайты попадают в очередь с разным качеством. Нет фильтрации по уверенности — мусорные инсайты засоряют граф и бэклог эволюции.

## Цель
Добавить порог confidence для инсайтов, ниже которого они отбрасываются.

## Зачем
Чтобы memory не засорялась мусором и evolve не тратил ресурсы на шум.

## Проект/контекст
муравейник/инсайты

## Что сделать
- [ ] Добавить поле confidence (0-1) в структуру инсайта
- [ ] Реализовать фильтр в insight_catcher.sh — отбрасывать < 0.4
- [ ] Добавить логирование отброшенных инсайтов в insights_rejected.json
- [ ] Обновить insight_push.sh для приёма confidence
- [ ] Протестировать на реальных данных

## Критерии готовности
- [ ] Инсайты с confidence < 0.4 не попадают в очередь
- [ ] Отброшенные логируются
- [ ] Существующие инсайты без confidence получают default 0.5
