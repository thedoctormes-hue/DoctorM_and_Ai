# BL-049: Глубокое изучение Playwright для браузерной автоматизации

## Контекст
Playwright — ключевой инструмент для автоматизации браузерных задач (публикация на vc.ru, парсинг, тестирование). Текущее использование поверхностное — базовая навигация и скриншоты. Нужно систематизировать знания и освоить продвинутые фичи.

## Цель
Сформировать полное понимание возможностей Playwright и создать reusable-паттерны для типовых задач лаборатории.

## Зачем
- Автоматизация публикации контента (vc.ru, другие платформы)
- Парсинг данных с JS-зависимых сайтов
- E2E тестирование WebApp
- Качество выше чем у Puppeteer, лучше интеграция с Python

## Проект/контекст
bestia — агент для автоматизации контента. Паттерны будут переиспользоваться в других проектах.

## Примечания
### Найденные проблемы (из текущей сессии)
1. **Куки из файла могут быть невалидными** — даже если `auth/refresh` возвращает новый токен, API может требовать дополнительные куки или localStorage
2. **`page.evaluate` не решает проблему авторизации** — браузер отправляет те же куки, что и при прямом fetch
3. **Модальные оверлеи блокируют клики** — нужен `force=True` или закрытие оверлея
4. **`text=Войти` не работает в `document.querySelector`** — это Playwright-специфичный селектор, в DOM использовать CSS/XPath
5. **vc.ru использует localStorage для токенов** — `auth-refresh-token` в localStorage должен быть заполнен для работы API
