# BL-005: Реализовать dead letter queue для проваленных инсайтов

## Контект
Инсайты с низким confidence просто отбрасываются. Нет возможности пересмотреть их позже — потенциально ценные инсайты теряются навсегда.

## Цель
Создать dead letter queue (DLQ) для проваленных инсайтов с TTL и retry.

## Зачем
Не терять потенциально полезные инсайты, давать им второй шанс через N часов.

## Проект/контекст
муравейник/инсайты

## Что сделать
- [ ] Создать dlq.json в формате {insight, reason, failed_at, retry_at}
- [ ] Модифицировать insight_catcher.sh — проваленные инсайты → DLQ
- [ ] Реализовать dlq_processor.sh — переодическая проверка retry_at
- [ ] Добавить retry логику с повышением confidence на 0.1
- [ ] Автоматическая очистка записей старше 72ч

## Критерии готовности
- [ ] Проваленные инсайты попадают в DLQ, а не теряются
- [ ] Retry работает с exponential backoff
- [ ] TTL 72ч — старые записи удаляются
- [ ] DLQ мониторится через health check

## Зависимости
- BL-001 — confidence фильтр (DLQ потребляет его выход)

## Назначение
- **Вес:** 3
- **Скиллы:** cascade-brainstorm, council
- **Статус:** pending
- **Приоритет:** medium

## Примечания
Retry не более 3 раз. После 3 провалов — автоудаление из DLQ.
