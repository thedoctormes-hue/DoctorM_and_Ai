# BL-010: Реализовать валидацию спецификаций при создании

## Контекст
При создании спецификаций нет валидации — можно создать спеку без обязательных полей (Зачем, Проект/контекст, Вес). Это ломает sync_specs.sh и приводит к битым задачам в myrmex.json.

## Цель
Реализовать validate_spec.sh — проверку обязательных полей и формата.

## Зачем
Гарантировать целостность данных: все спеки валидны, sync не ломается.

## Проект/контекст
муравейник/бэклог

## Что сделать
- [ ] Создать validate_spec.sh — проверка обязательных полей
- [ ] Обязательные: Контекст, Цель, Зачем, Проект/контекст, Что сделать, Вес, Статус, Приоритет
- [ ] Проверка формата Вес (1-5), Статус (pending/in_progress/done/blocked), Приоритет (high/medium/low)
- [ ] Интегрировать в pre-commit hook
- [ ] Добавить validate_all.sh — валидация всех BL-*.md разом
- [ ] Вывод ошибок в человекочитаемом формате

## Критерии готовности
- [ ] validate_spec.sh находит отсутствующие поля
- [ ] Невалидная спека не проходит pre-commit
- [ ] validate_all.sh проверяет все спеки за < 1с
- [ ] Ошибки выводятся с указанием файла и строки

## Зависимости
- нет

## Назначение
- **Вес:** 3
- **Скиллы:** cascade-brainstorm
- **Статус:** pending
- **Приоритет:** high

## Примечания
Возвращать exit code 1 при ошибках — для pre-commit hook.
