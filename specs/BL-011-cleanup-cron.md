# BL-009: Реализовать cleanup cron job для временных файлов

## Контекст
В /tmp и рабочих директориях накапливаются временные файлы: .bak, session checkpoints, кэши без TTL, логи. Нет автоматической очистки — диск засоряется.

## Цель
Реализовать cleanup_cron.sh для регулярной очистки временных файлов.

## Зачем
Предотвратить засорение диска мусором и автоматически удалять протухший кэш.

## Проект/контекст
муравейник/инфраструктура

## Что сделать
- [ ] Создать cleanup_cron.sh с правилами очистки по паттернам
- [ ] Очистка /tmp/labdoctor-* старше 24ч
- [ ] Очистка .bak файлов старше 7д
- [ ] Очистка session checkpoints старше 3д
- [ ] Ротация логов — хранить последние 1000 строк
- [ ] Зарегистрировать в cron (ежедневно в 3:00)

## Критерии готовности
- [ ] cleanup_cron.sh удаляет файлы по правилам без ошибок
- [ ] Cron зарегистрирован и работает
- [ ] После очистки освобождается > 100MB (при наличии мусора)
- [ ] Лог cleanup записывается в labdoctor_cleanup.log

## Зависимости
- нет
