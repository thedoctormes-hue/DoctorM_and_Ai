### 2.3. Принципы создания артефактов

1. **Один артефакт = один файл = единственный источник правды**. Никаких дублей.
2. **Каждый артефакт содержит frontmatter**: name, description, version, category, last_updated.
3. **Каждый артефакт содержит минимум 1 конкретную метрику/цифру**.
4. **Каждый артефакт содержит CTA** (call to action) — куда вести читателя дальше.
5. **Каждый артефакт версионируется** — дата последнего обновления в frontmatter.
6. **Артефакты ссылаются друг на друга** — через относительные ссылки, а не дублируют контент.

```bash
#!/bin/bash
# generate_artifact.sh — Генерация артефакта из шаблона
# Использование: ./generate_artifact.sh <type> <name> [source_data]

TYPE=$1 # case_study | brand | marketing | content
NAME=$2 # имя файла (без расширения)
SOURCE=$3 # путь к исходным данным (опционально)
DATE=$(date +%Y-%m-%d)

case $TYPE in
case_study)
TEMPLATE="ARTIFACT_TEMPLATE.md"
OUTPUT="cases/CASE_STUDY_${NAME}.md"
;;
brand)
TEMPLATE="ARTIFACT_TEMPLATE.md"
OUTPUT="${NAME}.md"
;;
marketing)
TEMPLATE="ARTIFACT_TEMPLATE.md"
OUTPUT="marketing/${NAME}.md"
;;
*)
echo "Unknown type: $TYPE"
exit 1
;;
esac

# 1. Копируем шаблон
cp "$TEMPLATE" "$OUTPUT"

# 2. Заполняем frontmatter
sed -i "s/{{NAME}}/$NAME/g" "$OUTPUT"
sed -i "s/{{DATE}}/$DATE/g" "$OUTPUT"
sed -i "s/{{TYPE}}/$TYPE/g" "$OUTPUT"

# 3. Если есть исходные данные — вызываем AI-агента для заполнения
if [ -n "$SOURCE" ]; then
echo "Calling AI agent to fill artifact from $SOURCE..."
# Здесь вызов агента через QWEN API или локально
# agent fill-artifact --template "$TEMPLATE" --source "$SOURCE" --output "$OUTPUT"
fi

echo "✅ Artifact generated: $OUTPUT"
```

### 5.3. Скрипт обновления метрик

```bash
#!/bin/bash
# refresh_metrics.sh — Автоматическое обновление метрик в артефактах
# Запуск: cron, каждый день в 08:00

DATE=$(date +%Y-%m-%d)
ARTIFACTS_DIR="/root/LabDoctorM/evangelist"

# 1. Собираем свежие метрики
VPN_USERS=$(curl -s "http://localhost:3000/api/vpn/stats" | jq '.total_users')
VPN_UPTIME=$(curl -s "http://localhost:3000/api/vpn/stats" | jq '.uptime')
TG_SUBS=$(curl -s "https://api.tgstat.ru/channels/get?token=$TGSTAT_TOKEN&channelId=@DoctorMES" | jq '.subscribers')
HABR_VIEWS=$(curl -s "https://habr.com/ru/users/doctor-ai/articles/" | grep -oP 'stats__views[^<]*\K[0-9]+' | head -1)

# 2. Обновляем метрики в кейс-стади
for file in "$ARTIFACTS_DIR"/cases/*.md; do
# Обновляем поле last_updated
sed -i "s/last_updated: .*/last_updated: $DATE/" "$file"

# Обновляем метрики (если есть маркеры в файле)
if grep -q "{{VPN_USERS}}" "$file"; then
sed -i "s/{{VPN_USERS}}/$VPN_USERS/g" "$file"
fi
if grep -q "{{TG_SUBS}}" "$file"; then
sed -i "s/{{TG_SUBS}}/$TG_SUBS/g" "$file"
fi
done

# 3. Обновляем дашборд
cat > "$ARTIFACTS_DIR/system/metrics_snapshot.json" << EOF
{
"date": "$DATE",
"vpn_users": $VPN_USERS,
"vpn_uptime": $VPN_UPTIME,
"tg_subscribers": $TG_SUBS,
"habr_views": $HABR_VIEWS
}
EOF

# 4. Проверяем: если метрики изменились >10% — генерируем контент
python3 "$ARTIFACTS_DIR/system/check_metric_changes.py"

echo "✅ Metrics refreshed: $DATE"
```
