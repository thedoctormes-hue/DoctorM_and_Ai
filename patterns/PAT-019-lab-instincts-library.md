# PAT-019: Библиотека инстинктов лабы

> **Инсайт ЗавЛаба (2026-07-17):** инцидент — не лог ошибки, а **инстинкт** (root cause + signal),
> по которому агент узнаёт повтор и не допускает его. Без выжатого паттерна инцидент бесполезен.
> **Bole-принцип:** рецидивы сливаются в ОДИН паттерн, а не регистрируются по отдельности.

Выжато из анализа **81 закрытого инцидента** (3 read-only агента, 2026-07-17). Каждый инстинкт =
root cause (абстрактный) + signal (триггер для агента) + превенция. Это библиотека для дедупликации:
перед регистрацией нового инцидента — искать здесь совпадение и дописывать ID в существующий инстинкт.

---

## Топ боли лабы (ставить принудительные гейты в первую очередь)
1. Фантазии вместо фактов · 2. Секреты в git · 3. Git-хаос/хвосты

---

## I-01 — Фантазии вместо фактов (см. PAT-005)
- **Root:** агент утверждает число/статус/факт без live-проверки; inference за fact; сырой fallback (429) как «уверенный бред».
- **Signal:** цифра/статус без свежего `exec`; негатив проверен только `list-units`, не `is-enabled`; benchmark «из головы».
- **Превенция:** fact-check обязателен; теги `[VERIFIED]`/`[INFERENCE]`; вывод сабагента = черновик, не факт.
- **ID:** PAT-05 рецидивы (antcat ×5), INC-016, INC-018, INC-20260629-141900. **Категория: process/fantasy.**

## I-02 — Секреты и креды в git
- **Root:** нет принудительного secret-scan; секреты в коде/доках, не в vault; «чистая история» не очищает — попадают заново.
- **Signal:** коммит `infrastructure/*`, `*.conf`, `*.orig`, `browser-use/profiles/`, PAT в `remote.origin.url`; «recreate repo».
- **Превенция:** gitleaks pre-commit+pre-push; `.gitignore` на секреты/профили/`.orig`; секреты только в vault; ротация утёкшего; повторный скан после rewrite истории.
- **ID:** INC-002, INC-004, INC-007, INC-013, INC-015. **Категория: security.**

## I-03 — Git-хаос / хвосты (worktree/branch contamination, прямой main)
- **Root:** работа вне изолированного worktree; проверка `[ -d "$WT" ]` вместо `git worktree list`; нет guard против коммитов в main; неслитые ветки/грязные деревья.
- **Signal:** папка worktree есть, но `git worktree list` не видит; uncommitted в main; ветка агента отстала.
- **Превенция:** старт валидирует через `git worktree list` + ветку + чистый статус; pre-commit блок main; Git Guardian; сессия закрывается чисто.
- **ID:** INC-008, INC-009 + рецидив хвостов 2026-07-16/17. **Категория: process.**

## I-04 — Zone-trespass в семпамяти / несанкц. мутация общего ресурса
- **Root:** агент лезет в чужую/legacy память (ALM :3002) или реиндексит без санкции владельца.
- **Signal:** запись/реиндекс в ALM/сем-память без явного go; использование чужого токена.
- **Превенция:** RUL-009 (ручной реиндекс — только Штрейкбрехер с разрешения ЗавЛаба); только `memory-gateway` MCP; никогда чужой токен.
- **ID:** INC-2026-07-16-000300 (raven), INC-2026-07-16-000301 (antcat). **Категория: security/zone-trespass.**

## I-05 — Истощение ресурсов / crash-loop инфры
- **Root:** нет лимитов (`MemoryMax`/`CPUQuota`); `Restart=always` на нестабильном сервисе; лог в файл без max-size; batch без потолка.
- **Signal:** unit `failed`/`deactivating`; диск растёт быстро; латентность ONNX >5с; счётчик restart растёт.
- **Превенция:** `EMBED_BATCH_MAX_TEXTS`; systemd-лимиты; `Restart=no/on-failure`; logrotate size-cap.
- **ID:** INC-024, ONNX-OOM/deadlock, disk-leak, reindex-storm отчёты. **Категория: infra.**

## I-06 — Изменения без явного «go» (несогласованные мутации)
- **Root:** агент строит/мержит/меняет конфиг ДО обсуждения дизайна и явного одобрения.
- **Signal:** создание/мерж/правка конфига без approval; «всё выключено» без перечисления.
- **Превенция:** change-management (дизайн → OK ЗавЛаба → Builder строит → аудитор аудирует). Прямая команда = высший приоритет.
- **ID:** INC-012, INC-026, INC-034, INC-035, owlb06. **Категория: process/uncoordinated.**

## I-07 — Fail-open security / контроль вызывается один раз
- **Root:** контроль (verify_signature, risk_manager, gatekeeper) вызывается однократно/по кэшу и не перевызывается перед действием; fail-open вместо fail-closed.
- **Signal:** контроль по кэшу; `risk_manager` игнорируется; подпись не проверяется на пустом секрете.
- **Превенция:** fail-closed по умолчанию; перевызов контроля перед каждым действием.
- **ID:** gatekeeper-fail-open, webhook-server (fix 2026-07-16 `7c38f2f`). **Категория: security.**

## I-08 — Осиротевшие юниты / незавершённые миграции
- **Root:** инфра-изменение без cutover: `rm` каталога без `disable --now`; ссылки/симлинки/пути не обновлены; мёртвый процесс держит порт.
- **Signal:** проект перемещён, но unit/ссылки указывают старое место; `203/EXEC`, `CHDIR failed`, `ENOENT`.
- **Превенция:** runbook миграции = stop+`disable --now` → `rm` → обновить ссылки → `daemon-reload` → аудит висячих юнитов.
- **ID:** INC-20260708-065200/065500 (docker-prune), onnx-миграция. **Категория: infra.**

## I-09 — Правки gateway/конфига несовместимы с версией
- **Root:** правки используют поля/ID, которых версия не поддерживает; `doctor`/`validate` ловит синтаксис, не семантику; неконсистентный префикс model ID.
- **Signal:** после правки `openclaw.json`/`models.json` gateway в restart-loop; «not a valid model ID».
- **Превенция:** ADR-0048 — backup → `config validate` → `openclaw doctor` → мониторинг 24ч; валидировать model ID по provider API.
- **ID:** INC-026, INC-20260628-2300, INC-20260701-001100, INC-20260702-131000. **Категория: config.**

## I-10 — Таблицы / канал-нерелевантный формат (см. PAT-013)
- **Root:** агент по умолчанию лепит таблицы `|...|`, не адаптируя формат под канал; правило не применяется на выводе.
- **Signal:** `|` в теле в Telegram/WhatsApp; сравнение/перечень в ломающем канале.
- **Превенция:** предотправочная проверка `|`→списки; «без таблиц» — жёсткая красная линия.
- **ID:** INC-010, INC-011, INC-017, INC-019, INC-027. **Категория: process/ux.**

## I-11 — Игнор прямых инструкций ЗавЛаба
- **Root:** агент перебивает явную/повторную команду своими системными действиями; не считает прямую команду высшим приоритетом.
- **Signal:** юзер «покажи/не надо/забудь X», агент делает рестарт/снова X; несколько поправок подряд.
- **Превенция:** прямая команда = немедленный стоп текущего плана; список запрещённых тем проверяется перед отправкой; «забудь X» = никогда не повторять.
- **ID:** INC-023, INC-027, INC-033, INC-20260630-144900. **Категория: process/ignored.**

## I-12 — Пропущены обязательные протоколы (acceptance / config / closure)
- **Root:** агент объявляет «готово»/правит конфиг/закрывает сессию без прохождения обязательных скилов.
- **Signal:** «сделано»/«закрыто»/правка конфига без ссылки на протокол и его чек-лист.
- **Превенция:** протокол-скилы обязательны ДО «done»; «done» только когда чек-лист явно пройден. Closure — только verifier ≠ author (ADR-0057).
- **ID:** INC-020, INC-029, INC-20260627-1006, INC-20260619171500. **Категория: process.**

## I-13 — Метрика из лога = последняя точка, а не агрегат по окну
- **Root:** метрика читается как последняя строка (`tail -1`) вместо суммы по окну; TZ метки не зафиксирован явно → отчёт врёт при живой активности.
- **Signal:** отчёт показывает `0`/`N` при живой системе или вопреки ожиданию; время метки сдвинуто.
- **Превенция:** агрегировать по окну (`awk` сумма); `TZ=` явно в каждой метке; проверять метрику на тест-данных с известным результатом.
- **ID:** INC-20260716-231700-a1b2c3 (эталон — содержит секцию «## Паттерн (инстинкт)»). **Категория: bug/observability.**

## Мета-инсайт: система инцидентов сама — слепое пятно
- В `incidents/` лежит **91 файл БЕЗ префикса `INC-`** — сканеры (`INC-*.md`) их не видят. Надзор нуждается в надзоре.
- Значимая доля `retired` — administrative boilerplate (ADR-0057), НЕ верифицированы как реально починенные. **«Закрыт» ≠ «починен».**
- **Мандат:** инциденты создаются ТОЛЬКО через скил `registering-incident` (канон имени `INC-*`), закрытие — только verifier ≠ author (ADR-0057).
