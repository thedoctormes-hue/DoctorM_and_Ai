# PAT-019: Библиотека инстинктов лабы

> **Инсайт ЗавЛаба (2026-07-17):** инцидент — не лог ошибки, а **инстинкт** (root cause + signal),
> по которому агент узнаёт повтор и не допускает его. Без выжатого паттерна инцидент бесполезен.
> **Bole-принцип:** рецидивы сливаются в ОДИН паттерн, а не регистрируются по отдельности.

Выжато из анализа **81 закрытого инцидента** (3 read-only агента, 2026-07-17). Каждый инстинкт =
root cause (абстрактный) + signal (триггер для агента) + превенция. Это библиотека для дедупликации:
перед регистрацией нового инцидента — искать здесь совпадение и дописывать ID в существующий инстинкт.

## Топ боли лабы (ставить принудительные гейты в первую очередь)
1. Фантазии вместо фактов · 2. Секреты в git · 3. Git-хаос/хвосты

## I-01 — Фантазии вместо фактов (см. PAT-005)
- **Root:** агент утверждает число/статус/факт без live-проверки; inference за fact; сырой fallback (429) как «уверенный бред».
- **Signal:** цифра/статус без свежего `exec`; негатив проверен только `list-units`, не `is-enabled`; benchmark «из головы».
- **Превенция:** fact-check обязателен; теги `[VERIFIED]`/`[INFERENCE]`; вывод сабагента = черновик, не факт.
- **ID:** PAT-05 рецидивы (antcat ×5), INC-016, INC-018, INC-20260629-141900, INC-20260718-220900-72aa8c (antcat: 5 фантазий в плане апгрейда OpenClaw — GPT-5.6, SQLite backup, Cron fixes, SIGUSR1, Cron model selection — все выдуманы без проверки CHANGELOG). **Категория: process/fantasy.**
- **Confidence:** high
- **Scope:** global

## I-02 — Секреты и креды в git
- **Root:** нет принудительного secret-scan; секреты в коде/доках, не в vault; «чистая история» не очищает — попадают заново.
- **Signal:** коммит `infrastructure/*`, `*.conf`, `*.orig`, `browser-use/profiles/`, PAT в `remote.origin.url`; «recreate repo».
- **Превенция:** gitleaks pre-commit+pre-push; `.gitignore` на секреты/профили/`.orig`; секреты только в vault; ротация утёкшего; повторный скан после rewrite истории.
- **ID:** INC-002, INC-004, INC-007, INC-013, INC-015. **Категория: security.**
- **Confidence:** high
- **Scope:** global

## I-03 — Git-хаос / хвосты (worktree/branch contamination, прямой main)
- **Root:** работа вне изолированного worktree; проверка `[ -d "$WT" ]` вместо `git worktree list`; нет guard против коммитов в main; неслитые ветки/грязные деревья.
- **Signal:** папка worktree есть, но `git worktree list` не видит; uncommitted в main; ветка агента отстала.
- **Превенция:** старт валидирует через `git worktree list` + ветку + чистый статус; Git Guardian; сессия закрывается чисто; **внедрён `bin/check-worktree-isolation.sh`** (вызывается из `git-hooks/pre-commit` и `bin/lab-commit.sh`) — принудительно блокирует коммит в главный checkout `/root/LabDoctorM/projects/DoctorM_and_Ai`, требует изолированный worktree; bypass для инфры/ЗавЛаба: `LAB_ALLOW_MAIN_COMMIT=1`. **Добавлен `bin/git-add-guard.sh`** (B8.1, 2026-07-19) — блокирует `git add` в главном checkout (git не имеет native pre-stage hook, поэтому stage запрещён через wrapper); агенты обязаны использовать `bin/git-add-guard.sh <файлы>` вместо `git add`; тот же bypass `LAB_ALLOW_MAIN_COMMIT=1`. **Закрывает `INC-20260718-133000-9d4e2a`.**
- **ID:** INC-008, INC-009 + рецидив хвостов 2026-07-16/17, **INC-20260718-101833-c4f1d4** (рецидив: antcat съел незакомиченный WIP owl при работе в общем checkout). **Категория: process.**, **INC-2026-07-18-193421-bb9733** (рецидив: coordinator not documenting tails at closing)
- **Confidence:** high
- **Scope:** global

## I-04 — Zone-trespass в семпамяти / несанкц. мутация общего ресурса
- **Root:** агент лезет в чужую/legacy память (ALM :3002) или реиндексит без санкции владельца.
- **Signal:** запись/реиндекс в ALM/сем-память без явного go; использование чужого токена.
- **Превенция:** RUL-009 (ручной реиндекс — только Штрейкбрехер с разрешения ЗавЛаба); только `memory-gateway` MCP; никогда чужой токен.
- **ID:** INC-2026-07-16-000300 (raven), INC-2026-07-16-000301 (antcat). **Категория: security/zone-trespass.**
- **Confidence:** medium
- **Scope:** global

## I-05 — Истощение ресурсов / crash-loop инфры
- **Root:** нет лимитов (`MemoryMax`/`CPUQuota`); `Restart=always` на нестабильном сервисе; лог в файл без max-size; batch без потолка.
- **Signal:** unit `failed`/`deactivating`; диск растёт быстро; латентность ONNX >5с; счётчик restart растёт.
- **Превенция:** `EMBED_BATCH_MAX_TEXTS`; systemd-лимиты; `Restart=no/on-failure`; logrotate size-cap.
- **ID:** INC-024, ONNX-OOM/deadlock, disk-leak, reindex-storm отчёты. **Категория: infra.**
- **Confidence:** medium
- **Scope:** global

## I-06 — Изменения без явного «go» (несогласованные мутации)
- **Root:** агент строит/мержит/меняет конфиг ДО обсуждения дизайна и явного одобрения.
- **Signal:** создание/мерж/правка конфига без approval; «всё выключено» без перечисления.
- **Превенция:** change-management (дизайн → OK ЗавЛаба → Builder строит → аудитор аудирует). Прямая команда = высший приоритет.
- **ID:** INC-012, INC-026, INC-034, INC-035, owlb06, INC-2026-07-18-112738-1ed23b (raven: создал проект `raven-patrol` + папку `projects/raven-patrol/` и записал туда код, ответив на вопрос ЗавЛаба «в какой папке?» ДЕЙСТВИЕМ без явного «go»). **Категория: process/uncoordinated.**
- **Confidence:** high
- **Scope:** global

## I-07 — Fail-open security / контроль вызывается один раз
- **Root:** контроль (verify_signature, risk_manager, gatekeeper) вызывается однократно/по кэшу и не перевызывается перед действием; fail-open вместо fail-closed.
- **Signal:** контроль по кэшу; `risk_manager` игнорируется; подпись не проверяется на пустом секрете.
- **Превенция:** fail-closed по умолчанию; перевызов контроля перед каждым действием.
- **ID:** gatekeeper-fail-open, webhook-server (fix 2026-07-16 `7c38f2f`). **Категория: security.**
- **Confidence:** medium
- **Scope:** global

## I-08 — Осиротевшие юниты / незавершённые миграции
- **Root:** инфра-изменение без cutover: `rm` каталога без `disable --now`; ссылки/симлинки/пути не обновлены; мёртвый процесс держит порт.
- **Signal:** проект перемещён, но unit/ссылки указывают старое место; `203/EXEC`, `CHDIR failed`, `ENOENT`.
- **Превенция:** runbook миграции = stop+`disable --now` → `rm` → обновить ссылки → `daemon-reload` → аудит висячих юнитов.
- **ID:** INC-20260708-065200/065500 (docker-prune), onnx-миграция. **Категория: infra.**
- **Confidence:** medium
- **Scope:** global

## I-09 — Правки gateway/конфига несовместимы с версией
- **Root:** правки используют поля/ID, которых версия не поддерживает; `doctor`/`validate` ловит синтаксис, не семантику; неконсистентный префикс model ID.
- **Signal:** после правки `openclaw.json`/`models.json` gateway в restart-loop; «not a valid model ID».
- **Превенция:** ADR-0048 — backup → `config validate` → `openclaw doctor` → мониторинг 24ч; валидировать model ID по provider API.
- **ID:** INC-026, INC-20260628-2300, INC-20260701-001100, INC-20260702-131000. **Категория: config.**
- **Confidence:** high
- **Scope:** global

## I-10 — Таблицы / канал-нерелевантный формат (см. PAT-013)
- **Root:** агент по умолчанию лепит таблицы `|...|`, не адаптируя формат под канал; правило не применяется на выводе.
- **Signal:** `|` в теле в Telegram/WhatsApp; сравнение/перечень в ломающем канале.
- **Превенция:** предотправочная проверка `|`→списки; «без таблиц» — жёсткая красная линия.
- **ID:** INC-010, INC-011, INC-017, INC-019, INC-027. **Категория: process/ux.**
- **Confidence:** high
- **Scope:** global

## I-11 — Игнор прямых инструкций ЗавЛаба
- **Root:** агент перебивает явную/повторную команду своими системными действиями; не считает прямую команду высшим приоритетом.
- **Signal:** юзер «покажи/не надо/забудь X», агент делает рестарт/снова X; несколько поправок подряд.
- **Превенция:** прямая команда = немедленный стоп текущего плана; список запрещённых тем проверяется перед отправкой; «забудь X» = никогда не повторять.
- **ID:** INC-023, INC-027, INC-033, INC-20260630-144900. **Категория: process/ignored.**
- **Confidence:** medium
- **Scope:** global

## I-12 — Пропущены обязательные протоколы (acceptance / config / closure)
- **Root:** агент объявляет «готово»/правит конфиг/закрывает сессию без прохождения обязательных скилов.
- **Signal:** «сделано»/«закрыто»/правка конфига без ссылки на протокол и его чек-лист.
- **Превенция:** протокол-скилы обязательны ДО «done»; «done» только когда чек-лист явно пройден. Closure — только verifier ≠ author (ADR-0057).
- **ID:** INC-020, INC-029, INC-20260627-1006, INC-20260619171500, INC-20260718-115655-ae5b78 (owl: нарушение порядка registering-incident). **Категория: process.**
- **Confidence:** high
- **Scope:** global

## I-13 — Метрика из лога = последняя точка, а не агрегат по окну
- **Root:** метрика читается как последняя строка (`tail -1`) вместо суммы по окну; TZ метки не зафиксирован явно → отчёт врёт при живой активности.
- **Signal:** отчёт показывает `0`/`N` при живой системе или вопреки ожиданию; время метки сдвинуто.
- **Превенция:** агрегировать по окну (`awk` сумма); `TZ=` явно в каждой метке; проверять метрику на тест-данных с известным результатом.
- **ID:** INC-20260716-231700-a1b2c3 (эталон — содержит секцию «## Паттерн (инстинкт)»). **Категория: bug/observability.**
- **Confidence:** medium
- **Scope:** global

## I-14 — Грязное дерево активной работы агента ≠ хвост (ложноположительный алерт монитора)
- **Root:** монитор хвостов трактует ЛЮБОЕ грязное дерево (M/?? файлы) как аномалию, не различая «агент активно работает над проектом» vs «забытый/осиротевший хвост». Рабочий WIP выглядит как грязная ветка.
- **Signal:** грязное дерево в репо, где агент сейчас/недавно работал (последний commit/WIP свежий, задача ещё идёт) → это нормальный рабочий процесс, НЕ хвост. Пример: `ANALYSIS_2026-07-17_ddp.md` Бестии — DDP-отчёт в процессе, а не забытый хвост.
- **Превенция:** сканер должен учитывать контекст активности (владелец репо работает? задача активна? изменения свежее порога?) и маркировать «рабочий WIP» отдельно от «stale-хвоста» (по времени бездействия); не алертить на свежий WIP. Аудитор классифицирует: активная работа → не докладывать как проблему.
- **ID:** наблюдение 2026-07-17 (Сова ошибочно flagged ANALYSIS-файл Бестии как хвост). **Категория: process/observability.**
- **Confidence:** low
- **Scope:** global

## I-15 — git rm auto-generated файла ломает зависимые systemd-сервисы (тихая деградация)
- **Root:** `git rm` (не `--cached`) удаляет файл с диска; если от него зависят systemd-сервисы (через `.env`, скрипты бэкапа, runtime-state), они либо FAILED, либо уходят в тихий fallback (работают, но данные не грузятся) — деградация незаметна до проверки.
- **Signal:** планируется `git rm` любого файла в инфра-репо, от которого что-то зависит; после `git rm` сервис ведёт себя «странно» (FAILED или тихий fallback).
- **Превенция:** перед `git rm` — `grep -r` по `.service`/`.env`/скриптам на зависимость; использовать `git rm --cached` (оставить на диске) вместо `git rm`; после ЛЮБОЙ git-мутации в инфра-репо — `systemctl status` зависимых сервисов.
- **ID:** INC-20260717-0917 (myrmex.json удалён → backup-myrmex FAILED + myrmex-control тихий fallback). **Категория: process/infra.**
- **Confidence:** medium
- **Scope:** project

## I-16 — Запуск systemd-сервиса без сборки dist → crash-loop
- **Root:** сервис (`ExecStart=node dist/server/index.js`) запускается до сборки артефакта; `dist/` отсутствует (никогда не коммитился, собирается локально через `npm run build`) → node падает с exit 1 в цикле.
- **Signal:** `systemctl start` сервиса, чей ExecStart ссылается на `dist/...`; `dist/` нет в репо/на диске; сервис в `activating (auto-restart)` с `exit-code 1`.
- **Превенция:** перед `systemctl start` — `test -f dist/server/index.js`; если нет — `npm run build:server` (+`build:client`); build-шаг должен входить в deploy-runbook.
- **ID:** наблюдение 2026-07-17 (myrmex-control crash-loop 516 до build). **Категория: process/deploy.**
- **Confidence:** medium
- **Scope:** project

## I-17 — Умный монитор наблюдает и делегирует (не копать руками)
- **Root:** агент-разведчик (Ворон) выполняет рутинный сбор/исправление САМ в основном чате вместо делегирования субагентам; держит «сырой объём» в контексте; не использует `sessions_spawn` для параллельного сбора.
- **Signal:** агент делает exec/web_search/правки сам, когда задача — мониторинг/сбор данных/исправление инфры; субагенты не задействованы; контекст забит raw-выводом.
- **Превенция:** монитор (Ворон) = наблюдает (проверки) + делегирует сбор/исполнение субагентам (`sessions_spawn`, agentId=raven); сам только анализирует результат и принимает решения; сырой объём не держать в чате (патруль-дизайн: субагент --collect, Ворон --digest-send).
- **ID:** указание ЗавЛаба 2026-07-17 10:49 МСК («Ты умный монитор! Ты наблюдаешь! Для работы у тебя есть субагенты! Умный монитор наблюдает и делигирует!»). **Категория: process/observability.**
- **Confidence:** medium
- **Scope:** global

## I-18 — Обход gatekeeper shim без регистрации порта + подъём на reserve-порту
- **Root:** агент поднимает сервис через `systemctl.real` (обход shim) для «быстрого фикса» и не регистрирует порт в gatekeeper; либо поднимает на порту из `blocked_ports` policy (reserved), который gatekeeper REJECT при регистрации.
- **Signal:** GK-AUDIT алерт «unauthorized listening socket» (порт слушает, но нет lease); порт в `blocked_ports` (напр. 3000) слушает сервисом; любой ресурс поднят через `systemctl.real` без последующей `gatekeeper__register_port`.
- **Превенция:** при обходе shim (systemctl.real / kill-switch) — сразу `gatekeeper__register_port` (с пометкой bypass); НЕ поднимать сервисы на портах из `blocked_ports` (выбирать свободный, напр. 3001); gatekeeper должен быть persistent (ADR-0058), иначе lease истекают через 300с и дедуп висит.
- **ID:** INC-20260717-101833-348a9a (mcp-lab-research :8089 через systemctl.real без регистрации; myrmex-control :3000 reserved); INC-2026-07-18-004130-7d2f4a (bare nginx exec минует shim → orphan держит :443, systemd-юнит failed). **Категория: process/security.**
- **Confidence:** high
- **Scope:** project


## I-19 — Вывод о статусе инцидента по памяти, не по реестру (doc-drift)
- **Root:** агент цитирует статус инцидента из прозы (MEMORY.md, бэклог, IDENTITY, AGENTS), где статус устарел; реестр `incidents/*.md` (SSOT, ADR-0056) рассинхронизирован с прозой. Вывод «инцидент open/блокер» делается из доки, а не из реестра.
- **Signal:** в ответе/памяти инцидент помечен `open`/`жду`/`блокер`/`confirm`, а в `incidents/` он уже `resolved`/`closed`/`retired`; либо ссылка `INC-*` вообще отсутствует в реестре (мёртвая ссылка).
- **Превенция:** перед любым выводом о статусе инцидента — свериться с реестром (`incidents/*.md`), не с памятью/прозой. Рутинно прогонять `bin/incident-drift-check.sh` (ADR-0047-pattern): он парсит реестр как SSOT и флагает DRIFT/NOT_IN_REGISTRY по всем workspace-докам. «Открыт» = только если реестр подтверждает `open`/`investigating`/`unknown` (не retired/closed/resolved и не отсутствует).
- **ID:** наблюдение 2026-07-17 — сканер поймал реальный DRIFT: `owl/MEMORY.md:306` (`INC-20260713-180132-81a13b`) заявлял `(open)`, реестр = `retired`; плюс 2 мёртвые ссылки в `antcat/*`. **Категория: process/drift.**
- **Confidence:** medium
- **Scope:** global

## Мета-инсайт: система инцидентов сама — слепое пятно
- В `incidents/` лежит **91 файл БЕЗ префикса `INC-`** — сканеры (`INC-*.md`) их не видят. Надзор нуждается в надзоре.
- Значимая доля `retired` — administrative boilerplate (ADR-0057), НЕ верифицированы как реально починенные. **«Закрыт» ≠ «починен».**
- **Мандат:** инциденты создаются ТОЛЬКО через скил `registering-incident` (канон имени `INC-*`), закрытие — только verifier ≠ author (ADR-0057).
