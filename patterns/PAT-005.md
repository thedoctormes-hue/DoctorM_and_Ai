# PAT-005: Не выдавай непроверенные данные за факты

**Тип:** Жёлтая карточка (предупреждение)
**Создано:** 2026-06-16
**Автор инцидента:** owl (Сова)
**Связанный инцидент:** INC-018


## Правило

Прежде чем назвать число, факт или статус — проверь источник напрямую.

- Если можно прочитать конфиг — прочитай конфиг
- Если можно выполнить команду — выполни команду
- Если можно открыть файл — открой файл
- **НЕ додумывай по косвенным признакам** (список файлов, примеры, похожие случаи)

## Формулировка

Если ЗавЛаб спрашивает «сколько?» или «кто?» — ответ должен быть из первичного источника, а не из моей головы.

## Пример нарушения

`find` нашёл файлы → визуально посчитала → назвала число → число оказалось неверным → пересчитала → снова неверно → только потом открыла конфиг.

## Правильный подход

Увидела задачу → определила первичный источник (openclaw.json) → открыла → прочитала → назвала точное число.
