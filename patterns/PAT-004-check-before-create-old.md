# PAT-004: Сначала проверь — потом делай

## Проблема
Агенты создают дублирующие инструменты вместо использования существующих. Случаи:
- Бестия написала `monitor.py` (клон `lab-monitoring`)
- Ворон написал `patrol.py` (клон `stack_patrol.py`) + `publish_insights.py` (дубликат Myrmex API)

## Правило
**Перед реализацией любого нового функционала агент ОБЯЗАН:**
1. Выполнить `grep_search` / `glob` по всей лаборатории (`/root/LabDoctorM/projects/`)
2. Проверить `PROJECT.md` смежных проектов
3. Если функционал уже существует — использовать его или расширять
4. Если нужна новая функциональность — обосновать почему существующее не подходит

## Критерии «можно создавать новое»
- Существующий инструмент не покрывает требуемый use case
- Существующий инструмент находится в другой зоне ответственности и недоступен через API
- Создание дубликата целесообразно для изоляции (обосновать)

## Запрещено
- Писать код аналогичного назначения без проверки существующих решений
- Дублировать мониторинг, API-клиенты, сканеры, репортеры
- Хранить «личные» копии общих инструментов
