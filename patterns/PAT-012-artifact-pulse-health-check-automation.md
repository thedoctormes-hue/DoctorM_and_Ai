# PAT-012: Автоматизация проверки здоровья артефактов через artifact-pulse

## Контекст
Лаборатория генерирует большое количество артефактов (ADR, patterns, rules, specs, incidents, metrics). Ручное отслеживание их качества и актуальности невозможно. Существует система `artifact-pulse`, которая:
- Сканирует все markdown-файлы артефактов
- Проверяет frontmatter, ссылки, структуру
- Выдает Overall Health Score (0-100)

В текущем workflow:
- Запуск `artifact-pulse` производится sporadically вручную
- Нет автоматического оповещения при ухудшении здоровья артефактов
- Нет интеграции с Myrmex для отображения статуса артефактов
- Риск накопления технического долга в документации незамеченным

## Решение
Автоматизировать проверку здоровья артефактов с пороговыми алертами:

### 1. Регулярный запуск
- Добавить системный таймер/крисн-job для еженедельного запуска:
  ```bash
  bash /root/LabDoctorM/.qwen/scripts/artifact_health.sh --weekly
  ```
- Скрипт должен:
  - Запускать полную проверку всех артефактов
  - Сохранять результаты в `/root/LabDoctorM/.qwen/artifacts/health_report_$(date +%Y%W).json`
  - Сравнивать с предыдущим запуском (если есть)

### 2. Пороговое оповещение
- Если Overall Health Score < 80 → создавать инцидент в Myrmex:
  - Тип: `incident`
  - Источник: `artifact-pulse`
  - Описание: "Health score dropped to {score}/100. Check details in health report."
  - Severity: `warning` (если 60-79) или `error` (если <60)
- Если Health Score улучшился на ≥10 пунктов → создавать заметку в системе инсайтов типа `improvement`

### 3. Интеграция с Myrmex
- Добавить в каждую сущность в `myrmex.json` (проекты и agent_zones) поле:
  ```json
  "healthScore": { "value": 85, "updatedAt": "2026-06-18T06:00:00Z" }
  ```
- Обновлять поле после каждого запуска artifact-pulse для соответствующих артефактов
- Поле является опциональным и заполняется только для артефактов, связанных с конкретной сущностью

### 4. Дашборд и отчетность
- Еженедельно генерировать дайджест изменений:
  - Новые артефакты (+X)
  - Улучшенные артефакты (+Y баллов)
  - Ухудшенные артефакты (-Z баллов, требует внимания)
  - Рассылать в канал #lab-digest или аналогичный

## Последствия
### ✅ Позитивные последствия
- Проактивное выявление проблем в документации до того, как они станут критическими
- Снижение времени на ручные проверки и аудиты
- Историческая динамика здоровья артефактов для планирования работ
- Четкая обратная связь для авторов артефактов
- Интеграция с существующими системами (Myrmex, incident management)

### ⚠️ Негативные последствия / компромиссы
- Небольшая нагрузка на систему при еженедельных запускаs (~1-2 минуты)
- Требуется поддержка и обновление скриптов artifact_health.sh
- Нужно обучать агентов новым полям в myrmex.json
- Риск ложных срабатываний при временных проблемах (сетевые, загрузка)

## Альтернативы
| Вариант | Плюсы | Минусы | Почему отклонён |
|---------|-------|--------|-----------------|
| Ручное еженедельное напоминание в календаре | Простота, нет изменений в коде | Забываемость, отсутствие исторических данных, нет интеграции | Не достигает цели проактивного выявления |
| Запуск после каждого коммита в репозиторий лаборатории | Мгновенная обратная связь | Избыточная нагрузка (много коммитов в день), сложность триггеров | Чрезмерно частый, не нужен для документирования |
| Использование внешних сервисов (например, SonarQube для documentation) | Глубокий анализ, готовые интеграции | Стоимость, сложность настройки, внешняя зависимость | Перебор для текущих потребностей, lab-specific решения предпочтительнее |
| Интеграция с системой алертов лаборатории (Prometheus/Alertmanager) | Использует существующую инфраструктуру | Требует экспортера метрик, сложность настройки | artifact-pulse выдает сложный JSON, проще сделать прямой вызов в Myrmex |
