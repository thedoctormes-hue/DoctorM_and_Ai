# PAT-002: Работа с границами секций при редактировании кода

## Категория
coding

## Контекст
При редактировании больших файлов (>200 строк) с несколькими секциями/функциями. Риск захватить лишний код при удалении или изменении.

## Решение
1. Перед редактированием — определить границы секции (начало/конец, строки)
2. Проверить что соседние секции не затронуты
3. Использовать маркеры секций (`// === SECTION NAME ===`)
4. После редактирования — запустить тесты

## Примеры
```javascript
// === SESSION START ENDPOINT ===
app.get('/api/kanban/session-start', ...);
// === END SESSION START ===

// === MONITORING ENDPOINT ===
app.get('/api/monitoring/servers', ...);
// === END MONITORING ===
```

## Критерии применимости
- [ ] Файл >200 строк
- [ ] В файле несколько независимых секций
- [ ] Редактирование затрагивает одну секцию

## Примечание
Инцидент: при удалении decompose endpoint был захвачен блок 288 строк вместо одной секции. Потеря: ~30 минут на восстановление.
