# 300 Vision — анализ изображений через Cohere Vision

Использует Cohere `command-a-plus-05-2026` (436K context, vision) для анализа изображений через `/v2/chat` API.

## Возможности

- **OCR** — распознавание текста со скриншотов, документов, фото
- **Описание** — подробное описание изображений
- **Диаграммы** — анализ графиков, схем, диаграмм
- **Код** — чтение и анализ кода со скриншотов

## Workflow

1. Получить изображение от пользователя (путь к файлу или URL)
2. Если URL — скачать: `curl -sL -o /tmp/vision_input.jpg "<url>"`
3. Запустить скрипт: `{baseDir}/scripts/vision.sh <image_path> [--prompt "..."] [--lang ru|en|auto]`
4. Скрипт вернёт результат и обновит счётчик
5. Сообщить результат и статус счётчика

## Команды

```bash
# Анализ изображения (автоопределение языка)
{baseDir}/scripts/vision.sh /path/to/image.jpg

# Анализ на русском
{baseDir}/scripts/vision.sh /path/to/image.jpg --lang ru

# Свой промпт
{baseDir}/scripts/vision.sh /path/to/image.jpg --prompt "Какая версия Python на скриншоте?"

# Статус счётчика
{baseDir}/scripts/vision.sh --status

# Сброс счётчика (только для админа)
{baseDir}/scripts/vision.sh --reset
```

## Счётчик и ротация

- **Начальное значение:** 300 вызовов (с запасом от реальных ~400)
- **После каждого вызова:** `-1`
- **Формат ответа:** `"✅ Анализ выполнен. -1 vision cohere, осталось N vision"`
- **При N=0:** `"❌ Лимит 300 Vision исчерпан. Все 4 ключа Cohere исчерпаны."`
- **Ротация:** key1 → key2 → key3 → key4 → key1 (замкнутый fallback)
- **При ошибке (429/401/пустой ответ):** переход на следующий ключ, счётчик НЕ уменьшается

## Endpoint

- URL: `https://api.cohere.com/v2/chat`
- Method: POST
- Headers: `Authorization: Bearer <key>`, `Content-Type: application/json`

## Формат запроса

```json
{
  "model": "command-a-plus-05-2026",
  "messages": [{"role": "user", "content": [
    {"type": "text", "text": "<промпт>"},
    {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,<base64_data>"}}
  ]}],
  "max_tokens": 2000
}
```

## Извлечение результата

Ответ в `message.content[?].text` где `type == "text"`.

## Обработка ошибок

Скрипт `vision.sh` автоматически классифицирует HTTP-ошибки и реагирует:

### 401 Unauthorized
- Ключ невалидный или отозван
- Действие: перейти на следующий ключ в ротации, счётчик НЕ уменьшить
- Лог: `[WARN] Key N returned 401, rotating`

### 429 Too Many Requests
- Rate limit для текущего ключа
- Действие: экспоненциальная задержка (2с → 4с → 8с), затем переход на следующий ключ
- Счётчик НЕ уменьшается
- Лог: `[WARN] Key N rate limited (429), backing off`

### 5xx Server Error
- Внутренняя ошибка Cohere
- Действие: переход на следующий ключ, счётчик НЕ уменьшается
- Лог: `[ERROR] Key N returned 5xx, rotating`

### Fallback: все ключи исчерпаны

Если все 4 ключа вернули ошибку (401/429/5xx):
1. Скрипт останавливает ротацию
2. Выводит сообщение:
```
❌ Все 4 ключа Cohere исчерпаны.
Последняя ошибка: [описание]
Проверьте ключи в конфигурации.
Статус счётчика: осталось N vision (не израсходованы).
```
3. Счётчик сохраняет текущее значение — при следующем вызове с рабочим ключом анализ снова доступен
4. НЕ сбрасывать счётчик автоматически

## Ограничения

- 1 вызов Cohere за одно изображение
- Скорость: ~1-3 секунды на изображение
- Максимальный размер: ~10MB (base64)
- Не подходит для видео

## Файлы

- `scripts/vision.sh` — основной скрипт (анализ + счётчик + ротация, inline Python для генерации тела запроса)
- `references/vision-state.json` — состояние счётчика (создаётся автоматически)
