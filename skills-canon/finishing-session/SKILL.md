# Session End — Завершение сессии

## Назначение

Скил запускается при завершении сессии. Фиксирует итоги работы, обновляет память, проверяет инциденты и хвосты. Цель — каждая сессия оставляет чистый след в истории.

> **База:** правила закрытия сессии определены в AGENTS.md (раздел «Закрытие сессии»). Этот скил — расширение, а не дубликат.

## Триггеры

- «закрывай сессию»
- «сессия завершена»
- «всё на сегодня»
- «завершить сессию»

## Обязательные шаги

### 1. Записать итоги

Следуй правилам закрытия сессии из AGENTS.md (раздел «Закрытие сессии»).

Дополнительно:
- Если `memory/YYYY-MM-DD.md` уже существует — ДОПИСАТЬ, не перезаписывать
- `mkdir -p memory/` если не существует
- НЕ писать если нечего писать (AGENTS.md: never empty placeholders)

### 1.5 Pre-compaction flush

Если сессия >50 сообщений или была компакция:
- Записать ключевые решения и контекст в `memory/YYYY-MM-DD.md`
- Это гарантирует сохранение важного даже если compaction сожрёт историю

### 2. Обновить долгосрочную память

Следуй AGENTS.md (MEMORY.md — раздел «Memory»).

Дополнительно:
- `MEMORY.md` обновляется только если есть реальные изменения
- Устаревшая информация — убрать

### 3. Проверить инциденты

- Решённые → пометить `status: closed`
- Новые проблемы → создать `/root/LabDoctorM/projects/DoctorM_and_Ai/incidents/INC-XXX.md`
- Открытые → актуализировать описание
- НЕ закрывать инциденты вслепую — только после проверки что проблема решена

### 4. Проверить хвосты

Зависит от режима (см. «Режим без кода» ниже).

- Временные файлы → удалить
- Незавершённые tasks → в бэклог с приоритетом
- **Был код** → `git status`, незакоммиченные изменения, при необходимости запустить **accepting-work**. ⚠️ `workspaces/` НЕ является git-репо — git-операции (status/commit/push) делать ТОЛЬКО в `projects/<repo>`, а не в рабочей директории агента.
- **Кода не было** → пропустить git-проверки, просто убедиться что workspace чист

### 5. Отчитаться

Следуй AGENTS.md (раздел «Закрытие сессии», п. 4).

Формат:
```
🏁 Сессия завершена
📅 <дата> <время> UTC

📋 Что сделано:
- <результат 1>
- <результат 2>

⚠️ Инциденты:
- <новый/закрытый> (если есть)

📂 Коммиты:
- <хеш> — <описание>

🧹 Хвосты:
- Удалено: <файлы>
- Бэклог: <задачи>

🔜 Следующая сессия / Next steps:
- <что делать первым делом при старте>
- <контекст который нужно восстановить>

Память сохранена. До следующей сессии.
```

## Режим без кода

Если в сессии НЕ было написано/изменено кода:
- Шаг 4 (хвосты) — пропустить git-проверки
- Просто: записать итоги, проверить инциденты, отчитаться

## Валидация записей

- Записанное в memory — не пустышка, не «TODO», не placeholder
- MEMORY.md обновляется только если есть реальные изменения
- НЕ писать «Продолжить работу над X» — это не информация

## Обработка ошибок

- Что-то упало → записать в бэклог + сообщить пользователю
- Нечего делать → не создавать пустых записей

## Правила

- НЕ писать просто «✅ Session closed»
- НЕ пропускать шаги
- НЕ оставлять грязный git status (если был код)
- НЕ забывать про инциденты
- НЕ создавать пустых коммитов
- НЕ читать состояние повторно — starting-session уже это сделал при старте
- finishing-session ТОЛЬКО записывает и закрывает — не дублируй чтение из starting-session

## Связанные скилы

- **accepting-work** — приёмка работы с кодом (предшествующий шаг). Если в сессии был код и accepting-work не был запущен — запустить его.
- **starting-session** — старт сессии (симметричная пара).
