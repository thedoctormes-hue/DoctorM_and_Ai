# 🛡 AgentShield — аудит MCP-серверов и hooks

Аудит безопасности агентной среды: проверяет зарегистрированные MCP-серверы
и настроенные hooks на предмет неавторизованного доступа, опасных команд и
утечек. Дополняет `security-reviewer` (аудит кода) — фокус на инфраструктуре
агента (MCP-интеграции, lifecycle-hooks).

## Когда применять
- Перед деплоем нового агента или MCP-сервера.
- После изменения `openclaw.json` (hooks, mcp.servers).
- По запросу ЗавЛаба «проверь MCP/hooks на безопасность».
- Регулярный аудит (в составе owl-tails-watch или отдельно).

## Границы (RUL-009/010 — НЕ ТРОГАТЬ)
- НЕ трогать семантическую память (memory-gateway, ALM, onnx-embedder).
- НЕ делать ручную реиндексацию (это зона Штрейкбрехера, только с confirm ЗавЛаба).
- НЕ править `openclaw.json` (только читать/аудировать).
- Только ЧТЕНИЕ конфигов и проверка портов/сервисов.

## Чек-лист аудита

### 1. MCP-серверы
- Список зарегистрированных MCP (`openclaw mcp list` или `mcp.servers` в конфиге).
- Каждый сервер: авторизован ли? Кто владелец? Что он делает?
- Нет ли серверов с подозрительными командами (exec, curl на внешние хосты)?
- Локальные серверы: не слушают ли они на 0.0.0.0 (открытый доступ)?
- Сетевые серверы: HTTPS? Аутентификация?

### 2. Hooks (lifecycle)
- Список hooks (SessionStart/PreCompact/PostToolUse/SessionEnd и др.).
- Каждый hook: что запускает? Не выполняет ли опасные команды (rm -rf, curl | sh)?
- Не модифицирует ли конфиги/файлы вне зоны ответственности?
- Не отправляет ли данные наружу без ведома?

### 3. Порты и сервисы (через gatekeeper)
- `gatekeeper__list_ports` — слушающие порты. Нет ли неожиданных (на 0.0.0.0)?
- `gatekeeper__list_units` — systemd-юниты. Нет ли failed/неавторизованных?
- `gatekeeper__list_timers` — таймеры. Нет ли подозрительных расписаний?

### 4. Утечки
- В hooks/MCP-командах нет ли хардкодных ключей/токенов?
- Не логируются ли секреты?

## Анти-паттерны
- НЕ править конфиги «для исправления» — только докладывать ЗавЛабу.
- НЕ трогать семпамять (RUL-009/010).
- НЕ запускать найденные подозрительные команды (только анализ).

## Формат отчёта (AGENTSHIELD REPORT)
```
🛡 AGENTSHIELD REPORT (дата)
MCP-серверы: N проверено, X подозрительных
  - <имя>: <статус> — <находка>
Hooks: M проверено, Y подозрительных
  - <hook>: <статус> — <находка>
Порты: K слушает, Z неожиданных
Сервиcы: ...
Итог: OK | WARNING | CRITICAL
Рекомендации: <что исправить, кому (ЗавЛаб/владелец)>
```
