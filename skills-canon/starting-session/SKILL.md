# Session Start — Старт сессии

## Назначение

Скил запускается в начале сессии по явному триггеру ЗавЛаба. Два режима:

- **РЕЖИМ А (лёгкая сводка)** — триггеры: `привет`, `hi`, `hello`, `здравствуй`, `стартуем`. Экономный старт: краткая сводка актуального в чат, БЕЗ засирания контекста. Это основной режим для ежедневного старта (решение ЗавЛаба 2026-07-14: экономия токенов при старте).
- **РЕЖИМ Б (глубокий прогон)** — триггеры: `поехали`, `старт сессии`, `запусти сессию`, `session start`, `начинаем`, `ready to work`. Полный чек-лист: git-скан агента, утечка agents@, handoff-состояние, проверка инъекции. Реже, когда нужен полный аудит.

Определи режим по триггеру. Если неясно — делай РЕЖИМ А (лёгче).

Развёрнутый мониторинг состояния системы (диск/uptime/load/сервисы/docker/failed) при старте НЕ выполняется — он вынесен в hourly cron-монитор (агент Доминика), который сам маршрутизирует проблемы нужному агенту. НЕ лезь в инфраструктуру и НЕ лезь к другим агентам (решение ЗавЛаба 2026-07-14).

## Когда применять
- Пользователь говорит: «привет» / «hi» / «hello» / «здравствуй» / «стартуем» → Режим А (лёгкая сводка)
- Пользователь говорит: «поехали» / «старт сессии» / «запусти сессию» / «начинаем» → Режим Б (глубокий прогон)
- При начале каждой новой сессии
- При перезапуске после сбоя
- При первом контакте с агентом в сессии

## Границы применимости
- НЕ выполняет мониторинг инфраструктуры (диск/uptime/сервисы/docker) — это cron-монитор Доминики
- НЕ лезет к другим агентам колонии и их рантайм-данным (ADR-014, решение ЗавЛаба 2026-07-14)
- НЕ пушит коммиты автоматически при старте — только напоминает
- НЕ обновляет память — это зона finishing-session
- НЕ закрывает/открывает инциденты — это зона finishing-session
- Режим А — без полного git-скана всех projects/*, без handoff-json, без проверки инъекции
- Развёрнутый мониторинг состояния вынесен в hourly cron (Доминика)

## РЕЖИМ А — ЛЁГКАЯ СВОДКА (привет/hi/здравствуй/стартуем)

Быстро, дёшево, без тяжёлых операций. Состав (финальный, решение ЗавЛаба 2026-07-14):

1. **Открытые инциденты** — только РЕАЛЬНО активные (файлы со `status: open`/`active` либо БЕЗ frontmatter, т.е. untracked). Файлы `closed`/`resolved` НЕ показывать: реестр — это архив (~149 файлов, активных ~10). Парсим статус ИЗ frontmatter, а не по наличию файла (наивный `ls | tail -10` показывал мусор и терял активные — DDP 2026-07-15):
   ```bash
   INC=/root/LabDoctorM/projects/DoctorM_and_Ai/incidents
   python3 - "$INC" <<'PY'
   import os, re, sys
   d=sys.argv[1]
   total=live=0
   for fn in sorted(os.listdir(d)):
       if not fn.endswith('.md'): continue
       if fn.lower().startswith('readme') or 'template' in fn.lower(): continue
       total+=1
       t=open(os.path.join(d,fn),encoding='utf-8',errors='ignore').read()
       m=re.match(r'^---\s*\n(.*?)\n---', t, re.S)
       s='untracked'
       if m:
           sm=re.search(r'^status:\s*(\S+)', m.group(1), re.M)
           if sm: s=sm.group(1).strip().strip('"').strip("'")
       if s in ('open','active','untracked'):
           print('🔴', fn); live+=1
       elif s in ('closed','resolved'):
           pass
       else:
           print('⚠️', fn, '(status=%s)' % s)
   print('Актуальных: %d из %d' % (live, total))
   PY
   ```
   Только список, что горит.

2. **Хвосты (незавершёнка)** — лёгкая проверка своего workspace, БЕЗ полного git-скана всех projects/*:
   ```bash
   git -C /root/LabDoctorM/workspaces/mangust status -s 2>/dev/null
   ```
   Если есть незакоммиченное — одной строкой «⚠️ незакоммичено: <файлы>».

3. **Бэклог (активные задачи)** — из MEMORY.md (уже в контексте, бесплатно). Кратко: что в работе.

4. **БЛОКЕРЫ НА ЗАВЛАБЕ** — что агент ждёт от ЗавЛаба (самое важное, решение ЗавЛаба 2026-07-14). Из MEMORY.md/bэклога: висящие на ЗавЛабе решения. Примеры актуальных: Groq-ключи (ждём для фолбэка), orion-scan отчёт (03:30 МСК), фолбэк в бэклоге до отчёта.

5. **Готов к работе.**

НЕ делать в Режиме А: git-скан по всем projects/*, проверку утечки agents@, handoff-json, чтение дневника, мониторинг инфры, лезть к другим агентам.

## РЕЖИМ Б — ГЛУБОКИЙ ПРОГОН (поехали/старт сессии)

Полный чек-лист (дороже, реже). Параллельно:

0. **Персона агента** (индивидуализация):
   - Прочитать `<workspace>/IDENTITY.md` и `<workspace>/startup-profile.md` (если есть).
   - ⚠️ Скрипт `audit_lab.sh` НЕ существует (проверено). Не выполнять. Аудит myrmex.json делать вручную.
   - Строить чеклист ПОД роль агента.

1. **Инциденты:** только РЕАЛЬНО активные (тот же status-фильтр, что в Режиме А, шаг 1) — НЕ показывать `closed`/`resolved` архив:
   ```bash
   INC=/root/LabDoctorM/projects/DoctorM_and_Ai/incidents
   python3 - "$INC" <<'PY'
   import os, re, sys
   d=sys.argv[1]
   total=live=0
   for fn in sorted(os.listdir(d)):
       if not fn.endswith('.md'): continue
       if fn.lower().startswith('readme') or 'template' in fn.lower(): continue
       total+=1
       t=open(os.path.join(d,fn),encoding='utf-8',errors='ignore').read()
       m=re.match(r'^---\s*\n(.*?)\n---', t, re.S)
       s='untracked'
       if m:
           sm=re.search(r'^status:\s*(\S+)', m.group(1), re.M)
           if sm: s=sm.group(1).strip().strip('"').strip("'")
       if s in ('open','active','untracked'):
           print('🔴', fn); live+=1
       elif s in ('closed','resolved'):
           pass
       else:
           print('⚠️', fn, '(status=%s)' % s)
   print('Актуальных: %d из %d' % (live, total))
   PY
   ```

2. **Незавершённая работа агента** (напоминание, НЕ действие):
   - Вспомнить СВОЮ работу, которую не довёл до конца, и доложить ЗавЛабу.
   - AGENT_ID (приоритет: env → IDENTITY.md `id:` → имя каталога workspace):
     ```bash
     AGENT_ID="${AGENT_ID:-${LAB_AGENT:-}}"
     [ -z "$AGENT_ID" ] && [ -f IDENTITY.md ] && \
       AGENT_ID="$(grep -m1 -E '^id:' IDENTITY.md | sed -E 's/^id:[[:space:]]*//; s/[[:space:]]*$//')"
     AGENT_ID="${AGENT_ID:-$(basename "$PWD")}"
     ```
   - Корзины СВОЕГО по всем `projects/*` с .git:
     ```bash
     for d in /root/LabDoctorM/projects/*/; do
       [ -d "${d}.git" ] || continue
       base=""
       for cand in origin/main origin/master; do
         git -C "$d" rev-parse --verify "$cand" >/dev/null 2>&1 && { base="$cand"; break; }
       done
       [ -z "$base" ] && base="$(git -C "$d" rev-parse --abbrev-ref --symbolic-full-name @{upstream} 2>/dev/null)"
       if [ -z "$base" ]; then
         echo "$(basename "$d"): ⚠️ НЕТ БАЗЫ — не могу сравнить с main"
         continue
       fi
       has=$(git -C "$d" for-each-ref --format='%(refname:short)' refs/heads/ 2>/dev/null | grep -c "^$AGENT_ID/" || true)
       [ "${has:-0}" -gt 0 ] && git -C "$d" status -s 2>/dev/null | while read l; do echo "$(basename "$d"): [DIRTY] $l"; done
       for b in $(git -C "$d" for-each-ref --format='%(refname:short)' refs/heads/ 2>/dev/null); do
         git -C "$d" log --oneline "$base..$b" --author="$AGENT_ID" 2>/dev/null | \
           while read h m; do echo "$(basename "$d") [$b]: $h $m"; done
       done
       for b in $(git -C "$d" for-each-ref --format='%(refname:short)' refs/heads/ 2>/dev/null); do
         git -C "$d" log --oneline "$base..$b" --author="agents@labdoctorm.ru" 2>/dev/null | \
           while read h m; do echo "$(basename "$d") [$b]: ⚠️ AGENTS@ $h $m"; done
       done
     done
     ```
   - Если нашлось своё (а)/(б)/(в) — ⚠️ напомнить. Если (г) — отдельный ⚠️ про утечку agents@.
   - НЕ коммитить/пушить автоматически при старте — только напомнить.

3. **Проверка инъекции контекста (без дублей):**
   - Убедись, что каждый bootstrap-файл инъектирован РОВНО ОДИН раз. Дубли = context sprawl (жжёт токены).
   - Если видишь дублирование — СООБЩИ ЗавЛабу, НЕ исправляй сам.
   - Вывод скила НЕ должен ре-стейтить то, что уже в инъекте.

4. **Semантический поиск / fact-check** (точки входа, НЕ мониторинг):
   - ТОЛЬКО MCP `memory-gateway` (инструмент `memory-gateway__search_memory`, бэкенд ALM/AnythingLLM). Прямые вызовы `lab_search.py` / `labsearch` / `mcp-memory :8087` / `onnx-embedder :8082` / native `memory_search` / REST к AnythingLLM — ЗАПРЕЩЕНЫ (см. APPEND_SYSTEM.md).
   - Правило «НЕ СУЩЕСТВУЕТ ≠ DOWN» (см. полную версию в v3.3.3).

5. **Anti-loop state:** не хранится как файл по умолчанию. Проверяй через скилл anti-loop; если не задано — `{}`.

6. **Handoff-состояние** (для downstream-скиллов):
   - Записать `memory/<date>-session-state.json`: `{ agent, role, checks_summary, open_incidents, open_questions, relevant_services }`.
