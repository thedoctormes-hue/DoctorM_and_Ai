# Report — двойной финальный отчёт

Скил вызывается, когда пользователь пишет **report** / **отчёт** в конце задачи.
Агент выдаёт итог в двух частях за один раз: профессионально и по-человечески,
чтобы не пришлось просить «сумманизируй / понятно / без воды».

## Когда применять
- Пользователь написал `report` или `отчёт` (явный триггер).
- Задача фактически выполнена и есть что отчитывать: результаты, правки, выводы, статус.
- Язык отчёта — русский.

## Границы
- НЕ сочинять факты. Отчёт только на том, что реально сделано и проверено в сессии (PAT-05).
- НЕ использовать таблицы (PAT-06) — только списки и отступы, даже в профи-части.
- Скил не запускает действий: не правит код, не перезапускает сервисы, не трогает конфиги.
- Если задача не закрыта — честно сказать об этом, не выдумывать завершение.

## Чек-лист
1. Собрать факты сессии: что сделано, какие решения, блокеры, хвосты.
2. **Часть 1 — Профессиональный отчёт.** Технически грамотно, по делу, без воды.
   Структура: что / как / результат / статус. Списки, не таблицы.
3. **Часть 2 — Понятное резюме.** Тот же смысл человеческим языком для вайбкодера:
   ноль жаргона, аналогии при надобности, так, чтобы не пришлось переспрашивать «объясни».
4. Порядок строго: Часть 1 → Часть 2.
5. Язык — русский. Обе части в одном сообщении.

## Анти-паттерны
- ❌ Одна часть вместо двух (забыть про «для человека»).
- ❌ Вода и вступления («Итак, дорогой друг…»), повторы.
- ❌ Таблицы в Telegram.
- ❌ Выдумка фактов, когда реального результата нет.
- ❌ Перепутанный порядок (сначала простое, потом профи).
- ❌ Английский в теле отчёта — язык русский.

## Шаблон вывода

**🛠 Часть 1 · Профессиональный отчёт**
- Что сделано: …
- Как реализовано: …
- Результат / статус: …
- Блокеры / хвосты: …

**🧭 Часть 2 · Понятное резюме**
- Коротко по-человечески: что получилось и зачем это нужно.
