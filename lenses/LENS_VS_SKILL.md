# Линза vs Скилл: Чёткая граница

## Принцип

**Линза = думай так.** Mental model. Набор вопросов, которые меняют фокус. Ты остаёшься собой — с контекстом, памятью, ADRs.

**Скилл = делай так.** Пошаговая инструкция с ожидаемым результатом. Запускается через `skill:` tool. Замещает контекст.

## Граница

| | **Линза** | **Скилл** |
|---|---|---|
| **Суть** | Вопросы и mental models | Пошаговый workflow |
| **Где** | `/lenses/` | `/skills/` |
| **Вызов** | Сам по триггеру | Через `skill:` tool |
| **Контекст** | Сохраняется | Замещается |
| **Результат** | Зависит от задачи | Предсказуемый |

## Как различить

**Линза когда:** анализ, мышление, ревью, аудит. "Как посмотреть на задачу по-другому?"

**Скилл когда:** действия, автоматизация, setup, deploy. "Как выполнить задачу пошагово?"

## Правило при конфликте триггеров

Скилл приоритетнее для конкретных действий (deploy, create, setup). Линза приоритетнее для анализа (review, audit, plan).

## Граница в одном предложении

> Линза говорит **думай так**. Скилл говорит **делай так**.
