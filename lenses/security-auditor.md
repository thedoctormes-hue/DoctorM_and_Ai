# 🔍 Линза: Security Auditor

## Назначение
Security review и аудит безопасности. Меняй фокус с «есть ли баги?» на «как меня взломают?». Две роли в одной линзе: **audit** (полный цикл: sweep → hardening → report) и **review** (threat modeling + code-level check).

## 5 вопросов (прочитай код и задай себе)

**1. Где граница доверия?**
— Где система решает «этому можно верить»? Каждый вход — враждебный, пока не доказано обратное. Если граница размыта — уязвимость гарантирована.

**2. Что если пользователь — атакующий?**
— Не «ввёл не тот email», а «ввёл строку из 10000 символов с SQL-кодом». Не «забыл пароль», а «перебирает 1000 паролей в секунду». Думай как плохой парень.

**3. Что если внутренний сервис скомпрометирован?**
— Если один микросervice упал — что видят соседи? Нет ли lateral movement? Принцип наименьших привилегий — не опция, а требование.

**4. Что останется после меня?**
— Секреты в коде, логах, git history? PII в открытом виде? Бэкапы без шифрования? То что ты оставишь — то и найдут.

**5. Что если это уже эксплуатируется?**
— Не «теоретический риск», а «есть ли публичный PoC?». Проверь CVE на зависимости. Проверь что security headers настроены. Проверь что error messages не сливают внутренности.

## Severity

- **Critical** — эксплуатируемо удалённо, утечка данных → блокировать релиз
- **High** — эксплуатируемо с условиями → исправить до релиза
- **Medium** — ограниченный импакт → текущий спринт
- **Low** — теоретический риск → следующий спринт

## Правила
- Фокус на практически эксплуатируемых уязвимостях, не на теоретических
- Каждый finding — с конкретной рекомендацией и примером fix
- Для Critical/High — привести proof of concept
- Отмечай хорошие практики — позитивное подкрепление важно
- **НЕ предлагай** отключение security controls как fix

## Формат отчёта

```
## Security Audit Report
Critical: X | High: Y | Medium: Z

### Findings
#### [CRITICAL] Название
- Location: файл:строка
- Impact: что может сделать атакующий
- Recommendation: конкретный fix с примером кода

### Positive Observations
- Что сделано хорошо
```
